## Weight Rule

Use scene weights to protect reading flow, not to force equal length. A short bridge scene can be 200-400 characters; a core conflict scene can be 800-1200 characters; a climax or emotional turn can be longer if the chapter target allows it.

## Temporary Rule

Keep `temporary: true` unless the author explicitly asks to promote the pack into a permanent narrative-system document.

## New Execution Rule

The execution pack must transform outline-level style chains into drafting controls. It should not merely repeat the outline. It must tell the prose-draft skill how to:

- make the chapter move through concrete life pressure;
- make misread semantics build naturally;
- keep reader, viewpoint, and support-character cognition separate;
- release humor through setup, escalation, landing, and echo;
- protect one pressure point per core scene while reducing joke density;
- preserve one main ending hook and keep lighter afterbeats secondary;
- keep important support characters from becoming only helpers, explainers, or joke receivers;
- keep dialogue from becoming clean Q&A;
- introduce later reversal/evidence material lightly;
- mark anti-AI risks before prose is generated.
- assign scene grades and word ranges so only core reader-experience scenes receive full drafting weight.

## System-Source Boundary

`核心笑点递进表` may define setup, escalation, landing position, and echo function, but the landing must remain a structural slot rather than a finished punchline from non-system material. `关键对话断裂点` may define interruption, repair, silence, dodge, or answer mismatch, but must not provide full finished dialogue from non-system material.
