# Draft To Public Workflow

## 1. Freeze

Record:

- core events;
- scene order and function;
- character position, relationships, decisions, and behavior logic;
- POV and knowledge state;
- reader information boundary;
- required tension and uncertainty;
- ending function and continuation interface.

If any item is unclear, use `ASK-A`.

## 2. Register Intervention Nodes

|Node|Trigger|
|---|---|
|ASK-A|Frozen layer is unclear|
|ASK-B|Confirmed aesthetic rules conflict|
|ASK-C|L5 is required or L4 changes a frozen item|
|ASK-D|Two expression directions protect different values and cannot be ranked|
|ASK-E|New fact, motive, ability, or relationship history is required|
|ASK-F|POV or information boundary must change|
|ASK-G|Primary tone is unclear and changes the whole scene|
|ASK-H|Ending function or continuation interface changes|
|ASK-I|Method migration boundary lacks author confirmation|
|ASK-J|Public Skill architecture cannot be resolved from existing responsibilities|

## 3. Audit Functional Fragments

For each fragment record:

- function;
- required facts;
- character purposes;
- relationship start and end;
- reader effect;
- current expression mechanism;
- concrete aesthetic problems;
- L0-L5 decision;
- reconstruction task;
- prohibited changes;
- intervention status;
- depth acceptance questions.

Do not use “AI feel” as the final diagnosis. Name the observable mechanism.

## 4. Select Methods

Choose one primary method and at most two auxiliary methods from the formal expression library.

Common routing:

|Problem|Primary method|
|---|---|
|Standard Q&A|EXPRESSION-P0-001|
|Information list|EXPRESSION-P0-002|
|Repeated explanatory narration|EXPRESSION-P0-003|
|Abstract character judgment|EXPRESSION-P0-004|
|Consecutive conclusions|EXPRESSION-P0-005|
|Abrupt mixed tone|EXPRESSION-P0-006|
|Over-explained landing|EXPRESSION-P0-007|
|Formulaic metaphor|EXPRESSION-P0-008|
|Paragraph organization failure|EXPRESSION-P0-009|
|Whole-fragment expression failure|EXPRESSION-P0-010|

## 5. L3 Procedure

1. Write fragment function and frozen constraints.
2. Split Draft material into facts, actions, purposes, relationship changes, and narration functions.
3. Hide the original sentences.
4. Select a new engine: action, conflict word, object, or character purpose.
5. Reassemble trigger, reaction, relationship change, information consequence, and landing.
6. Restore necessary transitions.
7. Compare facts and expression skeleton, not sentence polish.

## 6. L4 Reconstruction Card

Before prose, record:

- fragment function;
- required facts;
- character purposes;
- relationship movement;
- POV and information boundary;
- primary reader effect;
- prohibited changes;
- prior and next interface;
- original failure mechanism.

Then:

1. close the original paragraph;
2. choose a new initiating action and conflict;
3. design at least two information-entry types;
4. design line win-rate and relationship movement;
5. assign narration a specific new function;
6. choose one landing;
7. write from blank space;
8. reopen the Draft only for fact verification.

## 7. Assembly

After fragment passes:

- remove duplicated setup and conclusions at fragment joins;
- keep POV and knowledge continuous;
- keep the selected primary tone continuous;
- retain useful L0 material;
- ensure local reconstruction is not flattened during whole-chapter smoothing.
