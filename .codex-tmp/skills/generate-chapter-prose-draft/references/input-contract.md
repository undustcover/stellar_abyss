# Input Contract

Use this file when generating, revising, or checking a whole-chapter prose draft.

Formal output template:

```text
System_alpha/00_总控台/02_模板库/TPL-CHAPTER-PROSE-DRAFT_正文草稿交付版模板.md
```

## Read Order

1. Chapter creative control plan
   - Prefer `type: chapter_creative_control_plan`.
   - Use it as the authority for division of labor, artifact references, conflict priority, ASK_USER triggers, and token boundaries.
2. Chapter narrative expression contract
   - Use it as the authority for author intent, reader information boundary, POV filtering, hidden/revealed lines, and hard prohibitions.
3. Chapter detail outline
   - Use it as the authority for scene order, scene function, plot movement, and ending state.
4. Scene performance contract
   - Prefer `type: scene_performance_contract`.
   - Use it as the authority for prose texture, wording, dialogue naturalness, narration boundaries, and anti-AI controls.
5. Chapter execution pack
   - Prefer `type: chapter_execution_pack`.
   - Use it as the dispatch index for word target, scene weights, transitions, risk routing, intent-control interface fields, and generation order.
6. Character intent result
   - Prefer `type: character_intent_result`.
   - Formal template: `System_alpha/00_总控台/02_模板库/TPL-CHARACTER-INTENT-RESULT_人物意图结果模板.md`.
   - Use the chapter-level character intent baseline, key-fragment tracks, `CONTINUE`/`RESUME` action, runtime-state update suggestions, and execution-pack light-reference coverage.
   - If the result is `ASK_USER`, do not draft. Return control to intent alignment.
7. Chapter runtime state
   - Use it for confirmed intent rules, resolved divergences, pending divergences, resume points, and generated fragment status.
8. Optional single-scene execution brief
   - Use only for one high-risk scene explicitly included by the user.
   - Do not require one for every scene.
9. Approved common rule files
   - Use only reusable rule files that belong to the skill, setting, event, or narrative system.
   - Always include `public-derived-anti-ai-prevention.md` and verify its mapped `method ID@version` against the formal expression method library before prose generation.
10. Author request for the current turn
   - Use for length, tone, rough/polished mode, or specific repair focus.
   - Do not let it silently override world facts or information boundaries.

## Allowed Source Boundary

Generate the draft only from:

- chapter execution packs;
- chapter creative control plans;
- chapter narrative expression contracts;
- scene performance contracts;
- character intent results generated by `control-character-intent`;
- chapter runtime states;
- chapter detail outlines;
- confirmed unit context packages;
- chapter outline cards;
- character cards;
- event cards;
- setting cards;
- approved common rule files inside the system;
- explicit author requirements for the current task.

Do not use external prose drafts, comparison reports, finished sample text, or non-system writing material as generation input.

## Required Inputs

- One chapter creative control plan.
- One chapter narrative expression contract.
- One chapter detail outline.
- One scene performance contract, unless the author explicitly chooses to draft without it.
- One lightweight chapter execution pack or equivalent dispatch plan.
- For key fragment drafting: one `character_intent_result` with `CONTINUE` or `RESUME`, unless the task is only to assemble already generated fragments.
- Chapter ID, title or working title.
- Word target. Default to around 3500 Chinese characters if absent.

## Required Output Shape

Use the formal `正文草稿交付版模板` as the product shape. The generated instance may omit `## 模板接口`, but must preserve YAML source fields, `## 正文`, and `## 生成备注` for system delivery.

## Optional Inputs

- Unit context package.
- Prior or next chapter summary.
- One high-risk scene execution brief.
- Short excerpts from character, event, or setting cards.
- Approved reusable rule files that are part of the system.

## Missing Input Handling

If only a chapter detail outline is provided, stop and ask for the missing creative control plan, narrative expression contract, scene performance contract, and execution pack. If the user explicitly asks for a rough draft anyway, draft conservatively and add the missing-artifact risk to `生成备注`.

If only raw cards are provided, do not generate a full chapter. Request or recommend chapter detail outline and chapter execution pack first.

If a key fragment lacks coverage in the character intent result, do not draft it. Request or run `control-character-intent` first.

## Conflict Handling

If inputs conflict, follow the chapter creative control plan's conflict priority and ASK_USER triggers. If no priority exists, ask the author instead of silently choosing. Record unresolved conflicts in `生成备注`.

Common conflicts:

- Creative control plan does not define which artifact has priority.
- Narrative expression contract and outline disagree about what may be revealed.
- Execution pack and outline disagree about scene ending.
- Character intent result conflicts with execution pack or runtime state.
- A scene requires knowledge the viewpoint character does not have.
- Author asks for a reveal marked hidden.
- Requested length cannot fit all scene weights.
- A high-risk scene requires a setting detail not confirmed upstream.
- Input package includes external prose drafts, comparison reports, finished sample text, or other non-system writing material as if it were generation context.

## Source Pollution Handling

If non-system writing material is provided before drafting, exclude it from generation and record this in `生成备注`. If the execution pack itself contains suspected non-system finished material, avoid reproducing it as prose and mark the risk in `生成备注`.
