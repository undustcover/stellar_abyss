## 详细问题

### P0C3-ISSUE-001：正文文字质量仍然很差，AI 感强烈

现象：

- 正文虽然遵守信息边界和人物意图，但句子仍然显得工整、顺滑、会总结。
- 马雯雯的吐槽、旁白判断和场景收束仍有明显 AI 组织感。
- “不显影”“不越界”通过了，但“像作者要的小说”没有通过。

待讨论：

- 当前 `generate-chapter-prose-draft` 是否只是合格的总导演调度器，而不是合格的文字导演。
- 现有导演层文本质量规则是否太抽象，无法约束具体句子。
- 是否需要把逐句校准后的规则写入 `director-prose-quality-rules.md`、正文草稿模板和生成前检查清单。

### P0C3-ISSUE-002：作者没有参与文字质量讨论

现象：

- 本轮链路测试在产物齐备后自动生成了正文草稿。
- 作者没有在正文生成导演层的“文字质感、示例句、吐槽方向、对白口感、AI 感判断”上参与讨论和确认。
- 文字质量已经剥离到正文生成 Skill 的导演层 reference，不能再把文字质量共创主要放回场景演出契约。

待讨论：

- 是否必须在 `generate-chapter-prose-draft` 中新增“正文生成前导演层文字质量共创”环节。
- 哪些章节必须要求作者参与，哪些章节可沿用已有规则。
- 是否允许示例句；如果允许，示例句如何避免变成正文成品污染。

### P0C3-ISSUE-003：缺少自动前章承接状态

现象：

- P0C3 开头像凭空进入云水汇门口，没有明确承接 P0C2 的结尾状态。
- 人物站位、上一章末尾动作、情绪余温、未完成对话没有进入正文生成约束。
- 导致开头虽然“场景合理”，但和前一章之间缺少连续呼吸。

待讨论：

- 是否为正文链路新增由系统自动提取的 `前章承接状态 / previous_chapter_handoff` 字段或产物。
- 该字段归属哪个 skill：章节创作总控规划、章节细纲、章节执行包、正文草稿模板，还是单独 runtime state。
- 正文生成前是否必须读取上一章结尾原文或上一章结尾状态卡；仅在找不到版本、版本冲突、承接状态与本章契约冲突或需要改变前章事实时 ASK_USER。

### P0C3-ISSUE-004：逐句校准必须进入模板回流

现象：

- 仅靠现有总结性规则无法定位每一句为什么 AI。
- 后续需要逐句标注“保留 / 删除 / 改写 / AI 感原因 / 作者修正方向”。
- 校正结果不能只停留在一次性改稿，必须回流到模板和 skill。

待讨论：

- 建立逐句校准记录模板；但顺序应排在正文生成导演层作者共创之后。
- 将校准问题分类，例如：AI 总结腔、口吻不对、过度解释、上下文断裂、误会不好笑、人物不像人、句子太顺。
- 确认哪些规则属于本章局部，哪些应回流导演层通用规则。

### P0C3-ISSUE-005：是否需要新建文字质量 skill

待比较方案：

|方案|说明|优点|风险|
|---|---|---|---|
|增强 `generate-chapter-prose-draft`|在正文草稿生成 skill 内加入作者参与的导演层文字质量共创|符合当前职责分工，少一个产物|skill 继续膨胀，规则可能抽象化|
|强化场景演出契约|只处理本章或关键场景相对通用导演规则的差异|保留场景层局部差异|不能承担通用文字质量共创，否则会回到旧分工|
|新增 `calibrate-prose-line-quality` 类 skill|专门做逐句校准、规则提取和回流|最适合沉淀作者文字规则|新增链路节点，可能增加 token 成本|

当前暂定：

- 不立即新建 skill。
- 第一优先级是补 `generate-chapter-prose-draft` 的作者参与型导演层文字质量共创。
- 再根据新一轮正文结果决定是否做逐句校准，以及是否新增独立文字质量校准 skill。
