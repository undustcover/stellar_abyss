# 第三章 Draft→Public 测试与迭代任务

## 一、任务目标

使用第三章 Draft 验证新的 Draft→Public 表达重构链路能否：

1. 保留已经合格的剧情、场景、人物、POV、信息和章末功能；
2. 准确识别词汇、句式、对白、段落、表达路径、基调和审美决策问题；
3. 根据问题严重度执行 L0—L4 分级处理；
4. 将正式作者审美原则转化为片段级表达重构动作；
5. 避免 Public 再次退化为同义词替换、原句拆短或逐句映射；
6. 测试发现问题后，准确回写对应知识库、Skill、模板或索引；
7. 每次迭代后回到原失败片段复验，形成可追踪闭环；
8. 在需要作者裁决的节点立即停止并请求协助。

本任务不默认修改 Draft 的剧情结构，也不直接从头盲写整章。

## 三、已确认且默认冻结的边界

作者已确认第三章 Draft 的总体场景、解构、人物定位等上游创作决策合格。

本轮默认冻结：

- 核心事件；
- 场景顺序与场景功能；
- 人物定位和人物关系；
- 人物行为逻辑；
- 马雯雯限知 POV；
- 读者信息边界；
- 云水汇真实情况仍不揭晓；
- 马木不直接定性、不成为全知破案者；
- 陈艾艾、陈科、陈灏和工作人员的基本场景作用；
- 章末持续性不安和“明日反馈”后续接口。

如测试发现只有改变上述内容才能解决表达问题，必须触发作者介入，不得静默修改。

## 五、问题归因路由

每个测试问题必须选择一个“主归因模块”，可以记录次要影响模块，但禁止同时向多个模块无差别写入。

|问题表现|主归因模块|可能的次要模块|处理方式|
|---|---|---|---|
|不知道作者保护什么、拒绝什么|MOD-KB-AESTHETIC|MOD-KB-CASE|建立候选解释并请求作者校准|
|知道不合格，但不知道怎样重写句式或段落|MOD-KB-EXPRESSION|MOD-SKILL-PUBLIC|提取或创建表达重构方法|
|喜剧机制、包袱、误读或回调失败|MOD-KB-HUMOR|MOD-KB-EXPRESSION|修正方法适用条件或生成步骤|
|已有方法，但系统没有执行|MOD-SKILL-PUBLIC|MOD-TPL-PUBLIC|修正 Skill 工作流或强制字段|
|执行了方法，但仍只是换词|MOD-SKILL-PUBLIC|MOD-TPL-PUBLIC|加强片段重构步骤和修改深度门禁|
|模板无法区分 L2／L3／L4|MOD-TPL-PUBLIC|MOD-DESIGN-PUBLIC|补字段和判断标准|
|L4 实际触碰冻结层|MOD-DESIGN-PUBLIC|MOD-TPL-PUBLIC|触发 ASK_USER，不得继续|
|Draft 本应在首次生成时避免该问题|MOD-SKILL-DRAFT-QUALITY|MOD-SKILL-DRAFT-CHECK|回写首次生成规则和门禁|
|问题来自人物动机或认知而非表达|人物意图控制模块|MOD-SKILL-PUBLIC|停止 Public，回人物意图控制|
|问题来自场景顺序、章节功能或信息边界|章节总控／细纲／场景演出模块|MOD-DESIGN-PUBLIC|触发 L5 和 ASK_USER|
|新增模块后无法被后续模型找到|MOD-INDEX-DESIGN|MOD-STATE-AESTHETIC|更新总索引和 CURRENT|
|正式规则与案例证据不一致|MOD-KB-AESTHETIC|MOD-KB-CASE|建立冲突记录，作者确认前不覆盖|

## 七、作者介入地图

测试开始前预登记以下节点。

|节点 ID|触发条件|触发后停止范围|必须向作者说明|
|---|---|---|---|
|ASK-A|无法判断某内容是冻结层还是 Draft 临时表达|停止当前片段及后续审计|不变内容、争议内容、不同选择的影响|
|ASK-B|正式审美规则在当前片段发生冲突|停止当前片段重构|冲突规则、各自保护价值、候选优先级|
|ASK-C|某片段需要进入 L5|停止全部正文生成|为什么 L0—L4 无法解决、将改变哪些冻结层|
|ASK-D|两个表达方向分别保护不同作者价值且无法自动裁决|停止当前片段|两个方向的读者效果和风险|
|ASK-E|重构需要新增事实、人物动机、能力或关系历史|停止当前片段|缺失事实及不补充时的影响|
|ASK-F|重构需要改变 POV 或信息释放边界|停止全部正文生成|谁会新增知道什么、读者收益和悬念损失|
|ASK-G|轻喜、压力、不安等主要基调优先级不清|停止受影响场景|每种优先级如何改变整场读感|
|ASK-H|重构会改变章末第一功能或后续接口|停止章末及整章装配|原章末功能、新方案变化和长线影响|
|ASK-I|候选表达方法是否可迁移无法由既有作者确认推出|停止正式入库，不阻塞其他片段审计|候选方法、来源证据、适用与不适用条件|
|ASK-J|决定创建或扩展 Public 专用 Skill 时存在架构分歧|停止 Skill 实施，不阻塞知识审计|新建 Skill 与扩展旧 Skill 的职责差异|

触发后必须：

1. 停止；
2. 报告节点和片段；
3. 列出已确认不变项；
4. 提出不超过三个选项；
5. 等待作者回答；
6. 记录确认结果后恢复。

## 八、测试阶段

### T2：表达知识覆盖矩阵

目标：判断现有知识库是否足以指导每个片段的 L2—L4 重构。

每个片段检查：

- [ ] 现有审美原则能否准确判断问题；
- [ ] 是否存在可执行的表达方法；
- [ ] 是否只有喜剧方法而缺少通用表达方法；
- [ ] 是否需要从确认案例提取候选方法；
- [ ] 是否需要作者校准；
- [ ] 问题属于知识缺失还是 Skill 未执行。

产物：`ART-COVERAGE`。

通过标准：

- 每个高风险片段至少有一个明确判断来源；
- 每个 L2—L4 问题都有执行方法或明确的知识缺口；
- 不用“AI 感重”代替具体问题分类。

### T3：表达重构候选方法提取

目标：补齐审美判断到表达动作之间的执行层。

优先候选方法：

1. 标准问答转为关系驱动对白；
2. 信息清单重新分配入口；
3. 解释性旁白转为现场证据；
4. 人物判断转为行动证词；
5. 结论链转为认知运动；
6. 公式比喻的现实锚点检查与重构；
7. 轻喜与压力通过同一材料共存；
8. 段落落点与人物余波；
9. L3 段落重组操作法；
10. L4 片段表达重构卡。

每条候选方法必须包含：

- 目标问题；
- 识别信号；
- 必须保留；
- 适用等级；
- 重构机制；
- 执行步骤；
- 适用条件；
- 不适用条件；
- 失败边界；
- 修改深度检查；
- 自检问题；
- 抽象来源案例。

作者介入：

- 无法从已有作者确认推出适用边界时触发 ASK-I；
- 作者确认前只作为测试候选，不写入正式 L3 方法库。

产物：`ART-METHOD-CANDIDATE`。

### T4：Public 执行 Skill 承载决策

目标：确定新流程由哪个 Skill 执行。

候选：

|方案|说明|风险|
|---|---|---|
|A 新建 `reconstruct-draft-to-public`|职责专一，直接支持合格层冻结、片段审计、L0—L5 和三类门禁|新增 Skill 和维护成本|
|B 扩展 `revise-draft-common-rules`|复用“保留发生事实、重做读者读法”的现有基础|现有定位只处理局部失败场景，易与 Public 整章装配职责冲突|
|C 扩展 `generate-chapter-prose-draft`|减少 Skill 数量|会混淆首次 Draft 生成与 Draft→Public 审美重构|

默认建议：优先验证 A；不在测试证据不足时直接实施。

若方案选择存在分歧，触发 ASK-J。

产物：在 `ART-ITERATION` 记录架构决定；确认后再创建或修改 Skill。

### T5：高风险片段冒烟测试

整章生成前必须先测试两个片段。

#### T5.1 F04 马木盘问片段

验证重点：

- 标准问答能否转为关系驱动对白；
- 风险信息是否重新分配入口；
- 马木是否不再宣读完整反诈清单；
- 陈科是否通过主动辩护、回避和动作泄露信息；
- 陈艾艾的情绪是否改变对话权；
- 是否真正达到 L3 或 L4，而不是删几个问题。

通过标准：

- 与 Draft 不逐句对应；
- 保留全部必要事实；
- 信息入口至少发生两类以上变化；
- 对话同时表现人物目的和关系变化；
- 不改变冻结层。

产物：`ART-SMOKE-S3`。

#### T5.2 F05—F06 工作人员回拉片段

验证重点：

- 低危不安是否由服务行为、称呼、物件、站位和回应产生；
- 是否减少旁白宣布风险；
- 工作人员是否不再只是功能角色；
- 马雯雯保持限知；
- 礼貌、体贴与控制感是否由同一材料产生。

通过标准：

- 不新增云水汇真相；
- 至少一个原旁白结论改由现场证据承担；
- 工作人员的语言和动作同时改变陈科及家属关系；
- 不依靠恐怖气氛词制造不安；
- 达到 L3 或 L4。

产物：`ART-SMOKE-S4`。

停止条件：

- 任一片段仍主要是换词；
- 方法无法执行；
- 触发作者介入节点；
- 出现新增事实或冻结层变化。

## 九、知识库创建与更新规则

### 9.1 何时创建表达重构方法库

同时满足以下条件才创建 `MOD-KB-EXPRESSION`：

- 至少有两条非喜剧通用候选方法；
- 方法能跨两个不同片段使用；
- 每条方法包含识别信号、步骤、失败边界和深度检查；
- 作者确认方法边界，或明确允许以 L1 候选状态进入测试库；
- 不包含作者原句和第三章专属方案。

### 9.2 何时更新作者审美库

只有出现新的“保护价值或拒绝倾向”，并经过作者确认时更新。

不因以下情况新增审美原则：

- Skill 没有执行已有规则；
- 模板缺少字段；
- 某个句子写得不好；
- 一次性的词汇偏好；
- 某个表达方法缺少步骤。

### 9.3 何时更新笑话方法库

仅当问题属于喜剧机制、素材入口、递进、落点、回调或喜剧失败边界时更新。

### 9.4 何时更新 Draft Skill

只有确认某问题本应在第一次 Draft 生成时被现有规则拦截，且不是 Public 独有的审美加工时，才回写：

- `MOD-SKILL-DRAFT-QUALITY`
- `MOD-SKILL-DRAFT-CHECK`

### 9.5 何时更新 Public Skill

以下问题归 Public Skill：

- 未按片段审计；
- L2—L4 执行不足；
- 已有方法未调用；
- 逐句映射；
- 门禁运行不完整；
- 中断节点被忽略；
- 整章装配破坏局部重构效果。
