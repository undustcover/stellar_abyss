# C03 章节创作总控规划

## 本章总目标

|项目|内容|
|---|---|
|读者最终体验|读者和马雯雯仍基本贴着会所/高级服务场所框架理解现场，但开始觉得云水汇高级得过分、周到得有些怪。|
|作者核心意图|P0C3 不是反诈显影章，也不是更脏的会所章；本章要把“特殊会所误会继续加深，但高级感开始变怪”固定为后续产物的总边界。|
|本章不能写成|诈骗/反诈/投资/项目显影；低级、污秽、情色加深；马雯雯破案；陈艾艾解释真相；马木专业反诈；陈灏异常说明者。|

## 产物分工

|产物|负责内容|禁止承担|必须引用|
|---|---|---|---|
|章节级叙事表达契约|作者意图、叙事边界、读者信息边界、POV 过滤、误读方向和高层读感|剧情顺序、正文句法、执行调度|P0 临时单元级叙事表达契约；共创讨论记录|
|章节细纲|剧情骨架、场景顺序、行动推进、信息变化、认知差、承压点和人物意图风险预埋|作者审美裁决、去 AI 化、人物意图轨道|本规划；章节级叙事表达契约|
|场景演出契约|本章或关键场景的文字表达差异、句式节奏、对白自然化、旁白边界和局部去 AI 化重点|剧情改写、人物动机裁决、执行包调度、导演层通用规则全文|本规划；章节契约；章节细纲；导演层文本质量规则|
|人物意图控制|人物言行、认知、动机、关键片段 CONTINUE/ASK_USER/RESUME|整体文风、剧情重排、正文润色|本规划；章节契约；细纲相关片段；演出契约相关规则|
|章节执行包|产物索引、场景权重、生成顺序、转场状态、风险路由和人物意图覆盖状态|复述全部上游、替代正文生成、补写人物意图轨道|本规划；章节契约；细纲；演出契约；人物意图结果|
|正文草稿生成|最终总导演式合成正文，并执行导演层可复用文本质量规则|重建上游产物、静默裁决业务冲突、把生成备注写进正文|本规划；章节契约；细纲；演出契约；执行包；人物意图结果；运行态|

## 导演层文本质量规则读取关系

|项目|规则|
|---|---|
|规则库位置|`.codex-tmp/skills/generate-chapter-prose-draft/references/director-prose-quality-rules.md`|
|归属层级|正文草稿生成 Skill 的 reference，不是独立 skill，不是本规划正文内容|
|场景演出契约如何使用|只用于判断哪些规则属于通用规则；场景演出契约只记录本章新增、加强、例外、冲突或关键场景差异|
|正文草稿生成如何使用|必须读取并执行通用文本质量规则，再叠加场景演出契约中的本章差异|

|禁止行为|不得把导演层规则整包复制到本规划、场景演出契约、章节执行包或正文生成备注中|

## 互补关系

|关系|说明|
|---|---|
|章节契约 -> 本规划|章节契约锁定作者确认的表达边界；本规划把这些边界转成后续 skill 分工、互参和冲突规则。|
|本规划 -> 章节细纲|细纲只负责场景骨架和读者体验曲线，不重新判断本章是不是反诈显影或会所加脏。|
|本规划 -> 场景演出契约|演出契约只解决文字如何呈现“高级得怪、父女冷硬、误会继续加深”，不改变剧情和人物动机。|
|本规划 -> 人物意图控制|陈艾艾/陈科父女冷硬但底层相爱、陈科体面防御、马雯雯 POV 边界和陈灏熟客第一印象必须进入意图检查。|
|本规划 -> 章节执行包|执行包只做调度索引和风险路由，不能把章节契约、细纲、演出契约或人物意图结果扩写成总包。|
|本规划 -> 正文草稿生成|正文生成必须直接协调六类产物，不得单一依赖执行包，也不得把生成备注、分析标签或意图轨道写入正文。|

## 冲突优先级

|优先级|产物/来源|适用范围|
|---|---|---|
|1|作者当前明确确认|业务意图、作者审美、人物边界、信息边界|
|2|章节创作总控规划|分工、引用、冲突裁决、ASK_USER 触发、token 边界|
|3|章节级叙事表达契约|作者意图、读者信息、POV 过滤、误读方向、硬禁区|
|4|人物意图控制|人物言行、认知、动机、关键片段是否可继续|
|5|章节细纲|剧情顺序、场景功能、事件推进、读者体验曲线|
|6|场景演出契约|文字表达、句式节奏、对白自然化、局部去 AI 化差异|
|7|章节执行包|调度、权重、风险索引、生成顺序|

## ASK_USER 触发条件

|触发条件|必须询问的问题|停止范围|
|---|---|---|
|后续产物需要让读者或马雯雯察觉诈骗、反诈、投资、项目、退款、转账、三万元或金额锁定|本章是否允许从“高级得怪”推进到事实判断？|停止相关细纲、执行包和正文片段|
|场景动作会把云水汇写成低级、污秽、情色加深或明显不正常|本章的“怪”是否仍限定为高级服务过度周到？|停止相关场景设计和正文片段|
|陈艾艾/陈科父女关系被写成单纯不爱、纯冷漠或顺滑温情|父女关系应如何保持冷硬表达与底层相爱并存？|停止相关人物意图和正文片段|
|陈科行为动机无法解释为什么不直接离开或认错|陈科此刻被什么体面、尊重感或父女压力托住？|停止相关人物意图和正文片段|
|马木必须表现得像专业反诈才能推进|马木本章能否升级为专业判断，还是必须维持土法处理？|停止相关片段|
|陈灏登场需要解释异常或测评假高端|陈灏第一印象是否仍必须是会所熟客？|停止章末钩子相关片段|
|场景演出契约需要新增全局文本质量规则|该规则是否应回流导演层规则，而不是写入本章局部契约？|停止规则固化，不停止已确认剧情层产物|

## Token 边界

|规则|说明|
|---|---|
|禁止线性复述|后续产物只引用本规划的分工、冲突和接口，不复制全文|
|禁止总包化|本规划不得写完整剧情、完整场景、完整人物资料|
|保留追溯能力|需要追溯作者原意时读取共创讨论记录|
|导演层规则边界|只登记导演层文本质量规则的读取关系，不复制规则全文，不把本规划扩写为文本质量总包|
|正式测试边界|不得读取已归档的历史测试产物；后续正式测试必须从当前 V1.3 校准目录和正式模板产物开始|

## 越权排除检查

|检查项|通过标准|
|---|---|
|是否写成剧情大纲|没有完整剧情、场景顺序、事件推进或场景起止状态|
|是否写成场景演出契约|没有具体句式、对白自然化细则、去 AI 化操作清单或文字演出规则全文|
|是否写成人物意图结果|没有人物意图轨道、语言/行为意图、认知轨道或 `CONTINUE/ASK_USER/RESUME`|
|是否写成章节执行包|没有场景权重、生成顺序、转场表、风险路由或字数调度|
|是否写成正文草稿|没有正文成品句、完整对白、完整 punchline、段落顺序或结尾句|
|是否复制导演层规则|没有复制导演层通用文本质量规则全文，只登记读取关系和禁止复制边界|
|是否保持接口总线|只规定分工、互补、引用、冲突优先级、ASK_USER 和 token 边界|
