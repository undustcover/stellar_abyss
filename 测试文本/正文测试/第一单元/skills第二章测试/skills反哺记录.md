# 第二章 skills 反哺记录

## 一、文档定位

本文档用于记录第二章正文测试与作者逐轮修订过程中，暴露出的 skill、模板、执行包、细纲和评估流程问题。

它不是正文修改稿，也不是立即修改 skill 的指令。

它的作用是保留每一轮正文修改背后的深度思考，避免最后只根据最终正文结果一次性倒推系统，导致中间判断、问题演化和修正理由丢失。

## 二、记录原则

1. 每完成一轮正文评估、问题总结或作者修订对比，都必须追加一条反哺记录。
2. 记录重点是“这轮正文问题暴露了哪个流程层级的缺口”，不是复述正文好坏。
3. 暂不确认的问题也要记录，但标明为“待观察”。
4. 不把某一章的成品句、成品梗、具体桥段直接回写为通用 skill 规则。
5. 只有当同类问题在多轮或多章中反复出现，才进入正式 skill 修改。

## 三、记录模板

```markdown
## 第X轮：记录标题

|项目|内容|
|---|---|
|对应正文版本||
|对应问题总结/评估文件||
|本轮正文层问题||
|暴露出的流程层问题||
|可能归因对象||
|当前处理方式||
|暂不修改 skill 的原因||
|后续确认条件||
|可能的 skill 修改方向||
```

可选归因对象：

```text
正文生成 skill
章节执行包 skill
章节执行包模板
章节细纲 skill
章节细纲模板
章节大纲/单元大纲
单元上下文包
人物卡字段
事件卡字段
设定卡字段
结构化喜剧规则
角色语言与幽默字段
去 AI 化规则
正文测试评估模板
测试流程
```
