# 模板接口规范：总分总正文生成链路

生成日期：2026-06-23

## 一、规范目标

本文档用于统一总-分-总正文生成链路中各模板的职责、读取关系、引用边界、禁止复述范围和冲突处理方式。

本规范优先解决两个问题：

1. 避免旧线性链路造成的 token 黑洞。
2. 避免完全并联导致各产物彼此脱节。

总原则：

```text
总：章节创作总控规划统一目标、分工、引用和冲突规则
分：各模板只生成自己职责范围内的独立产物
总：正文草稿生成作为总导演统一调用各产物
```

## 二、统一接口字段

后续所有模板都应具备或隐含以下接口说明。正式模板可用表格呈现，也可在模板定位和使用规则中明确。

```markdown
## 模板职责

## 必须读取

## 只可引用

## 禁止复述

## 输出给

## 冲突处理

## ASK_USER 触发

## Token 边界
```

## 三、总链路顺序

```text
章节级叙事表达共创
  -> 章节级叙事表达契约
  -> 章节创作总控规划
  -> 章节细纲
  -> 场景演出契约
  -> 轻量章节执行包
  -> 人物意图控制结果
  -> 正文草稿生成
```

注意：

- 这不是线性复述链。
- 后一产物不得复述前一产物全文。
- 每个产物通过 `章节创作总控规划` 取得分工和冲突规则。
- 正文草稿生成 Skill 最终直接调用所有确认产物，而不是只读取执行包。

## 四、冲突优先级

默认冲突优先级如下。若章节创作总控规划另有明确规定，以当章总控规划为准。

|优先级|来源|适用范围|
|---|---|---|
|1|作者当前明确确认|业务意图、作者审美、人物边界、信息边界|
|2|章节创作总控规划|分工、引用、冲突裁决、ASK_USER 触发、token 边界|
|3|章节级叙事表达契约|作者意图、读者信息、POV 过滤、误读方向、硬禁区|
|4|人物意图控制结果|人物言行、认知、动机、是否可继续生成|
|5|章节细纲|剧情顺序、场景功能、事件推进、场景起止状态|
|6|场景演出契约|文字表达、文风自然化、对白节奏、去 AI 化|
|7|章节执行包|调度、权重、风险索引、生成顺序|

## 六、章节创作总控规划接口

### 模板职责

章节创作总控规划是前置总控层。它规定本章各产物之间的分工、互补、引用、冲突优先级和 token 边界。

### 必须读取

- 作者当前确认的章节级共创结论；
- 章节级叙事表达契约；
- 必要时读取共创讨论记录中的确认项。

### 只可引用

- 上级契约状态；
- 章节级契约中的作者意图和边界；
- 必要的章节编号、标题、相关产物路径。

### 禁止复述

- 完整剧情；
- 场景细节；
- 人物卡、事件卡、设定卡；
- 完整文字演出规则；
- 正文句子或对白。

### 输出给

- 章节细纲；
- 场景演出契约；
- 章节执行包；
- 人物意图控制；
- 正文草稿生成。

### 冲突处理

总控规划负责写明本章局部冲突优先级。若没有写明，按本文档第四节默认优先级处理。

### ASK_USER 触发

- 产物职责边界不清；
- 作者审美方向不清；
- 读者信息边界不清；
- 人物表现是否越界无法判断；
- 下游产物之间出现不可自动裁决的业务冲突。

### Token 边界

只写接口和规则，不写内容总包。

## 七、章节级叙事表达契约接口

### 模板职责

章节级叙事表达契约负责本章作者意图、叙事边界、读者信息边界、POV 过滤、误读方向和高层读感。

保留高层读感字段，例如：

- 表达成分配比；
- 起点读感；
- 中段读感推进；
- 章末落点；
- 本章最容易跑偏的表达方向。

但不得写具体文字演出。

### 必须读取

- 章节创作总控规划；
- 已存在的卷级/单元级契约或其占位状态；
- 作者确认的章节共创结论。

### 只可引用

- 世界事实来源；
- 单元大纲或章节讨论中已确认的必要约束；
- 共创讨论记录中的确认结果。

### 禁止复述

- 完整章节细纲；
- 具体场景执行动作；
- 具体句式、对白、去 AI 化规则；
- 人物卡全文；
- 事件卡全文。

### 输出给

- 章节创作总控规划；
- 章节细纲；
- 场景演出契约；
- 章节执行包；
- 正文草稿生成。

### 冲突处理

- 作者当前确认高于旧契约表述；
- 本章契约只写相对上级契约的差异；
- 可能影响多章的规则进入“待回流上级契约事项”。

### ASK_USER 触发

- 本章到底要表达什么不清；
- 读者能知道/不能知道的信息边界不清；
- POV 人物能感到/不能感到的内容不清；
- 高层读感方向不清；
- 作者明确纠正系统判断。

### Token 边界

契约只记录已确认差异，不记录讨论过程。

## 八、章节细纲接口

### 模板职责

章节细纲负责剧情主线、场景顺序、行动推进、信息变化、场景起止状态。

按作者确认，细纲可以保留轻量读者体验字段，但必须标注来源。它不能自行定稿作者审美。

允许保留的轻量读者体验字段：

- 场景读者体验目标；
- 读者期待/误会/怀疑；
- 情绪变化；
- 场景结束后读者想追的问题。

这些字段必须标注来自：

- 章节级叙事表达契约；
- 章节创作总控规划；
- 作者当前确认；
- 系统建议待确认。

### 必须读取

- 章节创作总控规划；
- 章节级叙事表达契约；
- 必要世界事实资料；
- 必要事件/设定资料。

### 只可引用

- 章节契约中的高层读感和信息边界；
- 总控规划中的分工和引用关系；
- 人物卡/事件卡/设定卡中的必要事实。

### 禁止复述

- 章节级叙事表达契约全文；
- 场景演出契约的具体文字规则；
- 完整人物意图轨道；
- 执行包调度表；
- 正文句子或完整对白。

### 输出给

- 场景演出契约；
- 章节执行包；
- 人物意图控制；
- 正文草稿生成。

### 冲突处理

- 若剧情动作与章节契约信息边界冲突，停下 ASK_USER；
- 若细纲需要新增世界事实，停下 ASK_USER 或回资料系统；
- 若只是技术性场景编号、顺序标记、表格格式，由系统自行处理。

### ASK_USER 触发

- 场景功能会改变作者表达；
- 场景顺序会改变读者误导；
- 读者体验字段无法从已确认契约中推出；
- 人物行为动机明显不成立。

### Token 边界

细纲只写剧情骨架和轻量读者体验，不写文风细则，不扩写为准正文。

## 九、场景演出契约接口

### 模板职责

场景演出契约负责本章或关键场景的文字表达差异、文风自然化差异、句式节奏、对白自然化、旁白边界、心理描写边界、动作/环境/物件承载方式和局部去 AI 化重点。

跨章节复用的文本质量、去 AI 化、对白自然化、幽默误读、句式反 AI 规则，应进入正文草稿生成 Skill 的导演层规则，例如 `generate-chapter-prose-draft/references/director-prose-quality-rules.md`。场景演出契约不得把这些通用规则整包复制进本章契约，只记录本章新增、加强、例外或冲突的规则。

### 必须读取

- 章节创作总控规划；
- 章节级叙事表达契约；
- 章节细纲；
- 作者当前确认的文字审美要求。
- 导演层可复用文本质量规则，用于判断哪些规则不应重复写入本章契约。

### 只可引用

- 章节契约的高层读感；
- 细纲中的场景编号、场景功能、行动推进；
- 总控规划中的分工、冲突和 token 边界。
- 导演层规则的规则名或短标签，不复制全文。

### 禁止复述

- 完整剧情；
- 完整细纲；
- 人物卡全文；
- 人物动机轨道；
- 执行包调度；
- 正文成品句、完整对白、完整 punchline。
- 导演层通用去 AI 化/文本质量规则全文。

### 输出给

- 章节执行包；
- 人物意图控制；
- 正文草稿生成。

### 冲突处理

- 若文字演出会改变剧情，回章节细纲；
- 若文字演出会改变作者意图或读者信息边界，回章节契约；
- 若文字演出需要人物改变动机，交人物意图控制或 ASK_USER。

### ASK_USER 触发

- 作者审美不清；
- 是否允许示例句不清；
- 旁白/心理描写尺度不清；
- 对白自然化方向不清；
- AI 感风险由作者审美决定。

### Token 边界

只写本章规则和差异，不复制讨论过程，不复述导演层通用规则，不逐场铺满普通场景。

## 十、章节执行包接口

### 模板职责

章节执行包负责产物索引、场景权重、生成顺序、转场、风险路由和人物意图覆盖状态。

执行包不是人物意图控制的下游流水线，也不是人物意图结果的汇总文件。它和人物意图结果同属分工产物，共同服从章节创作总控规划和章节级叙事表达契约。执行包可以引用已存在人物意图结果的路径、覆盖标签、风险标签或短摘要；如果人物意图结果尚未生成或覆盖不足，执行包只能记录缺口和回流建议，不能自行补写人物意图轨道。

### 必须读取

- 章节创作总控规划；
- 章节级叙事表达契约；
- 章节细纲；
- 场景演出契约；
- 章节级人物意图结果，若已存在；
- 必要人物/事件/设定资料。

### 只可引用

- 上游产物路径/ID；
- 上游产物的短标签或短摘要；
- 场景编号；
- 风险点；
- 调度规则。

### 禁止复述

- 完整章节契约；
- 完整细纲；
- 完整场景演出契约；
- 人物卡全文；
- 正文成品句。

### 输出给

- 正文草稿生成；
- 可选单场景执行 brief。
- 必要时回流人物意图控制。

### 冲突处理

- 若调度和总控规划冲突，按总控规划；
- 若执行包发现业务冲突，进入待确认问题；
- 不得用执行包静默改写上游产物。

### ASK_USER 触发

- 场景轻重影响作者审美；
- 主钩子/轻收束取舍不清；
- 高风险人物意图是否已有独立人物意图结果覆盖不清；
- 是否需要单场景执行 brief 不清且会影响正文质量。

### Token 边界

执行包是调度索引，不是总包。

## 十一、人物意图结果接口

### 模板职责

人物意图结果负责章节级人物语言意图、行为意图、认知边界、动机基线、CONTINUE/ASK_USER/RESUME 和运行态更新建议。

人物意图结果是总-分-总结构中的分工产物之一。它不由执行包触发，也不以执行包为唯一输入。它读取总控规划、章节契约、必要人物资料，并可局部引用细纲和场景演出契约，产出本章主要人物的行为基线、语言意图基线、认知边界和章节差异。执行包和正文总导演可以引用它的覆盖标签、风险标签和短摘要。

按作者确认，人物意图控制需要引用场景演出契约的文字规则，因为人物最终表现效果必须符合场景演出规划。

但引用范围仅限：

- 将文字规则转为语言策略；
- 将对白自然化要求转为说话方式约束；
- 将旁白/心理边界转为“人物不能显得像解释设定”的约束；
- 将去 AI 化要求转为人物行为和语言的自然化检查。

不得评价整体文风，也不得重写场景演出契约。

### 必须读取

- 章节创作总控规划；
- 章节级叙事表达契约；
- 章节细纲相关片段；
- 场景演出契约相关规则；
- 章节执行包标记的场景/风险索引，若已存在；
- 人物卡、事件卡、设定卡相关事实。

### 只可引用

- 本章主要人物需要的人物知识、动机、压力；
- 场景演出契约中与当前人物表达有关的文字规则；
- 执行包标记的未覆盖风险或需复核风险，若已存在。

### 禁止复述

- 完整场景演出契约；
- 完整细纲；
- 完整执行包；
- 正文段落；
- 作者未确认的人物意图。

### 输出给

- 正文草稿生成；
- 章节运行态；
- 必要时回到作者确认。

### 冲突处理

- 若人物动机与剧情推进冲突，ASK_USER；
- 若场景演出要求导致人物不符合人设，ASK_USER；
- 若人物知道了不该知道的信息，按章节契约和人物卡回退；
- 若只是句式自然化问题，作为语言策略约束输出，不改文风规则。

### ASK_USER 触发

- 人物为什么这样做不成立；
- 人物为什么不采取更简单行动不成立；
- 人物语言意图有多种业务含义；
- 场景演出要求和人物人设冲突；
- 是否继续生成需要作者裁决。

### Token 边界

生成章节级人物意图结果；只在回流补充/中断场景中生成当前关键片段需要的增量意图轨道。不得把人物意图控制改成逐场默认流水线。

## 十二、正文草稿模板接口

### 模板职责

正文草稿模板负责承载最终小说正文和生成备注。正文生成 Skill 作为总导演统一调用各产物。

正文草稿生成 Skill 同时承担可复用文本质量的执行层职责：读取导演层通用文本质量规则，并把场景演出契约中的本章差异叠加到具体正文生成中。

### 必须读取

- 章节创作总控规划；
- 章节级叙事表达契约；
- 章节细纲；
- 场景演出契约；
- 章节执行包；
- 人物意图控制结果；
- 章节运行态。
- 导演层可复用文本质量规则。

### 只可引用

- 已确认上游产物；
- 当前生成片段所需资料；
- 总控规划中的冲突优先级和 ASK_USER 规则。

### 禁止复述

- 任何上游模板表格；
- 过程标签；
- 意图轨道；
- 分析说明；
- 章节执行包结构短语。

### 输出给

- 正文文件；
- 正文评估；
- 局部返修；
- 必要时回流到共创或人物意图控制。

### 冲突处理

- 根据总控规划处理冲突；
- 没有优先级且涉及业务判断时，ASK_USER；
- 不静默改世界事实；
- 不静默改变作者审美。

### ASK_USER 触发

- 上游产物冲突无法裁决；
- 人物意图控制返回 ASK_USER；
- 需要改变剧情、信息边界、人物边界或作者审美；
- 正文生成需要使用非系统成品材料。

### Token 边界

正文只写正文。生成备注只写风险和追踪，不复制上游产物。
