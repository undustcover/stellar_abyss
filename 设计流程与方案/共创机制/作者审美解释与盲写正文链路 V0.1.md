# 作者审美解释与盲写正文链路 V0.1

## 一、结论

该机制方向正确，并且比“生成后去 AI 化、统一文风、逐句润色”更接近真正的文学共创。

它解决的不是句面模仿，而是三个更底层的问题：

1. 从作者修改中识别作者拒绝的模型倾向；
2. 把作者审美转译为当前场景的文学判断，而不是成品写法；
3. 让正文模型在不知道范文具体表达的情况下，独立解决同一个文学难题。

但它不能承诺“完全契合作者审美”。文学判断本身包含隐性、变化和语境依赖，系统能建设的是一条可验证、可纠错、逐步收敛的契合路径，而不是一次性复制作者。

该机制也不能单独解决草稿质量问题。完整链路必须同时包含：

```text
生成前作者共创
→ 草稿生成
→ 作者修改案例解释与审美库积累
→ 当前场景审美诊断
→ 多方向提出与审美裁决
→ 隔离范文后的独立正文生成
→ 污染与模板化检查
```

## 二、与已废弃后置修订链的区别

本方案不恢复“去 AI 化 → 文风校准 → 对白幽默化”的整章润色链。

两者的区别是：

|旧后置修订链|本方案|
|---|---|
|主要修改句面|先判断文学问题，再重新生成|
|追求把草稿修漂亮|追求让正文重新作出正确审美决策|
|容易逐句锚定原稿|高失败场景去表达化后重构|
|风格规则直接进入正文模型|作者案例只经过抽象解释后进入诊断层|
|模型容易学习成品方案|正文模型看不到范文、原句和案例桥段|

因此，本方案应被定义为“审美决策与独立重构链”，不是“后置润色链”。

## 三、完整链路

### 阶段 A：生成前作者协同

草稿生成不能只读取大纲、细纲和执行包。章节启动前必须确认：

- 本章真正要解决的文学问题；
- 作者已经明确的画面或关系压力；
- 必须保留的未确定性；
- 作者不接受的叙事方向；
- 哪些场景必须共创，哪些空间交给模型；
- 哪些事实和边界不能改变。

普通场景可以自动生成。核心冲突、重大误会、人物转折、高潮和结尾钩子应进入作者协同。

输出为“章节创作约定”或现有“章节创作总控规划”的正文层补充，不输出具体句子。

### 阶段 B：作者修改案例解释器

该阶段不是每章运行，而是在作者完成高价值修改后运行，用于建设案例库。

#### 输入

- 模型草稿；
- 作者修改稿；
- 章节上下文与原任务；
- 作者口述理由或批注（若有）；
- 已存在的作者审美决策库。

只有“草稿 + 修改稿”时，解释器只能提出原因假设，不能把推断写成已确认审美。相同改动可能同时来自节奏、人物、信息边界、幽默尺度或事实修正，必须允许多种解释。

#### 输出

每条案例只输出：

|字段|说明|
|---|---|
|案例编号|可追溯到原始对比材料，但下游正文模型不可读取原文|
|作者拒绝的模型倾向|模型默认滑向了什么|
|作者改变的叙事层面|事件、场面、人物、信息、节奏、语气、意象、句面中的哪一层|
|受保护的文学价值|作者实际在保护什么张力、含混、尊严、压力或阅读效果|
|可迁移的审美判断|不含成品桥段和表达的条件式判断|
|适用条件|什么场景下成立|
|不适用条件|什么场景下不能套用|
|置信度|推断、作者部分确认、作者确认|
|竞争解释|仍可能成立的其他修改原因|

#### 强制删除

- 作者原句；
- 修改后的具体桥段；
- 成品笑点、比喻、物件组合和落点；
- 段落顺序；
- 可直接复刻的动作设计；
- “遇到某类场景就使用某个固定方案”的处方。

原始对比材料进入隔离案例区，只允许案例解释器和评估器读取；正文模型永远不可读取。

### 阶段 C：作者审美决策库

审美库不应是一份“作者文风规则大全”，而应由带适用边界的判断单元组成。

建议分三级：

```text
L1 单案例假设：只在一次修改中出现，等待作者确认或后续复现
L2 重复倾向：跨多轮或多章出现，但仍保留场景标签
L3 作者确认原则：作者明确认可，且已验证不会把不同章节写成同一种结构
```

规则升级条件：

1. 作者明确确认；或
2. 同类倾向在不同场景重复出现；
3. 能表述为审美判断，而不是具体方案；
4. 写明适用和不适用条件；
5. 未导致后续章节表现同质化。

规则也必须允许降级、拆分和废弃。作者审美会发展，旧判断不能永久统治新文本。

### 阶段 D：当前场景审美诊断

#### 输入

- 当前草稿；
- 作者审美决策库；
- 章节上下文；
- 章节创作约定；
- 必须保持的事实与边界。

#### 输出

```text
1. 当前真正的文学难题
2. 草稿当前采用了什么默认解法，以及它为什么失效
3. 模型最可能写俗、写满、写假、写聪明的位置
4. 必须保留的未确定性、压力和张力
5. 本场允许改变的叙事层面
6. 本场禁止改变的事实与边界
7. 若干处理方向的评价坐标，但不提供具体句子或桥段
```

诊断必须区分：

- “草稿写得不好”；
- “草稿选错了解决问题的叙事层面”；
- “上游文学问题本身没有被定义清楚”。

若属于第三种，不应直接进入正文重写，应返回作者共创或章节总控层。

### 阶段 E：正文方向生成

正文模型或独立方案模型提出 2—3 个真正不同的方向。

每个方向只描述：

- 它把张力放在哪个叙事层面；
- 它选择靠近谁的感知；
- 它如何控制信息与节奏；
- 它保留什么不说；
- 它可能获得什么阅读效果；
- 它的主要风险是什么。

方向之间必须是结构性差异，不能只是“更幽默、较克制、更感人”之类形容词差异，也不能提供具体对白、比喻、动作链或结尾句。

### 阶段 F：审美编辑裁决

审美编辑读取审美诊断、方向提案和审美库，但不读取原始作者范文。

职责：

1. 淘汰套路化方向；
2. 淘汰直接复制案例结构的方向；
3. 淘汰把文学问题解释完的方向；
4. 淘汰违反人物、信息或事实边界的方向；
5. 选择一个方向，或组合两个互补方向；
6. 若全部不合格，返回重提，不强行三选一。

裁决输出只包含：

- 选定方向；
- 选择理由；
- 必须保留的张力；
- 必须避免的退化倾向；
- 创作自由区。

不得提供具体句子和正文方案。

### 阶段 G：隔离参照后的正文生成

“盲写”更准确的名称是“参照隔离生成”。正文模型不能看到：

- 原始作者范文；
- 作者原句；
- 原始修改对；
- 案例中的具体表达和桥段；
- 审美编辑淘汰方向的成品化内容。

正文模型只看到：

- 当前文学问题；
- 选定方向；
- 必须保持的事实和边界；
- 人物当前状态与章节上下文；
- 必须保留的未确定性和张力；
- 创作自由区。

#### 两种生成模式

|模式|正文模型是否看到当前草稿|适用情况|
|---|---|---|
|局部修复|只看到失败场景及必要相邻段|原稿场面选择正确，仅局部表达或节奏失败|
|独立重构|不看原草稿文字，只看去表达化后的事实骨架|原稿的叙事层面、场面组织或整体解法错误|

如果正文模型始终看到完整草稿，它会受到原稿句序、解释路径和场面选择的锚定，不能称为真正盲写。

### 阶段 H：生成后检查

生成后不进行句面润色，只做门禁检查：

1. 是否出现明显模板化表达；
2. 是否搬用案例方案或作者范文痕迹；
3. 是否回到通用 AI 的完整解释模式；
4. 是否违反事实、人物、POV 和信息边界；
5. 是否实际执行选定方向；
6. 是否破坏要求保留的未确定性和张力。

若失败，输出失败类型并重新生成失败场景。检查器不得顺手改句子，否则会重新变成后置润色模型。

## 四、创造力保护机制

要避免系统只会“越来越像作者过去写过的东西”，必须加入以下保护：

1. 审美库保存判断，不保存解法；
2. 每条规则必须有适用边界和失效条件；
3. 方向生成必须先于审美裁决，并保留至少一个非惯常方向；
4. 审美编辑可以排除退化，但不能把方向细化成桥段；
5. 正文模型拥有明确的创作自由区；
6. 评估标准是能否解决当前文学问题，不是像不像历史范文；
7. 允许模型提出超出作者既有案例的新解，只要不违反作者确认的拒绝项和章节边界；
8. 定期检查审美库是否造成章节结构、幽默机制、结尾方式或人物反应同质化。

这里最重要的区分是：

```text
作者审美负责定义“什么不能接受、什么必须保护”；
模型创造力负责寻找“还有什么此前没出现过的解决办法”。
```

## 五、从实际作者修改开始迭代是否可行

可行，而且这是当前最可靠的开发路径。

原因是作者很难脱离具体文本完整描述自己的审美，但能对一个已经发生的写作选择作出准确判断。模型也更适合先提出修改原因假设，再由作者确认、纠正或补充，而不是让作者从零填写抽象规则。

推荐迭代循环：

```text
模型生成草稿
→ 作者按真实创作直觉修改
→ 解释器提出“为什么这样改”的少量高价值假设
→ 作者确认、否定或补充
→ 形成单案例判断
→ 新章节验证
→ 重复出现后升级为稳定审美原则
```

不是所有改动都值得解释。优先处理：

- 改变整场读感的重构；
- 作者明确表示“俗、满、假、太聪明、太会解释”的位置；
- 删除看似漂亮但作者拒绝的内容；
- 保留不完整、不说明或不落地的决定；
- 人物从功能位恢复成活人的修改；
- 同类问题跨章节复现的位置。

错别字、普通语病、事实修正和纯粹个人临时偏好，不应自动升级为审美原则。

## 六、第一、二章已有材料能形成的首批待确认假设

以下只作为案例解释器的待确认输出，不直接写入稳定审美库：

|待确认假设|来源表现|当前置信度|
|---|---|---|
|作者拒绝旁白替人物完成判断|概括式幽默、完整心理解释被反复改为场内反应|较高|
|作者更看重笑点的关系与场面机制，而非单句机灵|第一章重建连续错位，第二章压低金句密度|较高|
|作者保护人物在压力中的体面，不愿过早审判或解释人物|陈科先滑稽、嘴硬，悲剧解释被延后|较高|
|作者倾向让误会双语境成立，并避免直白低俗|第二章会所误会被要求含混、体面|较高但可能只适用于误会喜剧|
|作者允许删减漂亮表达，但不允许减法伤害承压点|第二章第二版暴露出的失败|较高|
|作者偏好主钩子保留后续压力，轻喜只能作为余味|第二章结尾取舍|中等，需跨章验证|

这些假设必须经过作者逐条确认，并补充“不适用场景”，才能升级。

## 七、最小可行实施顺序

暂不一次性修改全部 skill。建议按以下顺序建立：

### M1：案例解释模板

先用第一章和第二章的已有草稿、作者稿、对比报告，生成 6—10 条高价值案例判断，让作者确认解释是否准确。

### M2：作者审美决策库

只收录作者确认后的判断；建立 L1/L2/L3、适用条件、不适用条件和废弃机制。

### M3：当前场景审美诊断

选择第三章一个高失败场景测试，验证诊断是否能指出“真正的文学难题”，而不是输出泛化写作建议。

### M4：方向生成与审美裁决

对同一场景提出三个结构性不同方向，检查审美编辑能否只做判断、不泄露具体写法。

### M5：参照隔离生成 A/B 测试

同一场景分别运行：

- A：正文模型看到当前草稿；
- B：正文模型只看到去表达化事实骨架。

比较两者的锚定程度、创造性、事实稳定性和作者接受度，再决定不同失败类型使用哪种模式。

### M6：接回生成前作者共创

验证后再把审美链与现有章节创作总控规划、场景演出契约和正文生成 skill 接口对齐。

## 八、方案成立的验收标准

不能以“生成稿更像作者范文”为验收标准。应检查：

1. 作者能否认可系统对修改原因的解释；
2. 新章节是否减少同类审美退化；
3. 正文是否没有复制历史案例的具体方案；
4. 不同章节是否仍然保持不同结构和气质；
5. 正文模型是否能提出作者案例库中没有的新解；
6. 新解是否仍然解决当前文学难题；
7. 作者否定一次后，系统能否准确更新判断而不扩大成僵硬规则；
8. 系统能否明确区分上游问题、方向问题和正文执行问题。

## 九、当前决策

1. 接受“案例解释 → 审美诊断 → 多方向 → 审美裁决 → 参照隔离生成”的总体方向；
2. 不承诺一次性完全复制作者审美，采用案例驱动的渐进收敛；
3. 不把作者范文、原句、桥段和笑点放入正文模型上下文；
4. 将完整草稿可见性拆成“局部修复”和“独立重构”两种模式；
5. 先建设案例解释模板与首批案例，不立即大改现有正文 skill；
6. 所有稳定审美原则必须经过作者确认或跨场景复验。
