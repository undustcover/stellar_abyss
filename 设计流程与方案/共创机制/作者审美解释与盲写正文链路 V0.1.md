# 作者审美解释与 Draft→Public 表达重构链路 V0.3

## 一、当前结论

Draft→Public 不是同义词替换、逐句润色，也不默认推翻 Draft 后整章盲写。

本链路的正式定位是：

> 冻结 Draft 中已经合格的剧情、场景、人物、POV 和信息结构；对不符合作者审美的句式、对白、段落、旁白、表达路径与写作基调进行分层审计，并按问题严重度执行词句修正、句式重写、段落重组或片段表达重构。

核心关系：

```text
Draft 提供已经成立的章节框架和待审计表达
Public 保留合格创作决策，重构不合格表达决策
```

Public 不是第二份 Draft。只有审计发现上游结构本身不成立时，才允许回到场景或章节设计层；该变化必须由作者介入确认。

## 二、Draft 与 Public 的职责边界

### 2.1 默认冻结的 Draft 合格层

作者或上游链路已经确认合格时，Public 默认不得改变：

- 章节核心事件；
- 场景顺序与场景功能；
- 人物定位、人物关系和行为逻辑；
- POV；
- 读者信息边界；
- 必须保留的误会、张力与未确定性；
- 章节主要读者效果；
- 章末功能和后续接口。

“默认冻结”不等于永远不能改变。若表达审计发现只有改变上述内容才能解决问题，系统必须停下，向作者说明冲突，不能静默越权。

### 2.2 Public 的默认工作层

Public 主要处理：

- 用词是否准确、自然并符合人物；
- 句式是否公式化、总结化、假精炼或带统一 AI 节奏；
- 对白是否成为标准问答、剧情宣读或角色互相配合；
- 段落是否只解释结论，缺少人物动作和关系变化；
- 信息是否全部靠询问、旁白或说明进入；
- 动作、物件、对白、心理和旁白是否彼此脱节；
- 喜剧是否只是孤立俏皮话，未进入人物和关系；
- 旁白是否服从场景底色并提供新增作用；
- 轻喜、压力、不安等基调是否自然连续；
- 同一功能片段是否需要重新安排句序、信息入口和段落重心。

## 三、必须区分的两种“结构”

### 3.1 剧情结构

剧情结构回答：

- 发生什么；
- 谁参与；
- 场景如何排列；
- 人物作出什么关键行为；
- 信息和结果如何变化。

Draft 已通过上游人物、细纲和章节控制时，Public 不默认重做剧情结构。

### 3.2 表达结构

表达结构回答：

- 一个信息先由动作、对白还是旁白出现；
- 人物反应与事实释放如何咬合；
- 一段对话如何打断、回避、抢话或转移；
- 句子和段落如何控制阅读呼吸；
- 读者通过什么材料自行判断；
- 笑点、压力与不安如何在同一片段中运动。

Draft→Public 的主要重构对象是表达结构。

## 四、Public 分层审计模型

每个功能片段必须先审计，再决定修改力度。不得默认逐句替换。

### 4.1 问题层级

|问题层|审计对象|常见失败|
|---|---|---|
|词汇层|名词、动词、形容、语气词|抽象、可替换、用词不贴人物、形容过度|
|句式层|句子组织和节奏|公式比喻、总结句、解释链、统一短句模板、假金句|
|对白层|人物语言运动|标准问答、完整说明、角色配合递信息、人人高胜率|
|段落层|动作、对白、心理和旁白关系|只解释结论、包袱后继续说明、动作只是装饰|
|表达路径层|信息进入正文的方式|风险全靠问出、人物靠旁白定义、剧情靠说明推进|
|基调层|场景整体文字运动|轻喜与压力切换生硬、旁白脱离现场、笑点冲淡承压|
|审美决策层|文本保护和拒绝的文学价值|写满、写透、写聪明、人物工具化、留白失效|

### 4.2 修改等级

|等级|名称|处理方式|是否默认需要作者介入|
|---|---|---|---|
|L0|直接保留|原表达已经合格，不为显示修改量而重写|否|
|L1|词句修正|保留原表达机制，只修正用词、语病和局部节奏|否|
|L2|句式重写|保留事实、句子功能和表达方向，重新写句子|否|
|L3|段落重组|保留片段事件与人物目的，重排动作、对白、旁白、句序和信息入口|否|
|L4|片段表达重构|保留场景功能、事件结果、人物与信息边界，暂时脱离原段落顺序，从空白处重写一个功能片段|否，但触碰冻结层时必须中断|
|L5|场景/剧情结构重构|改变场景顺序、核心行为、人物定位、POV、信息边界或章末功能|是，必须停下由作者确认|

Public 的默认有效工作区是 `L0—L4`。本轮作者确认的主要目标是扩大 `L2—L4` 的使用，而不是把绝大多数问题压缩成 `L1`。

## 五、片段级直接审计

Public 不逐句映射 Draft，而以“功能片段”为最小重构单元。功能片段可以是一组连续段落、一次对话交锋、一次物件回调或一个小型情绪转折。

每个需要处理的片段至少记录：

|字段|说明|
|---|---|
|片段 ID|便于追踪|
|片段功能|这段必须完成什么|
|必须保留|事实、人物目的、关系状态、信息边界和读者效果|
|Draft 当前表达机制|原稿通过什么句式和路径完成|
|审美问题|具体违反什么作者审美或文学价值|
|问题层级|词汇／句式／对白／段落／表达路径／基调／审美决策|
|修改等级|L0—L5|
|重构任务|需要改变哪些表达关系|
|禁止改变|本片段的冻结层|
|作者介入状态|not_needed / checkpoint_declared / ask_user / resolved|
|验收问题|重写后是否仍只是同义替换或逐句对应|

### 5.1 审美原则必须转成片段任务

正式库不能只作为检查清单。每条被调用的审美原则必须转译为当前片段的修改动作。

示例：

```text
审美原则：
剧情信息必须经过人物呈现

Draft 问题：
风险信息全部通过连续问答释放

片段重构任务：
保留风险事实，但重新分配信息入口：
- 一部分由人物主动辩护时泄露；
- 一部分由物件争夺或动作触发；
- 一部分由后续人物反应验证；
- 提问者只抓住关键矛盾，不宣读完整风险清单。
```

审美库负责判断“什么不合格、什么必须保护”；片段审计负责把它转换成“当前具体重构什么”。

## 六、生成前作者介入地图

每次 Draft→Public 开始前，系统必须先建立“作者介入地图”，明确本轮哪些节点可能需要作者协助。未先建立介入地图，不得进入正文生成。

### 6.1 必须预先登记的节点

|节点|系统先做什么|何时必须停下 ASK_USER|
|---|---|---|
|A. 合格层确认|登记本轮默认冻结的剧情、场景、人物、POV、信息和章末功能|无法判断某项是上游硬约束还是 Draft 临时写法|
|B. 审美规则冲突|列出适用规则及边界|两条作者确认规则在当前片段导向不同选择，且无法按适用条件裁决|
|C. 修改等级升级|为片段选择 L0—L4|发现必须进入 L5，或 L4 会实质改变冻结层|
|D. 表达方向分歧|内部比较可行表达方向|多个方向保护不同作者价值，现有正式库无法决定优先级|
|E. 新增事实或人物能力|检查重构是否只改变表达|需要新增事实、动机、能力、关系历史或世界规则才能成立|
|F. POV 与信息变化|检查读者和视角人物能知道什么|需要让人物知道原本不知道的信息，或需要越过 POV 发放答案|
|G. 基调边界|根据已确认审美自动裁决|喜剧、压力、悬念或温情的优先级不清，并会改变整场读感|
|H. 章末功能|保持已确认章末承诺|表达重构会改变章末第一功能、钩子或后续接口|

### 6.2 中断协议

触发作者介入节点时必须：

1. 立即停止当前片段及后续正文生成；
2. 明确指出触发节点和片段 ID；
3. 列出已经确认且不会改变的内容；
4. 说明无法自动裁决的具体分歧；
5. 给出不超过三个结构清楚的选项及各自影响；
6. 等待作者确认；
7. 将确认结果记录为 `resolved` 后再继续。

禁止：

- 保持沉默并自行选择；
- 先把整章写完再把冲突放进生成备注；
- 用“先生成一个版本看看”绕过作者裁决；
- 因为担心打断流程而弱化 ASK_USER。

### 6.3 不应打断作者的情况

以下属于系统应自行完成的专业判断：

- 普通同义词选择；
- 句子长短和段落换行；
- 明确违反正式审美库的模板句删除；
- L1—L3 范围内不触碰冻结层的表达调整；
- 已有规则足以裁决的旁白密度、解释压缩和对白自然化；
- 单纯语病、错别字和重复信息处理。

作者介入用于业务与审美裁决，不用于把所有写作选择退回作者。

## 七、Draft→Public 正式生成流程

```text
1. 冻结 Input Draft
2. 读取正式审美库、方法库和必要最小上下文
3. 建立合格层冻结表
4. 建立作者介入地图
5. 将 Draft 划分为功能片段
6. 对每个片段进行表达层直接审计
7. 为每个片段选择 L0—L4；若触发 L5 或其他介入节点，立即 ASK_USER
8. 将审美原则转译为片段重构任务
9. 按片段执行保留、句式重写、段落重组或表达重构
10. 完成全章基调、衔接和节奏统一
11. 运行事实门禁、作者审美门禁和修改深度门禁
12. 若门禁失败，只重做失败片段，不静默改变冻结层
13. 冻结 System Public V1
14. 作者基于 V1 生成 Author Public V2
15. 比较 V1／V2并由作者校准原因
16. 只将作者确认的抽象判断回灌正式库
```

## 八、Public 生成方式

### 8.1 Draft 可见，但不能逐句依附

Public 生成器可以读取 Draft，因为它需要继承已确认的内容和上下文。但是必须按功能片段工作：

```text
读取完整功能片段
→ 提取必须完成的事实、人物目的和读者效果
→ 明确原表达为什么不符合作者审美
→ 暂时脱离原句顺序
→ 重新组织片段
→ 与前后片段接合
```

正确原则：

> 看 Draft 是为了继承内容，不是为了继承句序。

### 8.2 L4 的局部隔离

进入 L4 时，系统应先生成片段级重构卡，再重写该片段。重写时可以保留前后衔接和必要事实，但不能逐句对照原片段。

只有进入 L5 且作者明确批准时，才考虑场景级或更大范围的去表达化重构。

## 九、三类门禁

### 9.1 事实与边界门禁

- 是否改变核心事件、人物定位或行为逻辑；
- 是否违反 POV 和信息边界；
- 是否新增未经授权的事实；
- 是否破坏必须保留的张力和章末功能。

### 9.2 作者审美门禁

- 是否仍有解释腔、总结腔和公式比喻；
- 人物是否仍在配合宣读剧情；
- 旁白是否替人物和读者完成判断；
- 笑点是否脱离人物、关系和场景；
- 是否写满、写透、写聪明；
- 被调用的正式条目是否实际转化为正文效果。

### 9.3 修改深度门禁

- 是否只替换同义词；
- 是否逐句对应 Draft；
- L2 片段是否真正重写句式；
- L3 片段是否改变动作、对白、旁白和信息入口的组织；
- L4 片段是否脱离原段落顺序完成重新表达；
- 标准问答是否只是删掉几个问题；
- 原解释性旁白是否只是拆成短句；
- 已经合格的 L0 内容是否遭到无意义改写；
- Public 是否解决审计记录中的具体审美问题。

若大面积 L2—L4 片段仍与 Draft 保持同一表达骨架，不得通过门禁。

## 十、独立 Public 测试闭环

比较关系保持：

```text
Input Draft
→ System Public V1
→ Author Public V2
→ V1／V2 审美解释
→ 作者校准
→ 正式库更新
```

四份材料职责：

|材料|职责|是否作为审美答案|
|---|---|---|
|Input Draft|提供已确认章节框架和待审计表达|否|
|System Public V1|检验系统能否正确保留合格层并重构表达层|否，必须冻结|
|Author Public V2|记录作者对系统表达选择的真实修正|是，本轮目标证据|
|V1／V2 校准案例|解释修改原因并决定是否入库|经作者确认后才成立|

历史作者成品、目标稿、原句和具体桥段继续与 Public 生成隔离。

## 十一、首轮第三章测试结论

`TEST-P00-C03-01` 的 `System Public V1` 已冻结，但判定为流程失败样本。

失败不是因为 Draft 的剧情、场景、人物和信息结构已被证明不合格，而是因为：

1. 未建立合格层冻结表；
2. 未建立作者介入地图；
3. 未进行片段级表达审计；
4. 未为片段选择 L0—L4；
5. 审美原则没有转译为片段重构任务；
6. 创作自由区被错误限制为调整词汇、句长和局部压缩；
7. 生成过程沿用 Draft 原句序和表达路径；
8. 门禁没有检查修改深度。

该 V1 不作为合格 Public，也不要求作者继续逐句修改。后续应保留其失败证据，并在新流程下另存新的系统测试版本。

## 十二、当前决策

1. Draft 的剧情、场景、人物、POV 和信息结构通过上游确认后，Public 默认冻结；
2. Public 的核心工作是表达层分级审计与重构；
3. 默认修改范围为 L0—L4，不把整章盲写设为常规步骤；
4. Public 可以读取 Draft，但以功能片段为单位重构，不逐句映射；
5. 每轮生成前必须建立作者介入地图；
6. 遇到介入节点必须停止并请求作者协助；
7. 审美库必须转译为片段级重构任务，不能只作为否决清单；
8. 新增修改深度门禁，防止 Public 退化为同义词替换；
9. `TEST-P00-C03-01 / System Public V1` 保留为失败样本；
10. 下一步先按 V0.3 流程重新审计第三章 Draft，再决定新的系统版本编号。
