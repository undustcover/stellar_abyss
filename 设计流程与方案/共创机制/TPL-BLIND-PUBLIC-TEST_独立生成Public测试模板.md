# Draft→Public 表达重构测试：TEST-XXXX

## 一、测试目标

本测试验证系统能否保留 Draft 已经合格的章节框架，同时对不符合作者审美的句式、对白、段落、表达路径和写作基调执行分层重构。

本测试不是逐句润色，也不默认整章盲写。

标准闭环：

```text
作者提供 Draft
→ 冻结合格层
→ 建立作者介入地图
→ 片段级表达审计
→ L0—L4 分级处理
→ System Public V1
→ 修改深度门禁
→ 冻结 V1
→ Author Public V2
→ V1／V2 校准
→ 更新正式库
```

## 三、输入隔离与读取边界

### 允许读取

- 本轮 Draft；
- 当前有效的作者审美决策库；
- 当前有效的笑话写作方法论库；
- 确认 Draft 合格层所必需的最小上游产物；
- 必须保持的事实、人物、POV、信息边界与章末功能。

### 禁止读取

- 本章作者目标稿、预改稿或隐藏参照稿；
- 本章未来的 Author Public V2；
- 历史案例中的作者原句、成品笑点、具体桥段和段落顺序；
- 能够泄露本章作者答案的旧修改意见。

### 检查

- [ ] 作者目标答案已隔离
- [ ] 历史经验只通过正式抽象库进入
- [ ] Draft 可见性用于继承内容，不用于逐句映射
- [ ] 必要的上游扩展读取已记录路径与原因

## 九、三类门禁

### 9.1 事实与边界门禁

- [ ] 未改变冻结的核心事件、人物、POV 和信息边界
- [ ] 未新增未经确认的事实、动机、能力或世界规则
- [ ] 未破坏必须保留的张力
- [ ] 未改变章末功能和后续接口

### 9.2 作者审美门禁

- [ ] 解释腔、总结腔和公式比喻已按审计处理
- [ ] 对白不再只是标准问答或剧情宣读
- [ ] 人物、关系和动作承担了应承担的信息
- [ ] 旁白有新增作用并服从场景底色
- [ ] 笑点没有脱离人物和关系
- [ ] 正式审美条目已转化为可见正文效果

### 9.3 修改深度门禁

- [ ] Public 不是同义词替换版
- [ ] Public 没有逐句映射 Draft
- [ ] L2 片段已真正重写句式
- [ ] L3 片段已重排动作、对白、旁白或信息入口
- [ ] L4 片段已脱离原段落顺序重新表达
- [ ] 标准问答不是只删掉几个问题
- [ ] 解释性旁白不是只拆成短句
- [ ] L0 合格内容未被无意义改写
- [ ] 每个审计问题均有明确处理结果

任一关键门禁失败时，不得冻结为合格 V1。应标记失败片段并重新处理；若修复会触碰作者介入节点，立即 ASK_USER。
