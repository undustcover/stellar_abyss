## 一、改造背景

当前 V1.3 链路存在一个关键风险：正文生成流程容易被理解为线性转述链。

旧线性理解类似：

```text
章节级叙事表达契约
-> 章节细纲
-> 章节正文总体规划
-> 场景演出契约
-> 章节执行包
-> 人物意图控制
-> 正文草稿
```

这种链路会导致每一步都复述上一步内容，最终形成 token 黑洞：

1. 同一章的作者意图、剧情信息、人物状态、场景重点被多次转写。
2. 下游产物不再直接读取原始决策，而是读取二手、三手摘要。
3. 执行包容易膨胀为总包，正文草稿生成只依赖执行包，丢失其他产物的独立价值。
4. 每个 skill 的职责边界变模糊，质量问题难以归因。

本次改造目标是将链路从“线性转述”改为“总-分-总导演式调用”。

只做并联独立也有风险：各产物可能不再复述，但彼此缺少内在联系，最终正文生成时变成硬拼几份互不咬合的文件。因此本轮方案不是“完全松散并联”，而是：

```text
总：章节创作总控规划，定义本章总目标、产物分工、互补关系、引用关系和冲突优先级
分：各 skill 按职责生成独立产物
总：正文草稿生成 skill 作为总导演统一调用各产物生成正文
```

## 二、核心决议

正文创作采用类似电影导演的方法，但必须采用“总-分-总”结构。

前期需要一个轻量的 `章节创作总控规划`，先统一各产物和 skill 之间的分工、接口、互补关系和引用关系。

中期由多个 skill 分别生成产物。各产物相互独立，但不彼此孤立。它们都应服从 `章节创作总控规划`，只在自己的职责范围内输出内容。

后期正文草稿生成 skill 是总导演。它不只读取章节执行包，也不要求章节执行包复述所有上游产物，而是直接调用总控规划和各独立产物。

多个独立产物分别承担不同创作部门的职责，正文草稿生成 skill 在生成正文时直接引用这些产物：

```text
章节创作总控规划
        ↓
章节级叙事表达契约 ┐
章节细纲             │
场景演出契约         ├─> 正文草稿生成 skill（总导演）
人物意图控制         │
章节执行包           ┘
```

章节执行包不再是“所有内容的总复述文件”，而是调度单、索引单和风险清单。

### 2.1 非线性分工原则

总-分-总不是把所有产物排成单向流水线。

错误理解：

```text
章节契约 -> 章节细纲 -> 场景演出契约 -> 人物意图结果 -> 章节执行包 -> 正文草稿
```

这个理解会重新制造线性复述链和 token 黑洞。

正确理解：

```text
总控规划 + 章节契约
        ↓
分工产物并行/半并行生成：
  - 章节细纲
  - 场景演出契约
  - 人物意图结果
  - 章节执行包
        ↓
正文草稿生成 skill 统一调用
```

分工产物可以局部互参，但只能引用路径、ID、场景编号、风险标签、覆盖标签或短摘要；不得把另一个分工产物当作唯一输入，也不得复制另一个产物的全文。

执行包和人物意图结果尤其不能被理解为固定先后流水线。人物意图结果不是执行包的下游，执行包也不是人物意图结果的总包。二者都服从章节创作总控规划和章节级叙事表达契约。执行包可以记录人物意图覆盖状态和回流需求，人物意图结果也可以参考执行包暴露的风险标签；但两者都不得替代对方。

## 二点五、实施顺序复盘与校正

本轮已经完成部分模板和 skill 改造，但实施顺序存在一个需要校正的问题：

```text
已实际发生：
新增部分模板
-> 直接改造若干 skill
-> 再发现整套模板接口还没有完成全局统一
```

这个顺序不等于方向错误，但存在后续风险：

1. skill 已经开始要求新输入，但上游模板未必都提供对应字段。
2. 新模板已经存在，但旧模板的职责边界尚未统一重审。
3. 执行包和正文草稿生成已改为总-分-总方向，但当时章节契约、细纲、人物意图结果模板还没有完成接口校准。当前人物意图结果模板和正文草稿交付版模板已完成正式收口，剩余模板仍需复核。
4. 如果直接进入第三章测试，可能暴露的不是正文质量问题，而是模板接口不一致问题。

因此，后续实施顺序必须调整为：

```text
模板接口总设计
-> 修正/确认全部关键模板
-> 二次校准已改造 skill
-> 再进入 P0C3 总-分-总链路测试
```

当前已完成的改造不回滚。原因是方向正确：它已经把系统从线性复述链拉回总-分-总链路。下一步不是推倒重来，而是在模板接口层补齐统一规范，再回头校准 skill。

## 二点六、模板接口优先原则

从本节之后，所有 skill 改造都必须服从模板接口规范。

每个模板必须明确：

- 本模板负责什么；
- 本模板禁止承担什么；
- 必须读取哪些上游产物；
- 只允许引用哪些产物；
- 禁止复述哪些内容；
- 输出给哪些下游 skill；
- 产物冲突时按什么优先级处理；
- 哪些问题必须 ASK_USER。

没有完成模板接口确认前，不应继续深改正文生成链路，也不应直接进入第三章完整链路测试。

## 三、产物职责边界

### 3.0 章节创作总控规划

职责：前置总规划。规定本章各产物和 skill 之间如何分工、如何互补、如何引用、如何处理冲突。

它不是剧情大纲，不是正文总体规划，也不是执行包。它只提供创作架构和接口约束。

主要回答：

- 本章总目标是什么；
- 哪些问题交给章节级叙事表达契约；
- 哪些问题交给章节细纲；
- 哪些问题交给场景演出契约；
- 哪些问题交给人物意图控制；
- 哪些问题交给章节执行包；
- 各产物之间哪些地方必须互补；
- 各产物之间哪些内容禁止重复；
- 正文草稿生成时产物冲突如何裁决；
- 哪些冲突必须 ASK_USER。

不得承担：

- 不复述剧情；
- 不写场景细节；
- 不写正文句法；
- 不替代章节细纲；
- 不替代场景演出契约；
- 不替代章节执行包；
- 不成为新的大而全总包。

建议字段：

```markdown
## 章节创作总控规划

### 本章总目标

### 产物分工

|产物|负责内容|禁止承担|必须引用|
|---|---|---|---|

### 互补关系

|关系|说明|
|---|---|

### 引用关系

|生成阶段|必须读取|只可引用|禁止复述|
|---|---|---|---|

### 冲突优先级

### ASK_USER 触发条件

### Token 边界
```

### 3.1 章节级叙事表达契约

职责：规定章节作者意图、叙事边界和读者信息边界。

主要回答：

- 这一章作者想表达什么；
- 读者应该误解什么、知道什么、暂时不知道什么；
- 哪些后台真相不能前台显影；
- 哪些人物不能被写偏；
- 本章相对单元规则有什么差异；
- 作者不接受的叙事方向是什么。

不得承担：

- 不写完整剧情顺序；
- 不写正文句法；
- 不替代章节细纲；
- 不下沉到具体字句自然化。

### 3.2 章节细纲

职责：规定本章主线剧情、场景结构和事件推进。

主要回答：

- 本章发生什么；
- 场景顺序是什么；
- 每场必须完成哪些剧情动作；
- 事件如何推进；
- 伏笔、误会、冲突如何进入场景骨架。

不得承担：

- 不复述章节级叙事表达契约；
- 不替作者决定正文文风；
- 不承担去 AI 化和句法自然化；
- 不把每个场景都写成准正文。

### 3.3 场景演出契约

职责：规定正文文字表达方式，重点服务文风、自然化、去 AI 化和场景文字演出。

主要回答：

- 本章或关键场景的文字质感是什么；
- 句子长短、段落节奏、对白断裂、动作承载如何处理；
- 如何避免说明书式、总结式、AI 腔；
- 心理描写、环境描写、旁白吐槽的边界；
- 关键场景应该怎样写得更像人写；
- 哪些用词、句式、表达方式禁止出现。

不得承担：

- 不重述完整剧情；
- 不替代章节细纲；
- 不替代章节级叙事表达契约；
- 不做执行包的字数预算和生成顺序；
- 不逐场铺满所有普通场景，只处理关键场景或全章文字规则。

### 3.4 人物意图控制

职责：约束人物言行举止符合人设、认知、动机和当下处境。

主要回答：

- 人物为什么说这句话；
- 人物为什么做这个动作；
- 人物此刻知道什么、不知道什么；
- 人物是否有足够动机进入某个选择；
- 关键片段是否需要 ASK_USER 中断。

总-分-总原则：

- 人物意图控制是分工产物之一，不是执行包的下游，也不是执行包的前置流水线步骤。
- 它和章节细纲、场景演出契约、章节执行包一样，共同服从章节创作总控规划和章节级叙事表达契约。
- 它可以局部引用细纲中的场景编号、关系压力和认知边界，也可以引用场景演出契约中的局部表达规则，但不得吞入完整细纲或完整演出契约。
- P0 级人物意图应有总体控制；章节层只做本章差异化处理。
- 执行包可以引用人物意图结果的路径、覆盖标签、风险标签或短摘要，但不得替人物意图控制挑若干高风险片段临时生成轨道。
- 若执行包或正文总导演发现人物意图覆盖不足，只能标记为“回流人物意图控制/ASK_USER”，不得静默自行补意图。

不得承担：

- 不处理正文整体文风；
- 不替代场景演出契约；
- 不把人物动机解释成前台正文；
- 不将所有正文质量问题都归因给人物意图。

### 3.5 章节执行包

职责：调度、索引、权重、风险提示。

主要回答：

- 正文草稿生成需要读取哪些独立产物；
- 每个场景生成顺序是什么；
- 哪些场景重写、中写、轻写；
- 哪些场景存在高风险；
- 哪些人物意图风险已有独立人物意图结果覆盖；
- 哪些人物意图风险覆盖不足，需要回流人物意图控制或 ASK_USER；
- 哪些风险应交给场景演出契约或正文生成处理。

不得承担：

- 不复述所有上游产物；
- 不替代正文草稿生成 skill；
- 不成为唯一正文输入；
- 不把场景演出契约、人物意图控制、章节细纲全文复制进来。
- 不按场景临时生成人物意图轨道。
- 不替代章节级人物意图结果。
- 不把分工产物改回线性流水线。

### 3.6 正文草稿生成 Skill

职责：最终总导演。读取 `章节创作总控规划` 和多个独立产物，生成章节正文草稿。

主要回答：

- 如何按总控规划把作者意图、剧情骨架、文字演出、人物意图、调度权重合成为正文；
- 哪些信息写在前台，哪些只作为后台控制；
- 哪些场景需要完整演出，哪些快速过桥；
- 当独立产物之间发生冲突时，如何按总控规划裁决或 ASK_USER；
- 如何生成自然、可读、符合作者审美的小说正文。

不得承担：

- 不自行重建章节级叙事表达契约；
- 不自行改写章节细纲；
- 不自行替代人物意图控制；
- 不把所有输入展开为说明文。

### 3.7 导演层文本质量规则库

职责：作为正文草稿生成 Skill 的可迭代 reference，承载跨章节复用的文本质量、去 AI 化、对白自然化、幽默误读、句式反 AI、物件承载和作者审美规则。

当前实现位置：

```text
.codex-tmp/skills/generate-chapter-prose-draft/references/director-prose-quality-rules.md
```

它不是独立 skill。它不单独产出文件，不单独和作者共创，也不替代场景演出契约。它属于最终总导演 skill 的规则库，由 `generate-chapter-prose-draft` 在写正文时读取并执行。

拆出 reference 的原因：

1. 文本质量规则迭代速度极快，前期几乎每写一个新章都会从作者反馈中增加、删除或重排规则。
2. 不同章节、不同文风、不同场景类型会快速产生不同规则。如果全部写进 `SKILL.md`，正文生成 skill 会过度膨胀。
3. 场景演出契约只应记录本章/关键场景差异，不应反复复制跨章通用去 AI 化规则。
4. reference 可以按需要渐进加载，后续可拆成多个小文件，而不改变 skill 主流程。

后续拆分路线：

```text
generate-chapter-prose-draft/
  references/
    director-prose-quality-rules.md          # 当前总入口
    quality/
      anti-ai.md                             # 句式、解释腔、总结腔、AI 感
      dialogue-naturalness.md                # 对白断裂、抢话、回避、人物说话胜率
      comedy-and-misread.md                  # 幽默、误会、双层语义、笑点落地
      objects-procedure-pressure.md          # 物件、流程、钱、家庭/职场压力承载
      webnovel-rhythm.md                     # 网文节奏、段落密度、钩子和过桥
      style-mode-light-comedy.md             # 轻喜误会型章节
      style-mode-serious-pressure.md         # 正剧压力型章节
      author-style-p0.md                     # P0 阶段作者审美沉淀
```

拆分触发条件：

- `director-prose-quality-rules.md` 超过约 200 行，或单次生成不需要加载其中大部分内容；
- 同一类规则连续在 2-3 章中被反复修改；
- 作者反馈明显分化为不同文风模式，例如轻喜误会型、正剧压力型、冷幽默反讽型、信息推进型；
- 规则开始需要区分场景类型，例如家庭冲突、服务流程、电话对话、群像压迫、章末钩子；
- 某类规则开始形成独立评估或返修工作流。

只有当这些规则开始承担独立工作流时，才考虑新增 skill。例如：

- 从作者修改稿中抽取风格规则；
- 对比 AI 草稿和作者稿生成风格差异报告；
- 专门做正文质量评估；
- 专门做去 AI 化返修；
- 专门做对白自然化修订。

在此之前，导演层文本质量内容保持为 `generate-chapter-prose-draft` 的 reference，而不是独立 skill。

## 四、推荐新链路

### 4.1 前置总规划层

```text
章节创作总控规划：总目标、产物分工、互补关系、引用关系、冲突优先级、token 边界
```

这一层的作用是防止各产物虽然独立，却彼此脱节。

它只写接口和分工，不写剧情，不写正文，不替代任何下游产物。

### 4.2 分工产物生成层

```text
章节级叙事表达契约：作者意图与边界
章节细纲：剧情与场景骨架
场景演出契约：文字表达与自然化约束
人物意图结果：人物言行与认知轨道
```

这些产物相互独立，各自只输出自己的职责内容；同时它们必须服从 `章节创作总控规划` 的分工和引用关系。

### 4.3 调度层

```text
章节执行包：索引、权重、顺序、风险，不复述全文
```

章节执行包的核心价值是告诉正文草稿生成 skill 如何使用各产物，而不是把各产物重新写一遍。

### 4.4 最终总导演生成层

```text
正文草稿生成 skill：总导演，读取总控规划和各独立产物，统一生成正文
```

正文草稿生成 skill 应直接读取：

- 章节创作总控规划；
- 章节级叙事表达契约；
- 章节细纲；
- 场景演出契约；
- 人物意图结果；
- 章节执行包。

## 五、Token 控制原则

### 5.1 总控规划轻量化

`章节创作总控规划` 是防止产物脱节的接口文件，不是内容总包。

硬性限制：

- 只写总目标、分工、互补、引用、冲突、token 边界；
- 不写完整剧情；
- 不逐场展开；
- 不复述人物卡、事件卡、章节细纲；
- 不写正文示例；
- 默认控制在 500-800 字内。

### 5.2 禁止线性复述

任何 skill 生成产物时，不得默认复述其他产物的完整内容。

允许引用：

- 文件路径；
- 产物编号；
- 场景编号；
- 简短决策摘要；
- 与本 skill 职责直接相关的局部约束；
- 总控规划中的分工、引用和冲突规则。

禁止：

- 大段复制章节细纲；
- 重写章节契约；
- 把人物卡、事件卡重新摘要到每个产物里；
- 每一步都生成“上一阶段内容回顾”。

### 5.3 执行包轻量化

章节执行包应转为轻量调度文件。

建议字段：

```markdown
## 读取产物索引

|产物|路径|状态|用途|
|---|---|---|---|

## 场景调度表

|场景|生成权重|主要输入|风险类型|处理方式|
|---|---|---|---|---|

## 高风险清单

|风险|来源产物|处理层|
|---|---|---|

## 生成顺序

1. S1 ...
2. S2 ...
```

### 5.4 场景演出契约轻量化

场景演出契约只写文字层约束。

建议字段：

```markdown
## 全章文字演出基调

- 句式节奏：
- 对白方式：
- 旁白方式：
- 动作承载：
- 心理描写：
- 环境描写：
- 去 AI 化重点：
- 禁止表达：

## 关键场景文字演出契约

### Sx 场景名

- 文字质感：
- 对白节奏：
- 动作与环境：
- 心理/旁白边界：
- 禁止写法：
```

### 5.5 正文草稿生成读取策略

正文草稿生成 skill 不读取共创讨论全过程，默认只读取确认后的结构化产物和 `章节创作总控规划`。

共创讨论记录只在以下情况读取：

- 作者要求追溯某个决定；
- 产物之间出现冲突；
- 需要判断某项规则是否应回流上级契约。

## 九、后续讨论问题

1. 场景演出契约是否只针对关键场景，还是也需要给全章提供统一文字基调。

3. `章节创作总控规划` 应由哪个步骤生成：章节级叙事表达共创时同步生成，还是正文链路启动时单独轻量生成。
4. 正文草稿生成 skill 在产物冲突时，是否必须 ASK_USER，还是可以按总控规划中的优先级自动裁决。

6. 章节执行包保留字数预算，但由章节创作总控规划决定总目标；执行包只引用总目标并拆成场景级建议范围。
7. 章节执行包已确认是临时产物，正文确定后可以归档，不进入长期世界事实。

## 十、当前结论

本轮改造不应继续增加线性层级，也不应把各产物做成完全松散的并列文件。

正确方向是：

```text
总：章节创作总控规划统一目标、分工、接口、引用和冲突规则
分：独立产物各司其职，按总控规划互补生成
总：正文草稿生成 skill 作为总导演统一调用并生成正文
```

这样既能避免线性复述造成 token 黑洞，也能避免完全并联导致各产物衔接不上。正文生成时，模型不是硬拼几份文件，而是在同一个章节创作总控规划下调度各部门产物，生成统一、自然、符合作者审美的正文。
