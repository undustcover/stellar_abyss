## 三、下一步唯一任务

# 首轮独立生成 Public 测试

作者提供一份新的 Draft 后，系统直接依据现有正式经验生成 Public，不预先读取该章作者修改稿：

1. 冻结作者提供的 `Input Draft`；
2. 只读取正式审美库、正式笑话方法库和理解 Draft 必需的最小上下文；
3. 独立完成诊断、方向裁决和完整 `System Public V1` 生成；
4. 通过事实、人物、POV、信息和模板化门禁后冻结 V1；
5. 作者基于 V1 修改成 `Author Public V2`；
6. 后续只比较 V1／V2，校准修改原因并继续更新审美库；
7. 原始 Draft 只作为输入和基线，不再作为作者审美答案。

## 五、独立测试边界

### 生成前

- 作者只提供 Draft，不提供目标 Public；
- 系统不得要求作者先替系统完成审美选择；
- 系统内部完成诊断、方向提出与裁决；
- 正式库按适用条件调用，不机械堆叠全部条目。

### 生成后

- System Public V1 交付后必须冻结；
- Author Public V2 另存，不覆盖 V1；
- 新案例比较 V1／V2，而不是重新比较 Draft／V2；
- 作者确认前，模型归因仍只是候选解释；
- 下一轮正文模型只读取更新后的抽象库，不读取 V2 原句。

## 七、已解决且不要重复询问的问题

### A4 人物非逻辑性

人物异常行为必然存在可追溯的人物原因。正文可以不解释、不展示，甚至让当前读者无法知道该原因；但系统不能把“人物不可控”误解为无来源的随机行为。

### 主角与视角

马木是实际主角。马雯雯承担线索人物和限知视角功能，类似通过观察者视角隐藏实际主角信息。不能因为使用马雯雯视角而减少马木的主角笔墨和立体性。

### 旁白

作者不反对强旁白。旁白必须有新增作用、服从场景底色并服务读者体验。

### 第一章结尾

主要功能是喜剧反差，钩子是顺带结果；不要重新把它解释成以悬念为第一目标。
