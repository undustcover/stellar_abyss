# 大纲 / 细纲意图控制接入改造方案

## 一、核心结论

大纲、章节大纲、章节细纲都需要接入意图控制。

但它们不能承担正文级动态对齐。

正确分层是：

```text
单元大纲 / 章节大纲：确认人物意图方向是否成立
章节细纲：预埋人物意图冲突和风险
章节执行包：提供意图控制器所需原材料
人物意图控制 Skill：生成轨道、判断中断、更新运行态
正文生成 Skill：根据已对齐轨道写正文
```

上游只标方向、压力、风险。

正文前才生成具体人物意图轨道。

---

# 二、为什么上游也需要接入

如果大纲和细纲完全不接入意图控制，正文阶段会太晚。

常见问题：

1. 人物为什么进入某章没有动机；
2. 人物为什么留在某场没有理由；
3. 人物关系变化只服务章节功能；
4. 关键对白风险在执行包才发现；
5. 正文生成时只能临时补救上游结构缺口；
6. 人物行为逻辑崩坏后，正文只能整段重写。

因此，上游必须提供人物意图材料。

但上游不能写死：

- 每句台词的完整语言意图；
- 每个动作的最终表达方式；
- 正文片段级人物意图轨道；
- 成品对白；
- 成品笑点。

---

# 三、整体接入层级

```text
资料调用层
    ↓
提供人物长期边界、认知状态、关系状态

单元大纲层
    ↓
确认单元人物驱动力和人物弧线是否成立

章节大纲层
    ↓
确认本章人物目标、关系压力、认知推进

章节细纲层
    ↓
拆出场景，并标注人物进入场景的基础意图与风险

章节执行包层
    ↓
标出关键对白风险、关键行为风险、禁止替人物决定项

人物意图控制层
    ↓
生成关键片段人物意图轨道，决定 CONTINUE / ASK_USER

正文生成层
    ↓
按已确认轨道生成正文片段并组装整章
```

---

# 四、单元大纲接入方案

## 4.1 接入目标

单元大纲不做细节意图控制，只检查宏观人物驱动力。

它应回答：

```text
这个单元中，人物为什么会持续参与？
人物关系为什么会发生变化？
主角认知为什么会推进？
人物行为是否来自事件事实，而不是作者安排？
```

## 4.2 建议新增字段

```markdown
## 人物意图方向

|人物|单元内核心欲望 / 压力|推动其持续参与的原因|认知变化方向|关系变化方向|风险|
|---|---|---|---|---|---|
```

```markdown
## 人物驱动力检查

|检查项|结论|
|---|---|
|主角是否有足够理由进入本单元事件||
|关键配角是否有独立目标，而不是只服务主角||
|人物关系变化是否有事件压力支撑||
|人物认知推进是否符合已知信息||
|是否存在需要作者确认的人物动机缺口||
```

## 4.3 禁止事项

单元大纲不得：

- 写正文对白；
- 写关键片段人物意图轨道；
- 按章节替人物设定临时语言策略；
- 把临时单元功能写回人物卡。

---

# 五、章节大纲接入方案

## 5.1 接入目标

章节大纲负责确认本章人物意图方向。

它应回答：

```text
本章每个关键人物为什么出现？
本章人物想得到什么？
本章人物认知有什么推进或误判？
本章人物关系压力在哪里？
本章哪些人物决定不能由模型自行补全？
```

## 5.2 建议新增字段

```markdown
## 本章人物意图总表

|人物|进入本章的原因|本章外在目标|本章隐藏压力|本章认知边界|本章出口状态|
|---|---|---|---|---|---|
```

```markdown
## 本章人物分歧风险

|人物|风险类型|可能分歧|影响范围|是否需作者确认|
|---|---|---|---|---|
```

风险类型包括：

- 语言意图；
- 行为意图；
- 认知边界；
- 关系变化；
- 信息释放。

## 5.3 禁止事项

章节大纲不得：

- 写完整对话策略；
- 写每个关键片段的意图轨道；
- 用章节功能替代人物动机；
- 为了钩子强行安排不符合人物的行为。

---

# 六、章节细纲接入方案

## 6.1 接入目标

章节细纲是上游接入意图控制的关键层。

它需要在场景级别标出：

```text
人物进入场景时知道什么
人物进入场景时想得到什么
人物与对方的关系压力是什么
本场哪些行为可能不成立
本场哪些对白可能需要意图控制器检查
```

细纲仍不生成完整人物意图轨道。

## 6.2 建议新增字段

每场新增：

```markdown
### 人物进入状态

|人物|进入本场的原因|当前认知|当前外在目标|当前关系压力|出口变化|
|---|---|---|---|---|---|
```

```markdown
### 人物行为风险

|人物|潜在行为|风险点|为什么可能不成立|下游检查要求|
|---|---|---|---|---|
```

```markdown
### 人物语言风险

|人物|可能对白方向|风险点|不能说出的内容|下游检查要求|
|---|---|---|---|---|
```

```markdown
### 意图控制器关注点

- 本场进入正文前，必须检查：
  - ...
- 可由模型自行处理：
  - ...
- 若不明确，必须 ASK_USER：
  - ...
```

## 6.3 细纲 Skill 修改点

`generate-chapter-detail-outline` 需要新增：

1. 识别每场关键人物；
2. 标出人物进入本场的基础认知；
3. 标出人物进入本场的外在目标；
4. 标出人物关系压力；
5. 标出行为动机缺口；
6. 标出语言意图风险；
7. 标出下游意图控制重点；
8. 保持正文表现空间，不生成成品对白。

## 6.4 禁止事项

细纲不得：

- 为每句台词分配完整语言目的；
- 写关键片段完整人物意图轨道；
- 生成成品对白；
- 让每场都变成作者审问表；
- 把不确定的作者意图伪装成已确认事实。

---

# 七、章节执行包接入方案

## 7.1 接入目标

执行包是意图控制器的直接上游。

它应把细纲中的人物风险转化为正文前可检查材料。

## 7.2 建议新增字段

```markdown
## 人物意图控制接口

### 关键人物关系压力

|人物|面对谁|关系压力|会影响的语言 / 行为|
|---|---|---|---|

### 人物认知边界

|人物|知道|不知道|误以为|怀疑|不能判断|
|---|---|---|---|---|---|

### 关键行为风险

|片段候选|人物|行为|为什么高风险|必须检查的问题|
|---|---|---|---|---|

### 关键对白风险

|片段候选|人物|对白方向|为什么高风险|不能说出的内容|
|---|---|---|---|---|

### 禁止替人物决定项

- ...

### 意图控制器建议检查片段

|片段候选|涉及人物|检查重点|建议风险等级|
|---|---|---|---|
```

## 7.3 执行包 Skill 修改点

`generate-chapter-execution-pack` 需要新增：

1. 从细纲继承人物进入状态；
2. 将行为风险转化为关键行为候选；
3. 将语言风险转化为关键对白候选；
4. 标注禁止替人物决定项；
5. 标注建议检查片段；
6. 不生成完整人物意图轨道；
7. 不代替作者决定 P0 人物意图分歧。

---

# 八、正文生成接入方案

正文生成不再直接从执行包一次生成整章。

新链路：

```text
执行包
    ↓
意图控制器生成关键片段人物意图轨道
    ↓
CONTINUE / ASK_USER
    ↓
正文生成 Skill 生成当前片段
    ↓
更新章节运行态
    ↓
下一关键片段
```

正文生成 Skill 必须：

- 读取意图控制器输出；
- 读取章节运行态；
- 按关键片段生成；
- 不自行忽略 ASK_USER；
- 最后组装整章 V1。

---

# 九、资料调用层接入方案

资料调用层需要为意图控制提供必要人物摘要。

建议提取：

- 人物长期语言习惯；
- 人物行为边界；
- 人物当前世界内状态；
- 人物已知 / 未知 / 误解信息；
- 人物关系状态；
- 人物不会说或不会做的内容及其原因。

注意：

```text
资料调用层只提取世界内人物事实。
不生成章节临时意图。
```

---

# 十、是否需要动态中断

上游大纲 / 细纲阶段不采用正文级动态中断。

它们只输出：

- 待作者确认问题；
- 人物动机缺口；
- 人物分歧风险；
- 下游必须检查项。

真正的动态中断主要发生在：

```text
人物意图控制 Skill
正文生成前 / 正文生成中
```

原因：

```text
上游阶段如果频繁中断，会把创作流程变成问卷；
正文阶段才最需要检查具体人物为什么说、为什么做。
```

---

# 十一、接入优先级

## P0

必须改：

- 章节执行包；
- 正文生成；
- 独立人物意图控制 Skill；
- 章节运行态。

## P1

必须轻改：

- 章节细纲。

## P2

后续评估：

- 单元大纲；
- 章节大纲；
- 资料调用摘要；
- 单场景执行单。

---

# 十二、启动判断

当前文档已经足以支持正式启动。

启动顺序应为：

```text
1. 创建独立人物意图控制 Skill
2. 改造章节执行包 Skill
3. 改造章节细纲 Skill
4. 改造正文生成 Skill
5. 用一章做完整链路测试
```

不建议先改单元大纲和章节大纲。

原因：

```text
当前最核心的失败发生在正文生成前后的关键片段，
先打通“执行包 -> 意图控制 -> 正文生成”闭环，
再向上反推单元大纲和章节大纲字段。
```
