# 人物意图控制 Skill 第一版需求

## 一、定位

人物意图控制 Skill 是正文生成过程中的动态对齐层。

它不直接替代章节细纲、章节执行包或正文生成 Skill，而是在正文生成前和生成过程中，显式建立并校准当前关键片段的人物意图轨道。

系统核心目标是：

```text
通过人物语言意图和人物行为意图，控制人物成立与剧情推进。
```

也就是说，正文不是先由章节需要决定人物说什么、做什么，而应当由人物在当前认知、关系、压力和目标下的语言与行为自然推动剧情。

---

# 二、层级定义

人物意图控制系统采用以下层级：

```text
交付单位：整章
结构单位：场景
控制单位：人物意图
落地单位：对话与行为
实际输出单位：关键片段
验证单位：关键对白 / 关键动作 / 最小片段
```

其中：

- 语言意图使用“一轮有效语言行为”作为最小分析单位；
- 行为意图使用“关键动作因果链”作为最小分析单位；
- 正文生成时以“小段落 / 对白回合 / 关键片段”作为实际输出单位。

---

# 三、第一版工作方式

第一版采用显式人物意图轨道。

每进入一个关键片段前，系统必须先列出相关人物的意图轨道，再生成正文片段。

成熟后可以预留内部检查模式：

```text
高风险片段：显式展示人物意图轨道
低风险片段：内部检查，直接 CONTINUE
```

但第一版默认全部显式展示。

---

# 四、输入资料

人物意图控制 Skill 可以读取：

1. 章节细纲；
2. 章节执行包；
3. 章节运行态；
4. 人物卡中的长期人格、语言习惯、行为边界、认知历史和关系状态；
5. 事件卡、设定卡中的事实限制；
6. 当前已经生成的正文片段；
7. 作者在对话中刚刚确认的意图规则。

人物卡只提供长期世界事实和长期人物边界。

当前场景下的临时目标、语言策略、行为选择，应写入章节运行态，不直接回写人物卡。

---

# 五、输出资料

本 Skill 输出三类内容。

## 5.1 CONTINUE

表示当前关键片段的人物语言意图和行为意图足够明确，可以继续生成正文。

第一版仍应展示人物意图轨道，然后继续生成。

## 5.2 ASK_USER

表示存在关键人物意图分歧，必须停止正文生成并向作者确认。

触发 ASK_USER 后，当前轮不得继续生成正文。

## 5.3 RESUME

作者确认后，系统必须：

1. 将作者回答转换为明确规则；
2. 判断影响范围；
3. 更新章节运行态；
4. 简短复述确认结果；
5. 从中断点继续生成正文片段。

---

# 六、人物意图轨道

每个关键片段生成前，应建立轻量人物意图轨道。

建议结构：

```yaml
intent_checkpoint:
  chapter_id:
  scene_id:
  fragment_id:
  fragment_scope:
  status: READY_TO_WRITE

character_intent_tracks:
  人物名:
    behavior_goal:
    behavior_motive:
    immediate_action:
    why_not_simpler_action:
    language_goal:
    language_strategy:
    emotional_pressure:
    knowledge:
      knows:
      does_not_know:
      assumes:
      suspects:
    forbidden:
    expected_effect_on_other:
    expected_consequence:
```

字段说明：

|字段|作用|
|---|---|
|behavior_goal|人物当前想完成什么|
|behavior_motive|人物为什么要这样做|
|immediate_action|当前片段准备采取的行动|
|why_not_simpler_action|为什么不采用更简单、更直接的行动|
|language_goal|人物本轮开口想实现什么|
|language_strategy|人物用什么方式表达或遮掩|
|emotional_pressure|当前情绪压力如何影响表达|
|knowledge|人物当前知道、不知道、误以为、怀疑什么|
|forbidden|此人物当前绝不能说或做什么|
|expected_effect_on_other|希望对方产生什么理解或反应|
|expected_consequence|语言或行为将造成什么后果|

---

# 七、语言意图控制

一轮有效语言行为的分析结构为：

```text
人物受到刺激
→ 形成语言目的
→ 选择表达策略
→ 说出具体内容
→ 对方按照自己的认知理解
→ 产生反应
```

语言意图必须检查六个维度：

1. 稳定语言人格；
2. 当前语言目标；
3. 当前认知边界；
4. 当前情绪压力；
5. 表达策略；
6. 语言禁区。

正文生成前必须确认：

- 人物为什么开口；
- 想让对方知道什么；
- 想让对方误解什么；
- 想从对方口中得到什么；
- 哪些内容不能说；
- 用什么方式说；
- 说出口的话和真实想法之间有什么距离。

---

# 八、行为意图控制

关键行为必须形成完整因果链：

```text
外部刺激
→ 人物认知
→ 人物目标
→ 可选行动
→ 人物选择
→ 选择理由
→ 行动结果
→ 状态变化
```

特别必须检查：

```text
如果人物没有采取最直接行动，必须有认知、情绪、关系、利益或外部压力理由。
```

例如人物不直接问、不直接走、不直接报警、不直接拒绝、不直接解释时，必须说明其人物内理由。

若理由无法成立，触发 ASK_USER。

---

# 九、强制中断条件

## 9.1 一级强制中断：人物语言意图分歧

出现以下任一情况时，必须 ASK_USER：

- 不清楚人物为什么开口；
- 同一句台词可以服务于两个不同目标；
- 不清楚人物是在试探、解释、遮掩、反击、敷衍还是转移话题；
- 不清楚人物想让对方知道还是误解；
- 不清楚人物是否会直接说出某件事；
- 不清楚人物在此关系下会采用什么态度；
- 两种语言方向会塑造出不同人物形象；
- 台词虽然能推动剧情，但可能违背人物。

## 9.2 二级强制中断：人物行为意图分歧

出现以下任一情况时，必须 ASK_USER：

- 不清楚人物当前目标；
- 同一人物存在两个同样合理的行动选择；
- 当前行为只服务剧情，不符合人物；
- 行为会显著改变情节走向；
- 人物主动还是被动不明确；
- 人物为什么不采取更简单行动无法解释；
- 行为会改变人物关系；
- 行为意味着人物态度发生转变。

## 9.3 三级可选中断：叙事意图分歧

节奏、详略、伏笔明显程度、喜剧强弱、环境比例、场景结束位置等问题，只有在反过来影响人物语言或行为时，才触发 ASK_USER。

否则模型自行处理。

---

# 十、中断提问格式

触发 ASK_USER 时，必须输出：

```markdown
## 人物意图中断：需要确认一个关键分歧

### 已确认

- ...

### 当前分歧

A方向：...

B方向：...

### 我的判断

建议采用...

原因：...

### 最小验证片段

...

请确认 A、B，或直接改写人物意图轨道。
```

作者可以直接改写人物意图轨道，系统必须接受并转化为章节运行态规则。

---

# 十一、恢复生成格式

作者回答后，系统必须先复述确认结果：

```markdown
已确认：
- ...

影响范围：
- 当前关键片段 / 当前场景 / 本章后续 / 长期人物规则候选

已更新章节运行态。

继续当前正文片段：
```

随后从断点继续生成。

不得重复前文，不得重写整章，不得把临时意图直接写回人物卡。

---

# 十二、与其他 Skill 的关系

## 12.1 与章节细纲 Skill

章节细纲提供：

- 场景结构；
- 每场功能；
- 人物认知边界；
- 信息隐藏要求；
- 可能的关键人物分歧。

它不生成完整人物意图轨道。

## 12.2 与章节执行包 Skill

章节执行包提供意图控制器所需的原材料：

- 人物关系压力；
- 人物认知边界；
- 本场可能误解点；
- 关键行为风险；
- 关键对白风险；
- 禁止替人物决定项。

它不写死每一句对白或每个动作的最终意图。

## 12.3 与正文生成 Skill

正文生成 Skill 根据人物意图轨道输出正文片段。

正文生成时，每个关键对白和关键行为都必须服务人物自身意图，而不是只服务大纲说明。

---

# 十三、验收标准

第一版通过验收，至少满足：

1. 能识别人物为什么说、为什么做的分歧；
2. 不会在人物意图不明时继续生成；
3. 作者可以直接修改人物意图轨道；
4. 作者确认后能更新章节运行态；
5. 生成正文中的对白符合人物目标、认知、情绪和关系；
6. 行为选择能形成明确因果链；
7. 剧情推进来自人物选择和后果，而不是人物配合大纲；
8. 笑点来自人物认真达成自身目标时产生的错位，而不是强塞段子；
9. 临时人物意图不污染人物卡；
10. 最终仍能组装为完整整章 V1。
