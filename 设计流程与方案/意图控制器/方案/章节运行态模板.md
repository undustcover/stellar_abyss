# 章节运行态模板

## 一、用途

章节运行态用于保存一章正文生成过程中的临时意图规则、断点位置、人物意图轨道、已解决分歧和正文生成进度。

每章一份。

建议命名：

```text
CXX_章节运行态_意图控制.md
```

章节运行态不是人物卡，不保存长期人物设定。

当前场景下的临时目标、语言策略、行为选择，只能先进入章节运行态。章节结束后，再判断是否有必要沉淀为长期人物规则候选。

# 六、已确认人物意图规则

作者确认后的规则写入此处。

```yaml
resolved_intents:
  - id:
    source_checkpoint:
    character:
    type: language_intent # language_intent / behavior_intent / cognition / relationship / reader_info
    decision:
    normalized_rule:
    scope: 当前片段 # 当前片段 / 当前场景 / 本章后续 / 长期规则候选
    confirmed_by_author: true
    created_at:
```

示例：

```yaml
resolved_intents:
  - id: C03-S02-LANG-01
    source_checkpoint: C03-S02-F01
    character: 马雯雯
    type: language_intent
    decision: 她是在试探陈艾艾，不是直接质问。
    normalized_rule: 马雯雯假装自己已经知道一部分，只追问细节；对白不能表现为笃定指控。
    scope: 当前场景
    confirmed_by_author: true
    created_at: 2026-06-22
```

# 九、正文片段记录

```yaml
draft_fragments:
  - id:
    scene_id:
    checkpoint_id:
    status: generated # generated / revised / blocked / assembled
    start_anchor:
    end_anchor:
    summary:
    inherited_rules:
      -
    unresolved_risks:
      -
```

正文片段可以在对话框中生成，最终组装进正文草稿文档。

运行态只记录片段摘要、断点和继承规则，不保存大量正文。

# 十、长期规则候选

章节结束后，再判断哪些内容值得沉淀到人物卡或系统规则。

```yaml
long_term_rule_candidates:
  - id:
    source:
    character:
    candidate_rule:
    reason:
    target_file:
    status: pending_review
```

注意：

```text
章节运行态不能自动回写人物卡。
```

是否沉淀为长期人物规则，必须经过作者确认。

---

# 十一、使用流程

```text
读取章节细纲和执行包
    ↓
创建本章章节运行态
    ↓
进入关键片段
    ↓
显式列人物意图轨道
    ↓
CONTINUE 或 ASK_USER
    ↓
作者确认或修改轨道
    ↓
更新 resolved_intents / resolved_divergences
    ↓
生成正文片段
    ↓
记录 draft_fragments
    ↓
进入下一关键片段
    ↓
组装整章 V1
```

---

# 十二、边界

章节运行态保存：

- 当前章临时人物意图；
- 作者确认的本章规则；
- 分歧解决记录；
- 断点；
- 正文片段生成状态；
- 长期规则候选。

章节运行态不保存：

- 完整人物卡；
- 长期世界事实；
- 完整正文全文；
- 未经作者确认的长期人物结论；
- 通用写作方法论长篇复述。
