# 正文生成 Skill 改造点

## 一、改造目标

当前正文生成 Skill 的核心职责需要从：

```text
根据章节执行包一次生成完整章节正文
```

调整为：

```text
在整章交付目标下，按关键片段显式执行人物意图轨道，对齐后连续生成正文，最终组装完整章节 V1。
```

交付单位仍然是整章。

但正文生成过程可以跨多轮完成。

本改造依赖独立人物意图控制 Skill：

```text
control-character-intent
```

正文生成 Skill 不应自行吞并意图控制层。它接收意图控制 Skill 输出的 CONTINUE / ASK_USER / RESUME、人物意图轨道和章节运行态规则，再负责生成正文片段与组装整章。

同时，第三章第二次测试确认：

```text
人物意图控制器解决的是人物为什么说、为什么做。
正文生成 Skill 仍需要负责正文输出控制：
读者如何获得信息、视角如何过滤信息、哪些内容轻写或留白、文本如何避免过满和说明文化。
```

当人物意图已经正确，但正文仍出现“所有内容都写满、视角过滤失败、AI感强、没有留白、读者提前识别真相”等问题时，不应继续把责任推给人物意图控制器，而应进入正文输出策略或草稿后作者意图修订。

---

# 二、新流程

```text
读取章节细纲
    ↓
读取章节执行包
    ↓
读取或创建章节运行态
    ↓
确认章节级意图
    ↓
确认正文输出策略
    ↓
进入关键片段
    ↓
显式列人物意图轨道
    ↓
判断 CONTINUE / ASK_USER
    ├─ CONTINUE：生成正文片段
    └─ ASK_USER：停止正文，等待作者确认
    ↓
作者确认后更新章节运行态
    ↓
从断点继续
    ↓
所有关键片段完成
    ↓
组装整章 V1
    ↓
整章一致性检查
    ↓
输出正文草稿与生成备注
    ↓
作者整章审阅
    ↓
若为正文表现偏差，进入作者意图修订
```

---

# 三、正文生成前新增动作

每个关键片段生成前，正文生成 Skill 必须执行：

```text
1. 确认本片段涉及哪些关键人物；
2. 为每个关键人物生成意图轨道；
3. 检查人物语言意图是否明确；
4. 检查人物行为意图是否明确；
5. 检查人物是否有理由不采取更简单行动；
6. 检查人物认知是否支撑其对白和行为；
7. 检查人物禁区；
8. 输出 CONTINUE 或 ASK_USER。
```

第一版必须显式输出人物意图轨道。

---

# 三点五、正文输出策略

正文生成前，正文生成 Skill 应读取或建立轻量 `正文输出策略`。

它的职责是控制作者希望本章如何被读者读到，而不是控制人物为什么说或为什么做。

建议在正文生成前确认：

```text
1. 本章读者是否只能通过某个视角人物获得信息；
2. 读者是否应和视角人物一起被误导；
3. 哪些正确线索不得被读者识别为正确线索；
4. 哪些内容只能作为背景噪音、生活细节、错接或物件出现；
5. 哪些场景重写，哪些场景轻写；
6. 哪些执行包内容不得写满；
7. 本章允许多少留白；
8. 哪些句段结构会导致 AI 感、说明文化或板正感。
```

输出策略示例：

```yaml
prose_output_strategy:
  viewpoint_filter: 马雯雯
  reader_rule: 读者只能通过马雯雯理解云水汇，马雯雯被误导时读者也应被误导
  main_reader_effect: 从会所误会滑向中老年养生理疗误会
  must_not_signal_to_reader:
    - 诈骗
    - 杀猪盘
    - 反诈线索
    - 严重犯罪
  correct_clues_handling:
    马木追问: 写成马雯雯眼中的堂哥查养生馆违规服务老人，不写成读者可识别的诈骗排查
    手机离手/家属不懂: 降噪为服务过界和老人被哄，不形成明确风险清单
  density_rule:
    write_full: 陈科体面护短与马雯雯误读改类
    write_light: 马木追问、陈艾艾焦虑、陈灏挑刺
    leave_blank: 云水汇真实风险、马木判断逻辑、后续证据意义
  prose_texture_rule:
    - 允许无效动作、半截话、错接和松散停顿
    - 避免每段都落结论
    - 避免把执行包控制点全部展示
```

正文输出策略不应变成长篇分析，也不应塞入正文主体。

它只作为正文生成和后续修订的表现层约束。

---

# 四、关键片段定义

关键片段可以是：

- 一个重要对白回合；
- 一个关键行为选择；
- 一个误会升级段；
- 一个信息释放段；
- 一个强笑点落地段；
- 一个关系变化段；
- 一个场景入口或出口段；
- 一个会影响后续剧情的动作链。

关键片段不是整场成品，也不是单句台词。

正文输出时以小段落、对白回合或连续动作段为单位。

---

# 五、CONTINUE 规则

只有满足以下条件时，才能 CONTINUE：

1. 关键人物为什么说、为什么做已经明确；
2. 人物行为目标、语言目标、认知边界、表达策略成立；
3. 人物没有采取更简单行动的理由成立；
4. 对话不会变成说明书；
5. 行为不会只服务剧情；
6. 信息边界不会被提前突破；
7. 人物声音和人物长期边界不冲突；
8. 当前片段能自然推动后果。

第一版 CONTINUE 时仍应展示轨道：

```markdown
## 人物意图轨道

...

## 检查结果

CONTINUE：不存在需要作者确认的人物语言或行为分歧。

## 正文片段

...
```

---

# 六、ASK_USER 规则

只要不清楚人物为什么说或为什么做，即使场景目标明确，也必须 ASK_USER。

强制中断优先级：

```text
P0：人物语言意图分歧
P0：人物行为意图分歧
P1：认知边界 / 信息差 / 误会机制
P1：场景主动权 / 伏笔 / 喜剧机制
P2：节奏 / 详略 / 环境 / 句子风格
```

P2 问题不得单独触发中断，除非影响人物语言或行为。

---

# 七、作者修改轨道

作者可以直接修改人物意图轨道。

例如系统判断：

```text
陈科语言目标：证明项目可靠。
```

作者可以改为：

```text
不是证明项目可靠，而是证明自己不是被人骗的蠢货。
```

正文生成 Skill 必须将该修改转化为章节运行态规则：

```text
陈科当前语言目标不是解释项目本身，而是维护自尊，避免承认自己判断失误。
```

随后再继续生成。

---

# 八、恢复生成规则

恢复时必须先复述：

```markdown
已确认：
- ...

影响范围：
- ...

已更新章节运行态。

继续当前正文片段：
```

不得：

- 重复前文；
- 从章节开头重写；
- 忘记已生成片段；
- 把临时意图直接写回人物卡；
- 在问题后继续生成正文。

---

# 九、正文片段生成规则

正文片段必须从人物意图中长出来。

生成顺序为：

```text
人物当前认知
→ 人物当前目标
→ 人物选择语言或行为策略
→ 对话与行动
→ 对方按自身认知理解
→ 现实后果
→ 下一片段压力
```

禁止写法：

```text
章节需要发生某件事
→ 安排人物说出必要信息
→ 安排人物做出必要动作
```

---

# 十、对白检查

每个关键对白片段生成后，必须内部检查三层。

## 10.1 意图正确性

- 每句话是否服务人物当前目标；
- 人物是否有理由说这句话；
- 人物是否说出了不该知道的内容；
- 人物是否为了向读者解释而开口。

## 10.2 人物声音

- 去掉姓名后是否能判断是谁说的；
- 句式、词汇、态度是否符合人物；
- 人物是否突然变得过度聪明、幽默或完整；
- 不同人物是否共享同一种 LLM 语言。

## 10.3 对白作用结果

- 对方为什么这样理解；
- 这句话是否改变关系、认知或行动；
- 删除这句话后场景是否没有变化；
- 是否存在没有语言行为功能的空对白。

---

# 十一、行为检查

每个关键行为片段生成后，必须检查：

- 行为来自人物目标，而不是剧情指令；
- 行为依据来自人物已知信息；
- 行为是否有可选项；
- 人物为什么选择当前行动；
- 人物为什么不采用更简单行动；
- 行为造成了什么现实后果；
- 行为是否改变人物关系或下一步压力。

---

# 十二、强笑点处理

强笑点必须服从人物语言意图。

生成顺序：

```text
人物语言目的
→ 人物真实表达
→ 对方错误理解或预期被打破
→ 形成笑点
```

禁止：

```text
需要一个笑点
→ 编一句有梗的话
→ 强行塞给人物
```

如果笑点会破坏人物目标、人物认知或人物声音，必须降级或删除。

---

# 十三、生成备注改造

正文草稿的生成备注应新增：

|项目|内容|
|---|---|
|人物意图轨道执行情况||
|语言意图分歧处理||
|行为意图分歧处理||
|作者直接改写轨道记录||
|为什么不采取更简单行动检查||
|人物语言禁区风险||
|人物行为动机风险||
|临时规则是否进入章节运行态||
|长期规则候选||
|正文输出策略执行情况||
|视角过滤执行情况||
|读者误导执行情况||
|信息降噪与留白执行情况||
|内容过满风险||
|说明文化 / AI感结构风险||

生成备注不得把完整过程分析塞入正文主体。

---

# 十四、草稿后作者意图修订

当作者审阅整章草稿后指出以下问题：

```text
内容太多
所有地方都写满
展示角度错误
读者没有被视角人物误导
读者通过第三方信息提前看懂真相
文章不好读
所有场景都像重点
文本像说明文
AI感强
句子和段落太板正
没有留白
```

正文生成 Skill 不应默认回退到人物意图控制器，也不应只做替换词语式润色。

应先归因为：

```text
正文表现偏差 / 作者意图修订
```

处理方式：

```text
1. 保持同一组事实、人物决定和世界边界；
2. 重新确认作者希望的读者体验；
3. 收紧或调整视角过滤；
4. 删除、降噪或背景化过于清楚的正确线索；
5. 重排详略，保留一个主阅读体验；
6. 增加留白、半截话、错接、无效动作和自然停顿；
7. 松动句段结构，避免每段都承担功能和结论；
8. 只在必要处局部重写，不因表现过满而默认整章推翻。
```

草稿后作者意图修订可以使用完整草稿作为修改对象。

原因：

```text
此时 LLM 已经理解文章内容；
问题不是内容不够，而是内容太多、展示方式错误；
因此应基于草稿做减法和表现方式调整。
```

但系统建设上，应把其中可前置的规则沉淀到 `正文输出策略`，减少以后每章都先生成过满草稿再大删。

---

# 十五、与现有正文生成 Skill 的具体修改点

需要修改 `generate-chapter-prose-draft`：

1. 将“Draft the whole chapter in one pass”改为“以整章为交付单位，允许跨多轮按关键片段连续生成”；
2. 增加章节运行态读取和更新；
3. 增加正文输出策略读取或临时建立；
4. 增加关键片段识别；
5. 增加显式人物意图轨道；
6. 增加 CONTINUE / ASK_USER / RESUME 协议；
7. 增加作者可直接改写人物意图轨道；
8. 增加“为什么不采取更简单行动”硬检查；
9. 增加语言意图和行为意图为 P0 中断条件；
10. 将场景检查降级为结构检查；
11. 增加草稿后作者意图修订归因；
12. 最终仍输出完整整章 V1。

需要修改 `generate-chapter-execution-pack`：

1. 增加人物关系压力；
2. 增加人物认知边界；
3. 增加本场可能误解点；
4. 增加关键行为风险；
5. 增加关键对白风险；
6. 增加禁止替人物决定项；
7. 为意图控制器提供原材料，但不写死完整人物意图轨道。

需要轻改 `generate-chapter-detail-outline`：

1. 标出每场涉及的核心人物；
2. 标出人物进入本场的基础认知；
3. 标出可能影响剧情的人物目标冲突；
4. 标出信息隐藏边界；
5. 标出需要下游意图控制器重点检查的片段。

---

# 十六、第一版验收标准

改造后正文生成 Skill 至少应满足：

1. 能按关键片段生成，而不是强行一次输出整章；
2. 每个关键片段前能显式列人物意图轨道；
3. 人物语言意图不明时会中断；
4. 人物行为意图不明时会中断；
5. 作者修改轨道后能正确吸收；
6. 恢复生成前会复述确认；
7. 临时意图进入章节运行态；
8. 正文对白不再只服务信息解释；
9. 正文行为不再只服务剧情推进；
10. 能根据正文输出策略控制读者获得信息的角度和数量；
11. 能识别“人物意图正确但正文表现失败”的情况；
12. 能把内容过满、视角过滤失败、AI感和留白不足归入作者意图修订；
13. 最终能组装为完整、连贯、可进入精修的整章 V1。
