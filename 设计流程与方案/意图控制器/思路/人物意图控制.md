

意图控制系统真正的核心应该是：

> **通过人物语言意图和人物行为意图，控制人物成立与剧情推进。**

其他内容大多只是为这两类意图提供约束条件。

原设计虽然提到了人物认知和场景目标，但没有把“人物到底为什么说这句话、为什么做这个动作”设为生成前的核心控制对象，因此仍然可能出现一种常见失败：

- 情节走向基本正确；
    
- 信息没有提前暴露；
    
- POV也没有违规；
    
- 但人物说出来的话不像本人；
    
- 人物为了推动情节而突然采取不符合自身目的的行为；
    
- 最终正文逻辑正确，却没有人物生命力。
    

这正是目前设计中最严重的缺口。

# 一、重新确定意图控制的优先级

整个系统应调整为三个层级。

## 第一优先级：人物语言意图

控制人物在当前对话中：

- 为什么开口；
    
- 想让对方知道什么；
    
- 想让对方误解什么；
    
- 想从对方口中得到什么；
    
- 哪些内容不能说；
    
- 用什么方式说；
    
- 说出口的话和真实想法之间有什么距离。
    

人物语言不是单纯的“语言风格”。

不能只建立：

```text
马木：直白、口语化、喜欢开玩笑
马雯雯：现实、吐槽、观察细致
陈科：自负、喜欢装体面
```

这种人物语言标签只能控制表面措辞，不能控制具体对白。

真正需要控制的是：

```text
这个人物此刻为什么要说这句话？
他说完以后，希望对方发生什么变化？
他为什么不用更直接的方式表达？
他是在解释、遮掩、试探、敷衍、反击，还是转移话题？
```

所以人物语言控制必须同时包含：

```text
人物稳定声音
+
当前语言目的
+
当前认知范围
+
当前情绪状态
+
当前表达策略
+
不能说出的内容
```

缺一不可。

---

## 第二优先级：人物行为意图

行为意图控制人物：

- 当前想完成什么；
    
- 为什么采取这个动作；
    
- 这个动作是主动选择还是被迫反应；
    
- 行为依据来自哪里；
    
- 行为会如何改变后续局势；
    
- 是否符合人物此前的认知和性格；
    
- 行为失败后人物会怎样调整。
    

人物行为不是“走过去、坐下来、拿起电话”这种动作描写。

真正的行为意图是：

```text
人物为什么走过去？
为什么现在打断？
为什么没有解释？
为什么把身份证交出去？
为什么选择留下而不是离开？
为什么在这一刻撒谎？
```

人物行为决定情节方向。

情节并不是大纲规定发生什么，然后人物配合完成；而应当是：

```text
人物拥有目标
→ 根据自身认知作出选择
→ 选择产生后果
→ 后果形成下一步剧情
```

如果行为意图不成立，剧情就会呈现明显的作者操控感。

---

## 第三优先级：场景与叙事意图

这一层才包括：

- 信息差；
    
- POV；
    
- 节奏；
    
- 伏笔；
    
- 喜剧机制；
    
- 读者领先程度；
    
- 详略；
    
- 场景功能；
    
- 章节钩子。
    

这些仍然重要，但它们主要服务于人物语言和行为。

例如“本场要维持误会”，不能直接控制人物对白，而应转换为：

```text
陈艾艾为什么不解释？
因为她认为马木已经说过。

马雯雯为什么不追问真实情况？
因为她认为自己已经通过细节判断出来。

陈科为什么不断强调项目可靠？
因为他需要证明自己的判断没有错。
```

只有转化成人物自己的语言目的和行为动机，误会才会自然持续。

否则只是作者为了保持误会，强行不让人物把话说清楚。

# 二、人物语言意图才是小说正文的最小控制单位

场景并不是正文生成中最小的意图单位。

对于以人物对话推进为主的小说，真正的最小控制单位应该是：

> **一轮有效语言行为。**

一轮有效语言行为不是一句台词，而是：

```text
人物受到刺激
→ 形成语言目的
→ 选择表达策略
→ 说出具体内容
→ 对方按照自己的认知理解
→ 产生反应
```

例如：

```text
陈艾艾：“马木哥应该已经和你说过了吧？”
```

这句台词不能只检查“是否自然”。

必须检查：

### 陈艾艾的语言意图

她想确认马雯雯是否已经知道诈骗风险。

### 陈艾艾的认知

她认为马木已经把情况说清楚。

### 陈艾艾的表达策略

不重新解释，只进行确认。

### 陈艾艾的语言边界

不能当众直接说出诈骗判断。

### 马雯雯的理解

她认为陈艾艾在确认马木是否介绍过这里的灰色业务。

### 对话结果

两人都认为沟通已经完成，误会继续。

这才是一轮完整的语言意图控制。

如果只规定：

```text
这里保持鸡同鸭讲。
```

模型很容易生成一组机械错位的对白。

如果明确双方每句话各自想达到什么目的，对话才能自然成立。

# 三、人物语言控制要区分六个维度

建议将每个主要人物的语言控制拆成六层。

## 1. 稳定语言人格

长期不轻易变化的表达特点：

- 词汇范围；
    
- 句子长短；
    
- 说话直接还是绕弯；
    
- 是否喜欢解释；
    
- 是否擅长比喻；
    
- 是否容易说大词；
    
- 是否通过吐槽表达情绪；
    
- 是否愿意示弱；
    
- 面对不同关系时的语言变化。
    

这是“这个人平时怎么说话”。

---

## 2. 当前语言目标

人物本轮开口想实现什么：

- 获得信息；
    
- 隐瞒信息；
    
- 说服对方；
    
- 稳住对方；
    
- 转移问题；
    
- 试探态度；
    
- 维护尊严；
    
- 争夺主动权；
    
- 结束话题；
    
- 逼迫对方表态。
    

这是“他为什么现在说”。

---

## 3. 当前认知边界

人物此刻：

- 知道什么；
    
- 不知道什么；
    
- 误以为什么；
    
- 怀疑什么；
    
- 对哪些内容过度自信；
    
- 哪些判断只有模糊感觉。
    

这是“他说话依据什么”。

---

## 4. 当前情绪压力

相同目标在不同情绪下表达完全不同：

- 心虚；
    
- 愤怒；
    
- 尴尬；
    
- 得意；
    
- 防备；
    
- 焦急；
    
- 敷衍；
    
- 想表现轻松；
    
- 表面平静但内部警觉。
    

这是“他说话时处于什么状态”。

---

## 5. 表达策略

人物为了达到目的所使用的方法：

- 直接询问；
    
- 假装随口一问；
    
- 先开玩笑再试探；
    
- 故意说一半；
    
- 借别人的话表达；
    
- 反问；
    
- 装作已经知道；
    
- 用专业词压制对方；
    
- 通过否认进行承认；
    
- 通过转移话题逃避回答。
    

这是“他选择怎么说”。

---

## 6. 语言禁区

人物此刻绝对不能说出的内容：

- 自己不知道的信息；
    
- 不符合身份的判断；
    
- 会提前暴露真相的词；
    
- 不符合人物关系的直白表达；
    
- 只为方便读者而出现的解释；
    
- 作者总结式台词；
    
- 与人物语言习惯不符的大词。
    

这是“他不能怎么说”。

# 四、行为意图要形成完整因果链

行为控制同样不能停留在动作层面。

每个关键行为都应具备以下结构：

```text
外部刺激
→ 人物认知
→ 人物目标
→ 可选行动
→ 人物选择
→ 选择理由
→ 行动结果
→ 状态变化
```

例如马雯雯把身份证送到云水汇。

不能只写：

```text
陈艾艾打电话要身份证
→ 马雯雯送过去
```

必须形成：

```text
外部刺激：
陌生女人打电话，准确知道马木刚离开。

马雯雯认知：
她先怀疑对方是警察，后来认为对方与马木所在场所有关。

人物目标：
弄清马木到底又惹了什么麻烦，同时把身份证送回去。

可选行动：
让马木自己回来拿；
让对方派人来取；
自己送过去。

人物选择：
亲自送过去。

选择理由：
电话里的信息让她不放心，同时地点不远，身份证又被说成急用。

行动结果：
她进入云水汇，进入下一层误会。
```

只有这样，人物才是推动剧情的人，而不是剧情搬运工具。

# 五、动态对齐应围绕“语言分歧”和“行为分歧”触发

之前的L0—L3分歧分类仍然可以保留，但必须调整中断优先级。

## 一级强制中断：人物语言意图分歧

出现以下情况时必须中断：

- 不清楚人物为什么开口；
    
- 同一句台词可以服务于两个不同目标；
    
- 不清楚人物是试探、解释、遮掩还是反击；
    
- 不清楚人物想让对方知道还是误解；
    
- 不清楚人物是否会直接说出某件事；
    
- 不清楚人物在此关系下会采用什么态度；
    
- 两种语言方向会塑造出不同的人物形象；
    
- 台词虽然能推动剧情，但可能违背人物。
    

例如：

```text
陈科此时继续介绍项目，究竟是：

A. 真心想把马雯雯也拉进来；

B. 察觉马雯雯怀疑自己，因此急于证明自己没有被骗。
```

这两个方向会产生完全不同的语言状态，必须对齐。

---

## 二级强制中断：人物行为意图分歧

出现以下情况时必须中断：

- 不清楚人物当前目标；
    
- 同一人物存在两个同样合理的行动选择；
    
- 当前行为只服务剧情，不符合人物；
    
- 行为会显著改变情节走向；
    
- 人物主动还是被动不明确；
    
- 人物为什么不采取更简单的行动无法解释；
    
- 行为会改变人物关系；
    
- 行为意味着人物态度发生转变。
    

例如：

```text
马雯雯发现现场不对后为什么还不离开？

A. 因为她要等马木出来；

B. 因为她认为身份证已经进入登记流程，必须拿回来；

C. 因为她开始担心马木涉及违法活动。
```

不同动机会决定后续整场行为，必须对齐。

---

## 三级可选中断：场景叙事意图分歧

包括：

- 节奏；
    
- 详略；
    
- 伏笔明显程度；
    
- 喜剧强弱；
    
- 环境描写比例；
    
- 场景结束位置。
    

只有这些内容反过来影响人物语言和行为时，才需要询问作者。

否则模型应自行处理。

# 六、场景检查点必须改为“人物驱动检查”

之前的场景检查表偏向叙事控制，应改为以下顺序。

## 第一层：行为检查

```text
本场每个主要人物想得到什么？
人物为什么进入或留在本场？
人物准备采取什么行动？
行动依据是什么？
如果没有阻碍，人物最直接的做法是什么？
为什么人物没有采用更简单的做法？
本场结束后，谁的目标实现、失败或发生变化？
```

## 第二层：语言检查

```text
人物为什么要与对方说话？
每一轮关键对白的语言目的是什么？
人物希望对方作出什么反应？
人物说出口的内容与真实意图是否一致？
人物省略了什么？
人物隐瞒了什么？
人物误以为对方已经知道什么？
人物会采用何种表达策略？
哪些话绝不能由这个人物说出？
```

## 第三层：认知检查

```text
人物知道什么？
不知道什么？
误解什么？
怀疑什么？
确定到什么程度？
```

## 第四层：叙事检查

```text
读者应获得什么？
哪些信息不能释放？
场景应偏喜剧还是偏暗流？
伏笔如何出现？
详略如何分配？
```

顺序不能颠倒。

因为叙事目的必须通过人物的语言和行为实现，而不是人物服从叙事安排。

# 七、正文生成前应形成“人物意图轨道”

每个场景不能只有一个笼统的场景意图单，还应生成轻量的人物意图轨道。

例如：

```yaml
character_intent_tracks:
  马雯雯:
    behavior_goal: 找到马木并确认他是否惹事
    immediate_action: 先观察现场，再决定是否追问
    language_goal: 从陈艾艾的话中套出马木在这里做什么
    language_strategy: 假装自己已经知道一部分，只问细节
    emotional_state: 警惕，但不愿表现得太明显
    knowledge:
      knows:
        - 马木来过
        - 身份证在这里要用
      suspects:
        - 现场可能有灰色业务
      does_not_know:
        - 陈科遭遇诈骗
    forbidden:
      - 直接说出诈骗
      - 表现得已经确定现场违法

  陈艾艾:
    behavior_goal: 尽快拿到身份证并继续处理陈科的事
    immediate_action: 接过身份证，引导马雯雯不要干扰现场
    language_goal: 确认马木是否已经向马雯雯说明情况
    language_strategy: 省略背景，只核对结果
    emotional_state: 着急但必须保持平静
    knowledge:
      knows:
        - 马木发现了陈科的问题
        - 马雯雯是来送身份证的
      assumes:
        - 马雯雯已经知道诈骗风险
    forbidden:
      - 当场完整解释诈骗
      - 把马雯雯当成完全不知情的人重新介绍
```

这份人物意图轨道比笼统的“双方鸡同鸭讲”更有控制力。

# 八、对白生成不能一次只看“台词好不好”

对白至少要通过三次检查。

## 第一遍：意图正确性

检查：

```text
每句话是否服务人物当前目标？
人物是否有理由说这句话？
人物是否说出了不该知道的内容？
人物是否为了向读者解释而开口？
```

## 第二遍：人物声音

检查：

```text
去掉姓名后，能否判断是谁说的？
句式、词汇、态度是否符合人物？
人物是否突然变得过度聪明、幽默或完整？
不同人物是否共享同一种LLM语言？
```

## 第三遍：对白作用结果

检查：

```text
对方为什么这样理解？
这句话是否改变了关系、认知或行动？
如果删除这句话，场景是否没有变化？
是否存在没有语言行为功能的空对白？
```

只有三层都成立，对话才算有效。

# 九、强笑点也必须服从人物语言意图

这点尤其重要。

不能为了制造笑点，让人物临时变成段子手。

强笑点必须来自人物按照自己的目标认真行动，却产生预期错位。

因此，笑点生成顺序应当是：

```text
人物语言目的
→ 人物真实表达
→ 对方错误理解或预期被打破
→ 形成笑点
```

而不是：

```text
需要一个笑点
→ 编一句有梗的话
→ 强行塞给人物
```

例如马木的台词是否搞笑，不应首先问“这句话有没有梗”，而应先问：

```text
马木此时想干什么？
他为什么要这样解释？
他对自己的判断有多自信？
他的表达怎样自然暴露这种自信？
读者的预期在哪里被打破？
```

这样产生的笑点才属于人物，而不是属于作者或模型。

# 十、意图控制系统的核心应重新定义

原来的定义偏向：

> 通过风险触发式中断，对齐章节和场景意图。

现在应升级为：

> **以人物语言意图和人物行为意图为核心，通过场景检查点识别人物驱动层面的分歧，并在关键分歧出现时中断生成，确保剧情由人物的表达与选择自然产生。**

整个控制结构应变成：

```text
人物长期设定
    ↓
人物当前认知
    ↓
人物当前行为目标
    ↓
人物当前语言目标
    ↓
人物选择表达与行动策略
    ↓
人物语言和行为
    ↓
对方反应及现实后果
    ↓
剧情推进
    ↓
读者体验、喜剧、伏笔、节奏
```

而不是：

```text
章节需要发生某件事
    ↓
安排人物说话和行动
```

# 十一、第一版需求优先级应重新排序

## P0：必须实现

1. 人物当前行为目标识别；
    
2. 人物关键行为动机检查；
    
3. 人物当前语言目标识别；
    
4. 关键对白的表达策略控制；
    
5. 人物认知边界控制；
    
6. 人物语言禁区控制；
    
7. 语言意图分歧强制中断；
    
8. 行为意图分歧强制中断；
    
9. 作者确认后的意图轨道更新；
    
10. 生成后人物语言和行为一致性检查。
    

## P1：重要功能

1. 场景信息差；
    
2. 读者领先程度；
    
3. 误会机制；
    
4. POV；
    
5. 喜剧机制；
    
6. 伏笔；
    
7. 场景主动权；
    
8. 场景节奏。
    

## P2：后续优化

1. 环境描写；
    
2. 句子润色；
    
3. 语言密度；
    
4. 段落长短；
    
5. 一般过渡；
    
6. token压缩。
    

也就是说：

> 如果人物语言和人物行为没有对齐，节奏、伏笔和信息差即使控制正确，正文依然不能成立。

# 十二、对前一版需求设计的修正结论

前一版不是完全错误，而是控制中心放错了。

应保留：

- 整章交付；
    
- 场景检查点；
    
- 条件中断；
    
- 分歧分级；
    
- 最小示例；
    
- 状态恢复；
    
- 整章一致性检查。
    

但场景检查点的核心内容必须从：

```text
场景功能、信息差、伏笔、节奏
```

调整为：

```text
人物行为目标
→ 人物行为选择
→ 人物语言目标
→ 人物表达策略
→ 认知边界
→ 语言和行为产生的后果
```

信息差、误会、喜剧、节奏和伏笔作为人物意图的外层约束。

这会使意图控制系统从“控制作者想写什么”，进一步变成：

> **控制人物为什么这样说、为什么这样做，以及人物的语言和行为如何自然生成剧情。**

这才是小说意图控制的核心。
