# 文档对齐检查报告

## 一、检查结论

当前文档已经可以支持正式启动意图控制系统开发。

启动方式：

```text
先创建独立人物意图控制 Skill，
再改造章节执行包、章节细纲和正文生成 Skill。
```

不建议继续只在正文生成 Skill 内部追加规则。

---

# 二、已对齐的核心口径

## 2.1 系统中心

已统一为：

```text
人物语言意图 + 人物行为意图
```

不再以“场景控制”作为系统中心。

场景只作为结构单位，关键片段作为实际输出单位。

## 2.2 控制层级

已统一为：

```text
交付单位：整章
结构单位：场景
控制单位：人物意图
落地单位：对话与行为
输出单位：关键片段
验证单位：关键对白 / 关键动作 / 最小片段
```

## 2.3 交互方式

已统一为：

```text
第一版全部显式列人物意图轨道。
高风险显式 / 低风险内部检查只作为后续预留。
```

## 2.4 状态保存

已统一为：

```text
每章一份章节运行态。
动态对齐在对话框完成。
确认结果写入章节运行态。
正文文档保持干净。
```

## 2.5 作者权限

已统一为：

```text
作者可以直接改写人物意图轨道。
系统必须将作者修改归一化为规则。
```

## 2.6 中断规则

已统一为：

```text
只要不清楚人物为什么说或为什么做，即使场景目标明确，也必须 ASK_USER。
```

## 2.7 人物卡边界

已统一为：

```text
人物卡保存长期世界内人物事实。
章节运行态保存当前章临时人物意图。
临时人物意图不得自动回写人物卡。
```

# 四、对大纲 / 细纲接入的判断

大纲和细纲需要接入意图控制，但不承担正文级动态中断。

正确边界：

```text
单元大纲 / 章节大纲：宏观人物驱动力和认知推进
章节细纲：人物进入场景的基础认知、目标、关系压力、风险
章节执行包：关键对白风险、关键行为风险、禁止替人物决定项
意图控制 Skill：人物意图轨道、CONTINUE / ASK_USER / RESUME
正文生成 Skill：正文片段生成与整章组装
```

因此：

- 需要改造章节细纲；
- 需要改造章节执行包；
- 单元大纲和章节大纲先作为 P2 后续评估；
- 不应在大纲阶段写死片段级人物意图轨道。

---

# 五、对独立 Skill 的判断

需要独立 Skill。

理由：

1. 意图控制横跨正文生成前、生成中和恢复阶段；
2. 它需要维护 CONTINUE / ASK_USER / RESUME 协议；
3. 它需要更新章节运行态；
4. 它不直接写正文；
5. 如果并入正文生成 Skill，会导致职责过重。

建议 Skill：

```text
.codex-tmp/skills/control-character-intent/
```
