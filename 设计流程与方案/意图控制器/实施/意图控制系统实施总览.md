# 意图控制系统实施总览

## 一、当前结论

意图控制系统需要正式启动独立开发。

当前系统已经完成方法论收敛和第一版需求设计，但尚未进入真正的 Skill 本体开发。

最终架构不是把意图控制简单塞进正文生成 Skill，而是形成一个独立控制层：

```text
资料调用 / 上游叙事资料
    ↓
单元大纲 / 章节大纲 / 章节细纲
    ↓
章节执行包
    ↓
独立人物意图控制 Skill
    ↓
正文生成 Skill
    ↓
章节运行态 / 正文 V1 / 生成备注
```

其中，意图控制系统的核心不是“控制场景”，而是：

```text
控制人物为什么说
控制人物为什么做
控制语言和行为如何自然生成剧情
```

---

# 二、系统目标

第一版目标：

```text
在不开发 GUI、不引入复杂 RAG、不自动回写人物卡的前提下，
通过独立人物意图控制 Skill + 章节运行态 + 上游接口字段，
显著提高正文 V1 的方向正确率和可保留正文比例。
```

核心验收标准：

1. 人物语言意图不明时能中断；
2. 人物行为意图不明时能中断；
3. 作者可以直接改写人物意图轨道；
4. 作者确认结果能进入每章一份章节运行态；
5. 正文生成能从已确认轨道继续；
6. 上游细纲和执行包能为意图控制提供原材料；
7. V1 不再默认依赖整章重写解决方向错误。

已完成：

- `思路/人物意图控制.md`

已确认结论：

- 人物才是意图主体；
- 语言意图和行为意图是控制核心；

- 行为意图无法控制，就无法让剧情有逻辑。


本阶段只做可执行 Skill，不直接生成正文。

## 阶段 2：改造章节执行包 Skill

目标：

```text
让执行包为意图控制器提供原材料。
```

新增字段：

- 人物关系压力；
- 人物认知边界；
- 本场可能误解点；
- 关键行为风险；
- 关键对白风险；
- 禁止替人物决定项；
- 意图控制器建议检查片段。

执行包不生成完整人物意图轨道。

## 阶段 3：改造章节细纲 Skill

目标：

```text
让细纲预埋人物意图风险，而不是只写场景功能。
```

新增字段：

- 每场涉及的关键人物；
- 人物进入本场的基础认知；
- 人物进入本场的外在目标；
- 人物关系压力；
- 行为动机缺口；
- 语言意图风险；
- 下游意图控制重点。

细纲不写完整台词意图。

## 阶段 4：改造正文生成 Skill

目标：

```text
正文生成 Skill 从“整章一次生成”改为“整章交付、关键片段连续生成”。
```

必须支持：

- 读取章节运行态；
- 调用或执行人物意图轨道检查；
- 接收 CONTINUE / ASK_USER / RESUME；
- 按关键片段生成正文；
- 最后组装整章 V1。

## 阶段 5：完整链路测试

测试链路：

```text
章节细纲
    ↓
章节执行包
    ↓
人物意图控制 Skill
    ↓
正文生成 Skill
    ↓
章节运行态
    ↓
整章 V1
```

建议先用已经反复修过的章节测试。

优先测试第三章或已有问题总结较充分的章节。

4. 作者交互方式已经明确；
5. 上游接入边界已经明确；
6. 目前缺的是可执行 Skill 结构和测试样例。

不建议直接改正文生成 Skill 的原因：

```text

正文生成 Skill 会同时承担轨道生成、中断判断、状态恢复和正文写作，
职责过重，后续很难定位问题。
```

# 九、风险提醒

## 9.1 过度显式化风险

第一版全部显式列轨道，有助于验证系统，但会增加交互和 token。

后续需要逐步转为：

```text
高风险片段显式
低风险片段内部检查
```

## 9.2 上游写死风险

细纲和执行包只能提供人物意图原材料，不应提前写死每句话或每个动作的完整意图。

否则正文会失去自然生长空间。

## 9.3 人物卡污染风险

章节运行态中的临时人物意图不能自动写回人物卡。

只有经过作者确认、具有长期稳定价值的内容，才进入长期规则候选。

## 9.4 职责混乱风险

独立意图控制 Skill 不写正文。

正文生成 Skill 不应自行替代意图控制器。

执行包不应生成完整人物意图轨道。
