# 章节文风模式字段设计

## 一、设计目的

章节大纲必须显式标注本章文风模式。

正文生成 skill 不应默认每章都使用同一种幽默强度，也不应机械规定“多少字一个笑点”。它应先读取章节大纲中的文风模式，再结合剧情功能、角色卡语言规则和执行包，决定本章的幽默密度、正剧情绪权重、笑点释放方式和禁止事项。

核心原则：

> 小说可以幽默打底，但每章必须知道自己是在搞笑、轻喜、冷讽、正剧，还是信息推进。  
> 幽默应从剧情机会中自然释放，不按字数或场景数量机械打点。

---

## 二、章节大纲新增字段

建议在章节大纲中新增固定章节：

```markdown
## 章节文风模式

|项目|内容|
|---|---|
|文风模式||
|幽默密度||
|正剧情绪权重||
|幽默来源||
|本章允许的幽默方式||
|本章禁止的幽默方式||
|是否允许小型脱口秀||
|是否允许冷幽默||
|是否允许网络梗/流行梗||
|结尾情绪落点||
|备注||
```

### 字段说明

|字段|填写要求|
|---|---|
|文风模式|从“开篇脱口秀型 / 轻喜误会型 / 冷幽默反讽型 / 正剧煽情型 / 信息推进型”中选择一个主模式；必要时可写“主模式 + 辅助模式”。|
|幽默密度|高 / 中 / 低 / 极低。不是机械笑点数量，而是本章整体幽默占比。|
|正剧情绪权重|低 / 中 / 高 / 极高。用于防止幽默冲淡关键情绪。|
|幽默来源|写本章幽默从哪里自然产生，如职场荒诞、家庭饭局、人物错位、会所误会、证据反讽、角色嘴硬、关系尴尬。|
|本章允许的幽默方式|写可用方式，如生活化吐槽、人物找补、误会双关、冷反讽、饭桌沉默。|
|本章禁止的幽默方式|写本章不能出现的方式，如小型脱口秀、密集段子、屎尿屁、破坏煽情的抖机灵、解释梗。|
|是否允许小型脱口秀|是 / 否。原则上只用于开篇1-3章或新卷开头。|
|是否允许冷幽默|是 / 否。危机、证据链反杀、正剧转折章可允许。|
|是否允许网络梗/流行梗|默认允许，但不得解释成注释，不得打断剧情；可在生成备注标记给作者校正。|
|结尾情绪落点|写本章最后留给读者的情绪，如好笑、荒唐、轻微不安、冷下来、心疼、窒息、反讽。|
|备注|记录特殊要求，例如“角色可以笑，但读者应感到难受”。|

## 三、文风模式定义

### 1. 开篇脱口秀型

适用范围：

- 第1-3章；
- 新卷开头；
- 需要快速建立主视角、现实压力、读者好感的章节。

特征：

- 段子密度高；
- 可以用小型脱口秀式开场；
- 用现实痛点快速抓读者；
- 幽默必须围绕统一主题，不能散成段子合集。

适合的幽默来源：

- 职场荒诞；
- 甲方/领导话术；
- 家庭饭局；
- 代际误解；
- 表面礼貌与内心崩溃。

禁止：

- 与剧情无关的社会段子；
- 解释梗；
- 低俗化开场；
- 让角色像段子手而不是小说人物。

示例填写：

```markdown
|文风模式|开篇脱口秀型|
|幽默密度|高|
|正剧情绪权重|低|
|幽默来源|职场荒诞、甲方话术、家庭比较、马雯雯表面礼貌与内心吐槽|
|本章允许的幽默方式|小型脱口秀式开场、生活化夸张、预期打破、人物反应显影|
|本章禁止的幽默方式|离题段子、解释网络梗、屎尿屁、脱离人物口吻的作者贫嘴|
|是否允许小型脱口秀|是|
|是否允许冷幽默|否|
|是否允许网络梗/流行梗|允许，生成备注标记可疑梗|
|结尾情绪落点|好笑、荒唐、误会成立|
```

---

### 2. 轻喜误会型

适用范围：

- 日常推进；
- 误会升级；
- 人物关系错位；
- 前期喜剧主线章节。

特征：

- 幽默打底；
- 笑点随剧情自然释放；
- 不做密集脱口秀；
- 喜剧来自误会、关系尴尬、人物装腔失败、信息错位。

适合的幽默来源：

- 会所误会；
- 人物错位；
- 鸡同鸭讲；
- 马木装腔失败；
- 陈科嘴硬体面；
- 亲戚安全话术。

禁止：

- 机械塞笑点；
- 把人物写成只会贫嘴；
- 为搞笑牺牲事件推进；
- 用旁白解释“这里为什么好笑”。

示例填写：

```markdown
|文风模式|轻喜误会型|
|幽默密度|中|
|正剧情绪权重|中低|
|幽默来源|会所误会、人物错位、陈科体面嘴硬、马木土法判断|
|本章允许的幽默方式|误会双关、人物找补、关系尴尬、饭桌/门口反应|
|本章禁止的幽默方式|小型脱口秀、硬插网络段子、过度解释误会|
|是否允许小型脱口秀|否|
|是否允许冷幽默|否|
|是否允许网络梗/流行梗|允许，不能打断剧情|
|结尾情绪落点|误会升级，读者想继续看热闹|
```

---

### 3. 冷幽默反讽型

适用范围：

- 喜剧逐渐转冷；
- 证据链开始显影；
- 危机前夜；
- 读者逐渐意识到前面笑点可能会反杀。

特征：

- 表层仍可有幽默；
- 读者感受开始不安；
- 笑点带反讽；
- 角色可能还在按喜剧逻辑说话，但事实已经变危险。

适合的幽默来源：

- 证据反讽；
- 角色嘴硬；
- 误会词汇翻面；
- “好心帮忙”变成“参与痕迹”；
- 正常话术突然显得不对劲。

禁止：

- 过度热闹；
- 密集段子；
- 冲淡风险；
- 让读者只觉得好笑、不觉得不安。

示例填写：

```markdown
|文风模式|冷幽默反讽型|
|幽默密度|低到中|
|正剧情绪权重|中高|
|幽默来源|证据反讽、角色嘴硬、误会词翻面、帮忙痕迹累积|
|本章允许的幽默方式|冷幽默、轻微反讽、角色嘴硬、读者后知后觉的不安|
|本章禁止的幽默方式|小型脱口秀、密集包袱、破坏危机感的贫嘴|
|是否允许小型脱口秀|否|
|是否允许冷幽默|是|
|是否允许网络梗/流行梗|允许，但应谨慎，生成备注标记|
|结尾情绪落点|笑意变冷，读者意识到事情不只是荒唐|
```

---

### 4. 正剧煽情型

适用范围：

- 情绪崩盘；
- 失去、伤害、关系破裂；
- 关键人物创伤显影；
- 单元情绪落点。

特征：

- 幽默极少或没有；
- 如果出现幽默，应是角色自我保护、嘴硬或反讽；
- 可以追求“角色在笑，读者在难受”的效果；
- 不用段子破坏情绪。

适合的幽默来源：

- 嘴硬；
- 旧习惯；
- 反讽；
- 角色不愿承认痛苦时的轻描淡写。

禁止：

- 抖机灵；
- 网络梗；
- 密集吐槽；
- 为保持轻松而削弱痛感。

示例填写：

```markdown
|文风模式|正剧煽情型|
|幽默密度|极低|
|正剧情绪权重|极高|
|幽默来源|角色嘴硬、自我保护式轻描淡写、冷反讽|
|本章允许的幽默方式|角色在痛处上的习惯性嘴硬；让读者感到更难受的轻笑|
|本章禁止的幽默方式|小型脱口秀、密集吐槽、网络热梗、破坏情绪的包袱|
|是否允许小型脱口秀|否|
|是否允许冷幽默|视情况允许|
|是否允许网络梗/流行梗|原则上不建议，若使用必须服务情绪反讽|
|结尾情绪落点|心疼、沉下去、回看前文笑点变痛|
```

---

### 5. 信息推进型

适用范围：

- 调查；
- 复盘；
- 解释必要信息；
- 多线汇总；
- 过渡章节。

特征：

- 信息必须通过冲突、行动、关系压力给出；
- 不能写成说明书；
- 幽默可少量存在，用于降低信息密度；
- 对话仍要由情绪和目的推动。

适合的幽默来源：

- 信息不对称；
- 追问与逃避；
- 角色不愿承认；
- 正式话术与生活理解错位。

禁止：

- 人物配合问答；
- 长篇解释；
- 作者代替人物总结；
- 为解释设定牺牲现场感。

示例填写：

```markdown
|文风模式|信息推进型|
|幽默密度|低|
|正剧情绪权重|中|
|幽默来源|追问与逃避、正式话术生活化、角色不愿承认|
|本章允许的幽默方式|轻微反讽、关系拉扯、信息错位|
|本章禁止的幽默方式|密集段子、脱口秀开场、人物配合解释设定|
|是否允许小型脱口秀|否|
|是否允许冷幽默|可少量允许|
|是否允许网络梗/流行梗|允许但不建议主动使用|
|结尾情绪落点|信息更清楚，但压力更重|
```

## 五、skill读取规则

### 1. 章节细纲 skill

应读取章节大纲中的 `章节文风模式`，并在细纲中继承，不得擅自改变。

如果原始章节大纲未标注文风模式，细纲 skill 应：

1. 根据章节功能做临时推断；
2. 在 `待确认问题` 中标记；
3. 不把推断当作 confirmed 事实。

细纲中建议新增场景级字段：

```markdown
|场景|文风执行|幽默来源|正剧情绪|禁止写法|
|---|---|---|---|
```

### 2. 章节执行包 skill

应把文风模式转成正文可执行策略，而不是机械分配笑点数量。

建议新增：

```markdown
## 喜剧/正剧执行策略

|场景|文风模式|剧情中的喜剧机会|自然释放方式|冷幽默/反讽方式|不能冲淡的情绪|
|---|---|---|---|---|---|
```

### 3. 正文生成 skill

应读取：

- 章节文风模式；
- 执行包喜剧/正剧执行策略；
- 角色卡语言与幽默方式；
- 作者风格规则；
- 样板句库与禁用句库。

正文生成时必须遵守：

1. 不按字数机械塞笑点；
2. 不在正剧煽情章强行保持轻松；
3. 不用小型脱口秀处理非开篇章节；
4. 不让人物为了讲清楚而配合问答；
5. 梗可以使用，但不能解释成注释；
6. 可在生成备注中标记疑似硬梗、疑似AI句、幽默冲淡正剧情绪风险。

生成备注建议新增：

```markdown
|项目|内容|
|---|---|
|文风模式执行情况||
|疑似AI感句子||
|疑似硬梗||
|角色语言越界||
|幽默冲淡正剧情绪风险||
|需要作者判断的梗||
```

## 六、后续待办

1. 更新章节大纲模板，加入 `章节文风模式`。
2. 更新角色卡模板，加入 `语言与幽默方式`。
3. 修改 `generate-chapter-detail-outline`，继承并拆分文风模式。
4. 修改 `generate-chapter-execution-pack`，将文风模式转成喜剧/正剧执行策略。
5. 修改 `generate-chapter-prose-draft`，读取文风模式与角色语言规则，并输出风险备注。
6. 建立长期维护的样板句库与AI感禁用句库。
