# 角色卡语言与幽默字段设计

## 一、设计目的

正文生成 skill 不应在自身规则中硬编码某个角色如何吐槽、如何说话、如何搞笑。

角色的语言习惯、幽默来源、关系反应和表达边界，应记录在人物卡中。skill 只负责读取这些世界内人物事实，并在正文生成时执行。

核心原则：

> 人物卡记录“这个人真实会怎么说、怎么反应、怎么掩饰、怎么找补”。  
> skill 不替角色设定语言性格，只调用角色卡中的语言事实。

---

## 二、与人物卡原则的关系

新增字段必须遵守人物卡世界内原则：

1. 只记录人物在小说世界中的稳定语言习惯、关系反应和行为边界。
2. 不写成“写作注意”“写作禁忌”“创作目的”。
3. 不按章节、单元、剧情阶段组织人物语言状态。
4. 如果语言习惯随时间发生变化，应放入人物时间轴中的明确世界内时间节点。
5. “不会使用的表达”应写成人物真实认知、身份、教育背景、关系位置或心理限制造成的语言边界，而不是作者命令。

---

## 三、人物卡新增章节

建议在人物卡 `## 语言习惯` 后新增：

```markdown
## 语言与幽默方式

|项目|内容|
|---|---|
|基础说话方式||
|情绪上来时的说话方式||
|幽默来源||
|吐槽方式||
|常用反应||
|解释信息时的习惯||
|被误解时的反应||
|装腔或失误后的反应||
|与亲近者说话方式||
|与陌生人说话方式||
|与权威/机构说话方式||
|与弱势/需要保护对象说话方式||
|不会使用的表达||
|语言边界形成原因||
```

### 字段说明

|字段|填写要求|
|---|---|
|基础说话方式|人物平常说话的自然形态，如直接、客气、含混、嘴硬、慢热、攻击性强。|
|情绪上来时的说话方式|急、怕、心虚、丢脸、愤怒、委屈时语言如何变形。|
|幽默来源|这个人的幽默从哪里来，如现实经验、身份错位、装腔失败、嘴硬、自嘲、代际误解。|
|吐槽方式|人物如何表达不满、嫌弃、看穿或不信任。|
|常用反应|被问、被催、被误会、被戳痛处时常见的第一反应。|
|解释信息时的习惯|人物在不得不解释时，是粗糙判断、讲流程、举例子、转移话题，还是拒绝解释。|
|被误解时的反应|人物会急着澄清、嘴硬、摆烂、反问、找补，还是先处理事情。|
|装腔或失误后的反应|人物说过头、装过头或露怯后如何补救。|
|与亲近者说话方式|与亲属、熟人、旧友等关系中的省略、顶嘴、损人、照顾方式。|
|与陌生人说话方式|与普通陌生人、服务人员、路人等关系中的语言距离。|
|与权威/机构说话方式|面对警察、银行、公司、医院、学校、政府窗口等正式对象时的语言策略。|
|与弱势/需要保护对象说话方式|面对孩子、老人、受害者、情绪崩溃者时的语言软硬度。|
|不会使用的表达|人物基于身份、认知、关系和语言习惯，通常不会说出的表达类型。|
|语言边界形成原因|解释上述边界来自人物真实经历、社会位置、教育背景、认知限制或心理防御。|

## 四、可选补充字段

当角色承担较强喜剧功能时，可增加：

```markdown
## 喜剧表现结构

|项目|内容|
|---|---|
|喜剧功能来源||
|常见错位||
|好心变添乱的方式||
|嘴硬与真实行动的关系||
|笑点失效边界||
```

字段说明：

|字段|填写要求|
|---|---|
|喜剧功能来源|这个角色为什么会产生喜剧效果，必须来自人物真实状态，而非作者强行安排。|
|常见错位|人物自我认知与他人眼中形象、语言与行动、身份与场景之间的错位。|
|好心变添乱的方式|人物明明想帮忙，却因为表达、身份、关系或方法造成误解的具体方式。|
|嘴硬与真实行动的关系|人物嘴上如何掩饰真实关心，行动上如何暴露真实倾向。|
|笑点失效边界|在哪些情境中这个角色的幽默会停止或变冷，如真正伤害、正式调查、关系破裂。|

## 五、正文生成 skill 读取规则

正文生成 skill 应读取人物卡中的 `语言习惯`、`语言与幽默方式` 和必要时的 `喜剧表现结构`。

生成时应遵守：

1. 不让所有角色共享同一种吐槽腔。
2. 不让角色为了向读者解释信息而变得过度配合。
3. 人物对白优先由情绪、目的、关系驱动，逻辑只作为底层支撑。
4. 解释必须被追问、误解、压力、利益或关系逼出来。
5. 角色的幽默来自自身处境和语言习惯，不来自作者临时塞梗。
6. 当人物卡未提供语言字段时，正文生成 skill 应在生成备注中标记缺口，而不是自行创造稳定语言设定。

生成备注建议增加：

```markdown
|项目|内容|
|---|---|
|角色语言字段缺口||
|角色语言越界风险||
|人物同质化风险||
|解释过度风险||
|疑似作者替人物说话||
```

## 六、角色卡模板更新建议

人物卡模板应保留原 `## 语言习惯`，并新增 `## 语言与幽默方式`。

两者分工：

|章节|功能|
|---|---|
|语言习惯|记录最基础的表达风格、语气、用词、沉默方式和认知语言边界。|
|语言与幽默方式|记录不同情绪、关系、误解和喜剧情境下的具体表达模式。|

对于承担喜剧功能较强的角色，可再新增 `## 喜剧表现结构`。

---

## 七、长期资产

角色语言字段应与以下长期资产配合维护：

|资产|用途|
|---|---|
|作者样板句库|记录作者认可的自然表达、幽默落点和对话节奏。|
|AI感禁用句库|记录明显AI感句式、解释型句式、过度漂亮比喻。|
|角色对白样例库|记录每个主要角色的代表性对白。|
|喜剧桥段样例库|记录已验证有效的铺垫、升级、落点结构。|

这些资产应长期积累，不要求一次完成。
