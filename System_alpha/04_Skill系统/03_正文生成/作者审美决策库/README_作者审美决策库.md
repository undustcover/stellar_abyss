# 作者审美决策库运行说明

## 一、目录定位

本目录是作者审美判断的正式系统来源，用于：

- 作者修改案例解释；
- 当前场景审美诊断；
- 正文方向审美裁决；
- 跨章节复验与长期迭代。

本目录不保存可供正文模型复刻的作者原句、成品笑点、具体桥段、动作链、段落顺序或结尾方案。

## 二、文件职责

|文件|职责|
|---|---|

|[[System_alpha/04_Skill系统/03_正文生成/作者审美决策库/CHANGELOG_作者审美决策库|变更日志]]|记录条目新增、升级、拆分、降级和废弃|

|[[设计流程与方案/共创机制/CURRENT_作者审美链工作状态|当前工作状态]]|跨对话续接、最小读取集和下一步任务|

## 三、判断等级

|等级|含义|使用方式|
|---|---|---|
|L1 单案例假设|来自单次文本差异，尚未获得作者确认|只能提示风险|
|L2 重复倾向|在多轮或多章复现，但适用范围仍需验证|结合场景条件使用|
|L3 作者确认原则|作者已明确确认|只在条目适用条件成立时使用|

L3 不等于全局硬规则。条目仍必须服从适用条件、不适用条件和当前章节事实。

## 四、四层长期结构

```text
原始文本与作者修改稿
→ 案例记录：保留来源和作者确认
→ 正式审美库：只保存抽象判断与边界
→ CURRENT：只保存当前阶段与下一步
```

原始文本用于证据核查，不直接进入正式审美库。正文模型不得读取原始作者范文或修改对。

## 五、每轮更新协议

每完成一次作者校准，按顺序执行：

1. 新建或更新对应案例记录；
2. 将已确认判断写入正式审美库；
3. 在变更日志追加一条记录；
4. 更新 `CURRENT_作者审美链工作状态.md`；
5. 明确下一轮最小读取集；
6. 检查是否误写入具体表达方案；
7. 检查旧判断是否需要收窄、拆分、降级或废弃。

## 六、跨对话续接原则

新对话默认不重读历史聊天和全部设计文档。先读取：

1. `CURRENT_作者审美链工作状态.md`；
2. 当前任务引用的正式审美库条目；
3. CURRENT 指定的原始文本片段。

只有出现冲突、来源不清或需要追溯设计理由时，才读取案例记录、变更日志或完整设计方案。

## 七、正文模型隔离

正文模型可以读取当前场景诊断后的：

- 当前文学问题；
- 选定处理方向；
- 必须保持的事实与边界；
- 必须保留的未确定性和张力；
- 创作自由区。

正文模型不得直接读取：

- 原始作者修正稿；
- 作者原句；
- 原始修改对；
- 案例中的具体表达证据；
- 历史章节的成品处理方案。

## 八、维护边界

- 稳定 ID 创建后不得因改名而变化。
- 新证据优先更新适用边界，不自动新增重复规则。
- 作者新判断与旧判断冲突时，不静默覆盖；必须建立冲突记录。
- 当前状态文件保持轻量，只保存继续工作所需信息。
- 变更日志只追加，不用聊天全文替代结构化记录。
