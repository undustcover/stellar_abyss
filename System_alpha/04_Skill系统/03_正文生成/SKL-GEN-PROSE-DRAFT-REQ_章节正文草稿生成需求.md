# 章节正文草稿生成需求

> [!warning] 2026-07-22 状态校准
> 本文件是现有 Draft 生成 Skill 的历史设计基线，不是正式 Skill 需求。当前 `.codex-tmp/skills/generate-chapter-prose-draft/` 仍处于设计和测试阶段，不迁移、不安装、不晋升。下一阶段以 `SKL-GEN-DRAFT-SOURCE-REDIRECT_源头Draft问题重定需求.md` 为问题重定入口；本文件中的规则须重新证明必要性，不能整份默认继承。

## 一、定位

`generate-chapter-prose-draft` 是正文生成链路中的测试版正文草稿生成 Skill，目前尚未正式化。

它不再依赖独立的去 AI 化、文风校准、对白幽默化后处理 skill。它的目标是：

```text
根据章节执行包中的分场景导演目标，生成可直接进入作者判断的自然正文底稿。
```

底稿必须比普通 AI 续写更接近人类手写状态，尤其要做到：

1. 按场景提前执行去 AI 感、文风一致性和对白幽默控制；
2. 保留作者已确认的喜剧组织方式，但不把幽默做成后置滤镜；
3. 让对话更像人物在真实场景中碰撞，而不是配合传递信息；
4. 让叙事推进像生活压力、误会、动作和关系自然滑动，而不是按大纲逐项说明；
5. 正确执行章节文风模式、人物语言边界、信息释放边界、场景等级、字数预算和生成备注机制。

本 Skill 的产物是正文草稿，不是分析报告。正文主体必须干净，风险、缺口和可疑句子放入 `## 生成备注`。

## 二、输入资料

正式生成时优先读取作者确认后的资料。若资料缺失，允许保守生成，但必须在生成备注中标记。

|输入|用途|读取要求|
|---|---|---|
|单元上下文包|确定本单元事实、信息释放、误会结构和隐藏真相|优先读取 `status: confirmed` 文件|
|章节大纲|确定本章功能、核心事件、结尾落点和章节文风模式|最高章节权威|
|章节细纲|提供场景顺序、场景目标、必须出现的行动和信息|不得擅自改写核心事件|
|章节执行包|提供场景执行策略、喜剧/正剧策略、风险提示|可被正文转化，不可机械照抄|
|人物卡|提供语言习惯、语言与幽默方式、认知边界、人物时间轴|角色语言最高权威|
|事件卡|提供世界事实、因果、状态变化和逻辑限制|只调用必要摘要，不全文搬运|
|已批准规则文件|提供梗使用、喜剧化写法、章节文风模式、角色语言规则和分场景导演规则|必须属于 Skill 系统、叙事系统、设定系统或事件数据库|

生成链路不得把外部正文草稿、对比报告、成品样文、成品梗、成品对白或其他非系统写作材料作为输入资料。若用户临时提供此类材料，只能标记为非系统资料并排除，不得参与细纲、执行包或正文生成。

## 三、权威层级

|层级|内容|处理方式|
|---|---|---|
|最高事实权威|人物卡、事件卡、设定卡|不得改写世界事实|
|最高章节权威|章节大纲|不得改写章节功能、核心事件、结尾情绪|
|人物语言权威|人物卡的 `语言习惯`、`语言与幽默方式`、`喜剧表现结构`|不得让角色为了讲清信息变成说明器|
|执行策略权威|章节细纲、章节执行包|转化为正文行动、对话、误会和场面推进|
|风格约束权威|章节文风模式、人物语言字段、结构化喜剧规则、分场景导演规则|控制去 AI 感、叙事流、读者体验和人类手写感|
|生成备注|风险记录|只提示问题，不替代正文|

当资料冲突时：

1. 世界事实优先于场景发挥；
2. 人物语言边界优先于信息讲清；
3. 章节大纲文风模式优先于执行包临时笑点安排；
4. 若必须牺牲信息完整度，也不能让人物突然变成解释机器；
5. 冲突写入生成备注。

## 四、生成目标重写

本 Skill 不能只把细纲扩写成正文。

它必须先读取章节执行包中的分场景导演目标，并把章节细纲/执行包转化为以下五条内部链路，再写正文：

|链路|作用|要求|
|---|---|---|
|生活压力链|让场景从具体生活经验中长出来|工作群、饭桌、电话、物件、家庭动作、钱和流程都要有可感细节|
|误会语义链|让伏笔和误会自然滑动|关键物件、词汇和动作出现前，必须有语境铺垫|
|人物情绪链|让对白由情绪和目的驱动|每个主要角色进入场景时要明确急什么、怕什么、想躲什么、想压住谁|
|反杀材料链|让前期笑点能在后期变味|所有帮忙、热心、到场、看资料、打电话等动作都要能留下现实痕迹|
|读者体验链|让章节按网文读者的阅读感受推进|每场明确期待、误会、压力、笑点、揭示或钩子，避免只写事件经过|

正文生成时，场景不是：

```text
事件A发生 -> 解释A意义 -> 事件B发生 -> 解释B意义
```

而应是：

```text
生活压力出现 -> 人物按自己的情绪反应 -> 反应造成误会或错位 -> 动作推进事实 -> 新物件/新词汇自然出现 -> 旧误会被加固或轻微改写
```

## 五、作者文风执行原则

### 1. 生活经验优先于抽象概括

不要先写“马雯雯很累”“饭局很压抑”“马木不靠谱”。

应先写能让读者感到这些状态的具体东西：

1. 工作群还亮着；
2. 血压仪、方案首页颜色、甲方改需求；
3. 饭桌上的排骨、筷子、茶杯和亲戚改话题；
4. 火柴盒、身份证夹、手牌、会员确认书；
5. 电话里被偷听到的碎片。

抽象判断可以出现，但必须在具体细节之后。

### 2. 场景之间要有滑动关系

不能换一个地点就重启一段说明。

每一场都要从上一场滑过来：

|上一场|下一场滑动方式|
|---|---|
|职场压力|家庭饭局像另一种绩效面谈|
|马木讲服务意识|云水汇、手牌、火柴盒的误会语境提前形成|
|陈科说位置和资格|马雯雯按会所语境误读，读者逐渐看出反诈语境|
|马木热心帮忙|后续正式流程把帮忙翻译成参与痕迹|

### 3. 幽默要有连续递进

正文不能靠孤立俏皮话。

核心笑点至少要有：

|阶段|说明|
|---|---|
|铺垫|正常生活压力或人物意图出现|
|升级|人物为了面子、关系或误会把话说歪|
|错位|读者看见两套语境同时成立|
|落点|一句话、一个动作或一个物件让错位显形|
|回声|后文继续影响误会、关系或反杀材料|

如果一个笑点只有一句话，没有前后回声，默认降级或删除。

### 4. 结尾要有多层功能

章节结尾不只是“完成钩子”。

优先形成三层：

1. 表层误会仍成立；
2. 真实事件正在推进；
3. 后文反杀或情绪翻面有伏笔。

若大纲原句不自然，允许改写原句，但必须保留章节功能和结尾情绪。

### 5. 二次改写必须先做取舍表

当正文进入二次修改或返修生成时，不能直接“继续加戏”或“整体压缩”。

必须先在内部确认四类内容：

|类别|处理规则|
|---|---|
|保留|已经承担章节功能、人物弧线、误会语义或后续钩子的内容必须保留。|
|压缩|密度过高但仍有行动功能、信息功能或情绪功能的内容，只压低表达，不删除承压点。|
|删除|只负责显得好笑、漂亮、聪明，但不改变局面、不暴露人物、不推进动作的句子优先删除。|
|延后|章节大纲不要求本章说透的真相、机制、长期矛盾和人物完整背景，延后到后续章节。|

二次改写的目标不是让文本更“满”，而是让每个场面只保留最有功能的压力、误会、动作和钩子。

### 6. 每场只保留一个主要笑点，但不能删掉承压点

每个场景最多明确一个主要笑点或主要错位。

但压缩笑点时，不能同时删掉以下承压点：

1. 人物被迫做选择的动作；
2. 物件、流程或钱造成的现实压力；
3. 支持角色情绪变化的转折；
4. 结尾主钩子的前置铺垫；
5. 下一场必须继承的误会或后果。

如果删掉一句金句后，场景只剩“事情发生了”，说明删错了。应改为保留行动，把漂亮句子压成更粗糙的人话、动作或沉默。

### 7. 角色台词要有胜率控制

关键角色不能每次开口都赢。

尤其是喜剧能力强、经验强或嘴硬的角色，必须遵守：

1. 强台词必须绑定行动推进、关系压力、误会加深或信息边界；
2. 只好笑但不改变局面的台词，默认降级或删除；
3. 一个场景内同一角色连续赢两次后，第三次应让动作、沉默、被打断、找补或失败承担节奏；
4. 角色不能因为作者需要金句而变成导师、裁判、反诈讲解员或稳定赢家。

马木可以敏锐、粗糙、有救火经验，但不能稳定解释全局。陈科可以嘴硬、要面子、半懂不懂，但不能替骗局念完整理论。马雯雯可以误会和吐槽，但不能成为全知旁白。陈艾艾可以处理现实问题，但必须先经过真实情绪压力。

### 8. 误会尺度要体面、含混、双语境成立

轻喜误会不是把低俗联想摆到台面上。

误会应优先通过以下方式成立：

1. 含混词汇；
2. 正规流程里的不对劲；
3. 人物不敢问出口的停顿；
4. 物件或地点的双重语义；
5. 对方越礼貌，主角越困惑；
6. 主角努力用成年人方式理解一个她不该理解的场面。

避免反复使用“扫黄”“不正经”“涉黄”等直白词汇。直白词会让误会变低级，也会让人物显得像段子工具。

### 9. 概括式幽默必须转成场内材料

发现“很努力”“很尴尬”“很像某种东西”“气氛突然变了”这类概括式幽默时，不得用更漂亮、更文气、更聪明的旁白替换。

优先改成：

1. 场内动作：停杯、停筷、换鞋、把登记表推过来；
2. 物件反应：证件夹、银行卡、手牌、立牌、手机定位；
3. 流程错位：预约、登记、本人确认、家属稍等；
4. 关系反应：有人装没听见、有人找补、有人被看得坐直；
5. 对白断裂：抢话、停顿、答非所问、话说过头后补救。

幽默必须在场里发生，不要只由叙述层告诉读者“这里好笑”。

### 10. 支持角色必须有情绪弧线

正文生成前要为本章重要支持角色确定最小情绪弧线。

支持角色不能只承担“求助、解释、搞笑、推动剧情”的单一功能。若角色在本章承担现实压力，应至少有两个状态变化。

例如：

```text
慌 -> 羞 -> 火 -> 控
```

这类弧线不必输出为分析，但必须进入正文动作：一开始可以失态，随后不愿说、被迫说，再爆发，最后开始处理证件、钱、手机、银行卡或人。

### 11. 结尾只能有一个主钩子

章节结尾可以有轻喜收束，但必须先确认主钩子。

规则：

1. 主钩子服务下一章推进；
2. 轻喜收束服务人物余味；
3. 情绪余波服务读者不适感或关系变化；
4. 三者同时存在时，主钩子优先级最高。

如果轻喜收束更抢眼，必须压低轻喜收束，或把它放在主钩子之后作为短回声。不得让“没钱打车”一类轻收束盖过“明天中午前保留位置”一类后续压力。

### 12. 分场景导演优先于后置修订

生成前必须按执行包确认每场的导演目标：

|场景等级|生成策略|字数控制|
|---|---|---|
|A 类核心场景|完整写出压力、误会、动作、人物反应和转折|优先分配字数|
|B 类过渡场景|只承接状态、地点、情绪或小信息|短写，不展开笑点|
|C 类信息场景|通过冲突、流程、物件或必要追问释放信息|压缩说明，避免问答块|

去 AI 感、文风一致性和对白幽默化都在本步骤内完成，不再交给独立修订 skill。若生成后某场失败，只进入局部替换，不整章重写。

## 六、场景导演硬规则

### 1. 逻辑藏在底层

底层因果必须严密，但表层不能像分析报告。

禁止默认写法：

```text
人物做出行为 -> 立刻解释动机 -> 总结这说明了什么。
```

改为：

```text
人物做出行为 -> 另一个人物误读或反应 -> 行为后果推动下一步 -> 必要解释被追问逼出来。
```

### 2. 不主动解释笑点、误会和主题

不要写：

1. “这就是马雯雯误会的原因”；
2. “陈科的荒唐来自体面焦虑”；
3. “这句话后来会成为反杀伏笔”；
4. “任战的话术击中了他的自尊”。

正文只写人物、动作、语言和场面。意义留给读者自己接。

### 3. 少写漂亮比喻，多写粗糙反应

AI 感常来自过度精致、过度完整、过度润色。

优先使用：

1. 半句话；
2. 找补；
3. 顶嘴；
4. 停顿；
5. 具体物件；
6. 生活化粗糙判断；
7. 关系里的尴尬沉默。

谨慎使用：

1. 漂亮排比；
2. 诗化比喻；
3. 金句式人生总结；
4. 旁白替人物总结；
5. 每段末尾都落一个漂亮句子。

### 4. 解释必须被逼出来

人物只有在以下情况下才解释：

1. 被追问；
2. 被误解；
3. 对方不照做；
4. 关系压力逼迫；
5. 利益相关，不解释不行；
6. 权威或机构要求说明。

马木不能主动讲完整反诈逻辑。陈科不能主动承认自己被骗。陈艾艾不能替作者总结父亲心理。任战不能解释入席机制。

### 5. 环境描写必须有功能

只保留三类环境描写：

|类型|说明|
|---|---|
|推动误会的物件|火柴盒、手牌、身份证夹、确认书、手机消息|
|暴露人物反应的动作|停筷、换话题、扣手机、看门、倒茶|
|参与包袱的道具|排骨、茶杯、血压仪、猫粮、猫砂、灯牌|

纯气氛描写默认删除或压缩。

## 七、对白生成规则

### 1. 对白先问三件事

每句对白生成前，先判断：

|问题|作用|
|---|---|
|这个人此刻急什么？|决定句子长度、是否抢话、是否重复|
|这个人想达成什么？|阻止、逃避、压制、求助、维护面子、推责|
|这个人和对方是什么关系？|决定能不能省略、顶嘴、阴阳怪气或沉默|

对白不是为了给读者讲清楚，而是人物在当下关系里争夺主动权。

### 2. 关键对话必须有自然断裂

每个关键对话段至少出现一种自然断裂：

1. 抢话；
2. 答非所问；
3. 话说过头后找补；
4. 被问到具体问题时停顿；
5. 用大词遮羞；
6. 用动作替代回答；
7. 两个人各说各的急事；
8. 旁人误读同一句话。

如果一段对话是标准问答，必须改写。

### 3. 人物语言不能同质化

|人物|语言驱动|不可写成|
|---|---|---|
|马雯雯|职场压力、年轻人现实吐槽、亲属误读|油腔滑调段子手|
|马木|粗糙经验、救火惯性、嘴硬找补|反诈导师、隐藏高手、人生哲学家|
|陈科|体面焦虑、大词遮羞、长辈防御|纯蠢老人或主动认错者|
|陈艾艾|现实追问、子女焦虑、短句逼问|搞笑担当或心理分析师|
|张富贵|粗糙父爱、嘴硬、土办法|单纯小丑或恶父|
|张凡|沉默、非语言反应、小满连接|解释自己心理的成熟旁白|
|陈灏|轻浮观察、富人经验、轻微异常感知|反诈专家或世界观讲解员|
|周睿|旧人分寸、短句点破、现实距离|替马木洗白的人|
|任战|稳定、柔和、施压、转换身份问题|搞笑角色或夸张反派|

### 4. 信息要靠冲突和动作带出

信息释放优先级：

1. 人物为了办事说出必要信息；
2. 对方误解后逼出部分解释；
3. 物件或消息自然露出；
4. 行动造成后果；
5. 旁白补一小句必要事实。

禁止让两个人坐下来配合讲清背景。

---

## 八、叙事流程规则

### 1. 场景推进要像生活，而不是像提纲

每个场景应包含：

|要素|说明|
|---|---|
|入口压力|人物为什么必须进入这个场景|
|场内动作|人物实际做了什么，不只说了什么|
|关系拉扯|谁想压住谁，谁想躲开谁，谁误会谁|
|信息露出|通过物件、电话、截图、错话、称呼露出|
|场景落点|本场给下一场留下什么误会、痕迹或情绪|

### 2. 伏笔不能孤立出现

火柴盒、身份证夹、会员确认书、三万元、锁位置等内容出现前，要有前置语义或动作铺垫。

例如：

```text
服务意识 -> 贵宾/手牌/正规 -> 火柴盒 -> 云水汇误会
```

而不是：

```text
马木突然掏出火柴盒，马雯雯开始怀疑。
```

### 3. 反杀材料要以轻松形式进入

前期证据链材料不能写得像法务清单。

它们应伪装成：

1. 顺手送东西；
2. 帮忙看一眼；
3. 电话里多问一句；
4. 替人截个图；
5. 陪到门口；
6. 被工作人员记住；
7. 吃饭时资料堆到桌上。

后期再由正式语言翻译成证据。

---

## 九、章节文风模式执行

正文生成必须读取章节大纲中的 `章节文风模式`。

|文风模式|正文执行重点|
|---|---|
|开篇脱口秀型|允许较高吐槽密度，但段子必须围绕同一生活压力链，不能散成段子合集|
|轻喜误会型|优先写双语境成立的误会，让人物认真维护自己的错误理解|
|冷幽默反讽型|幽默要带不安、反杀或现实刺感，不能把情绪写轻|
|正剧煽情型|少笑点，保留粗糙人话和动作，不写漂亮煽情总结|
|信息推进型|信息必须被冲突、追问、动作和利益压力带出，不能说明书化|

若章节大纲缺少文风模式，正文生成可以根据章节功能临时保守判断，但必须在生成备注标记缺口。

---

## 十、生成流程

正式生成必须按以下流程执行：

### 1. 输入审读

读取并确认：

1. 本章文风模式；
2. 本章核心事件；
3. 本章读者已知、主角已知、主角误解、隐藏真相；
4. 出场人物语言规则；
5. 必须出现的物件、动作、电话、证据痕迹；
6. 本章结尾情绪落点。

### 2. 风格预处理

在正文生成前内部构建：

|项目|内容|
|---|---|
|生活压力链||
|误会语义链||
|核心笑点递进||
|关键对话断裂点||
|反杀材料进入方式||
|本章 AI 感高风险||

这些内容不直接输出，除非生成备注需要说明。

### 3. 正文草稿生成

按章节自然顺序生成正文。

正文主体中不插入分析、不标注规则、不解释作者意图。

### 4. 分场景导演二次自检

生成后必须自查并改写：

1. 删除主动解释动机的句子；
2. 删除漂亮但无功能的比喻；
3. 把标准问答改成情绪碰撞；
4. 把信息说明改成动作、误会、追问或物件露出；
5. 检查每个角色是否共享同一种吐槽腔；
6. 检查场景之间是否只是并列推进；
7. 检查结尾是否只有钩子，没有反杀层。

二次改写还必须执行以下检查：

8. 是否已经列清本章保留、压缩、删除、延后的内容；
9. 是否每场只有一个主要笑点，且承压点没有被删掉；
10. 是否有关键角色连续开口都赢，变成金句输出器；
11. 是否把概括式幽默换成了更漂亮的旁白，而不是场内动作；
12. 是否为了延后真相，把本章主钩子也删掉或压没；
13. 是否把支持角色从情绪弧线改成了单一功能角色；
14. 是否用“一会儿后”“过了一阵”跳过了高风险冲突场面。
15. 是否按 A/B/C 场景等级控制字数，没有让每个场景都膨胀成核心场景；
16. 是否已经把去 AI 感、文风一致性和对白幽默内置到场景里，而不是只在句面润色；
17. 是否只需要局部替换失败场景，而不是整章重写。

### 5. 生成备注

正文之后输出 `## 生成备注`。

## 十一、输出格式

```markdown
# 第X章：章节名

正文内容。

## 生成备注

|项目|内容|
|---|---|
|输入缺口||
|文风模式执行情况||
|文风模式与剧情冲突||
|角色语言字段缺口||
|角色语言越界风险||
|人物同质化风险||
|疑似AI感句子||
|疑似硬梗||
|解释过度风险||
|对话配合问答风险||
|漂亮比喻过度风险||
|叙事说明书化风险||
|幽默冲淡正剧情绪风险||
|角色台词胜率风险||
|误会尺度风险||
|主钩子聚焦风险||
|承压场面被压缩风险||
|支持角色情绪弧线风险||
```

如果没有风险，写“未发现明显风险”，不要强行编问题。

## 十二、正文禁区清单

生成正文时必须避免：

1. 人物行为后立刻解释动机；
2. 对话承担说明书功能；
3. 人物配合问答；
4. 每句都完整、顺滑、工整；
5. 旁白替人物总结心理；
6. 漂亮比喻连续出现；
7. 环境描写无功能；
8. 为搞笑而脱离人物和场景；
9. 正剧情绪中强行轻松；
10. 马木过早显得像高手；
11. 陈科被写成纯蠢；
12. 陈艾艾被写成搞笑角色；
13. 马雯雯变成全知吐槽机；
14. 陈灏变成反诈专家；
15. 任战变成夸张反派或段子角色；
16. 张凡用成熟语言解释自己；
17. 周睿替马木讲完整过去；
18. 用章节、单元、创作意图解释人物状态。
19. 关键角色连续输出金句，每次开口都赢；
20. 把需要动作承压的场面压缩成一句过场；
21. 用更漂亮的旁白替代概括式幽默；
22. 误会表达过于直白、低俗或单语境；
23. 轻喜收束盖过章节主钩子；
24. 支持角色没有情绪弧线，只负责传递信息或制造气氛。

---

## 十三、质量验收标准

一章正文底稿合格，至少满足：

|维度|合格标准|
|---|---|
|可读性|读者能顺着场景自然读下去，不像在读大纲扩写|
|人类手写感|对话有断裂、找补、沉默和关系压力，不是标准问答|
|系统文风执行度|有生活压力链、连续递进笑点、具体物件和场景滑动，并符合章节文风模式|
|去 AI 感|逻辑在底层，解释不主动浮出|
|人物区分度|每个角色的说话方式来自人物卡，而非统一吐槽腔|
|信息边界|主角不知道的真相不提前泄露|
|误会有效性|误会不是靠人物降智，而是靠双语境同时成立|
|后续反杀|前期轻松动作能在后文被现实语言重组|
|台词胜率|强角色不能连续赢，关键台词必须绑定行动、关系或信息边界|
|主钩子聚焦|章末有且只有一个主钩子，其余落点不抢后续推进|
|承压场面|高风险冲突不被跳过，压缩后仍保留动作、物件、钱或流程压力|
|支持角色弧线|重要支持角色有可见状态变化，不只是功能性求助或说明|
|字数控制|A 类场景精写，B/C 场景压缩，没有全章均匀膨胀|
|局部返修|能定位失败场景并给出局部替换建议，而不是默认整章重写|
|生成备注|能指出风险和缺口，便于作者返修|

---

## 十四、与后续 Skill 的关系

本 Skill 是当前正文生成链路的主要落点。去 AI 化、文风一致性和对白幽默化不再作为独立后处理 skill 存在，而是并入：

```text
章节细纲 -> 章节执行包 -> 正文生成
```

后续如果正文测试发现读者体验不足，优先回查：

1. `generate-chapter-execution-pack`：补 `喜剧/正剧执行策略` 和 `误会语义链`；
2. `generate-chapter-detail-outline`：补 `读者体验曲线`、`场景导演目标` 和场景级文风分配；
3. `generate-unit-context-package` 或资料调用层：补单元上下文包中的人物语言、事件限制、信息隐藏和章节文风调用摘要。

生成稿失败后的处理方式是：

```text
草稿诊断 -> 定位失败场景 -> 分场景再导演 -> 局部替换
```

该流程只作为返修保险，不作为默认主流程。当前阶段不新增外部成稿校准 Skill，正文生成端必须直接执行系统内已批准的章节文风模式、人物语言规则、结构化喜剧规则和分场景导演规则。
