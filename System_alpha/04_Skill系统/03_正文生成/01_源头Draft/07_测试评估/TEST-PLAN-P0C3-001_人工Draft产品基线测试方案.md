# P0C3 人工 Draft 产品基线测试方案

## 测试目标

不修改或调用旧 Draft 测试 Skill，由 Codex 按 confirmed 新链路人工生成 P0C3 测试 Draft。作者只审阅最终 Draft；Codex 使用统一日志和定位地图反推问题位置。

本轮不是验证固定句子或复刻旧正文，而是验证章节轻重能否明确、需要亲历的节点能否通过人物言行与现实反馈形成画面，而不是给章节任务配置动作、物件和说明。

## 本章创作方向

- 主视角：马雯雯限知视角；不进入其他人物内心；
- 表层故事：马雯雯继续以会所／高级服务场所误会理解云水汇和相关人物；
- 人物推进：陈艾艾与陈科冷硬冲突但底层相爱；马木提出建议，被陈科反驳后心灰意冷地退出参与；陈灏不承担真相解释；
- 异常要求：必须有清晰、可感知的异常现象；
- POV 人物解释：马雯雯可以归因于眼花、疲劳、没休息好、灯光或现场状态；
- 读者所得：读者能够判断异常不是普通巧合或纯主观错觉，但不知道来源、机制和真实目的；
- 信息禁区：不解释任战、入席、灵能、异常体系或云水汇真实结构；不提前把章节写成反诈说明；
- 整体读感：喜剧误会仍是前景，人物关系是推进力，异常形成明确的整体故事坐标，而不是抢走正文变成设定展示。

## 章节轻重与体验节点

|节点|任务|本轮确认|
|---|---|---|
|四人离开云水汇、完成离场承接|推进|快速通过，不强行配画面|
|异常仍延伸到外部|短体验|马雯雯处理正常事务时，现实功能返回明显异常结果；她按现实方式处理，不解释、不复证|
|父女冲突牵连马木并导致退出|核心体验|陈科说“你和我什么关系，你管这么宽”；陈艾艾仍按原求助关系交接后续，马木不再接受，求助渠道第一次失效|
|陈灏以熟悉云水汇的状态接入|短体验|陈灏按自己的目的行动，工作人员直接以熟客方式回应，由人物互动显出熟悉|

以上只确认事件变化与体验任务，不预设具体异常形式、人物动作、物件、镜头句法或陈灏措辞。

本轮生成前，作者已进一步确认三个现实显形方向，详见 `BRIEF-P0-C03@1.4`。004只按上述事件过程生成，不增加新的场景任务或风险控制表。

## 异常表现测试假设

本轮只测试结果边界，不预先规定固定异象或成品桥段：

1. 至少有一次读者能清楚识别的异常，并与第二章已有的楼层数字跳变、三楼人影或云水汇状态形成一次前后呼应；
2. 异常从视觉／空间、身体感受、物件状态、声音、服务行为或因果反应中按需选择少量维度，不要求全部使用；
3. 马雯雯的合理化必须符合她的疲劳、经验和当前误会，不能为了隐藏真相突然变笨；
4. 文本不直接宣布“这是异常”，但读者不能只得到模糊气氛；
5. 异常不得破坏马雯雯限知 POV，不借旁白偷渡后台解释；
6. 具体异象、出现位置、反应和语言仍由生成时创造，本方案不锁定唯一写法。

以上只作为 P0C3 测试假设。通过作者审阅前，不晋升为通用方法或控制单元。

## 画面验证

作者无需逐项打分，只需阅读 Draft 后说明真实感受。Codex 在后台检查：

|维度|观察问题|首要定位点|
|---|---|---|
|叙事任务|只需推进的节点是否被强行场景化；需要亲历的节点是否仍停留在对白和说明|节点判断 / 执行|
|现实显形|事件具体改变了什么现实关系、功能、权限、习惯或行动结果|画面方法 / 执行|
|人物表现|人物是否在说话和处理现实，而不是用动作表演情绪|人物资料 / 画面方法 / 执行|
|POV镜头|注意力是否由声音、动作、空间、功能失败或他人反馈自然带到下一处|POV / 画面方法 / 执行|
|信息覆盖|人物言行与现实反馈已经成立的含义是否又被旁白、动作或物件复证|画面方法 / 执行|
|微量补缝|补入内容是否只解决空间、指代和因果真断点|画面方法 / 执行|
|客观边界|连续性、POV、隐藏信息和硬结果是否越界或遗漏|Brief / handoff / POV|

作者无需逐项打分，只需阅读 Draft 后说明真实感受。Codex 不以内部检查证明产品自然。

生成前必须具备并由作者确认：

1. P0 单元上下文包实例；
2. P0 单元总体规划的适用版本；
