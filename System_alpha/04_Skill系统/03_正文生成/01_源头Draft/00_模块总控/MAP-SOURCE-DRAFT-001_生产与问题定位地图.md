# 源头 Draft 生产与问题定位地图

## 地图定位

本地图回答三个问题：生产时从哪里进入、实际调用落在哪里、出现问题后首先修改哪里。

它是开发与诊断索引，不是新的章节产物或前台步骤。普通 Draft 运行只需由模块入口按需定位文件，不全文读取本地图连接的全部资料。

2026-07-22，用户已明确：当前开发的方案、结构和接口必须有地图标明位置，使 Draft 生产中发现的问题能够定位到具体执行模块并支持迭代。本地图是该要求在源头 Draft 样板中的实现；待结构确认后，再决定是否抽象为其他模块共用的地图规范。

## 三、问题定位地图

|观察到的问题|首要定位点|何时转向其他位置|
|---|---|---|
|人物、事件或设定原始事实错误|对应人物卡、事件卡或设定卡|原始资料正确但压缩错误时，转单元上下文包|
|本单元漏掉必要事实、来源版本错或事实冲突|confirmed 单元上下文包|问题来自作者如何组织事实时，转单元总体规划|
|单元目标、人物弧线、信息释放顺序或章节功能错误|confirmed 单元总体规划|单章落实错误时，转章节大纲／细纲|
|场景顺序、场景变化或章末结构错误|章节大纲／细纲|结构正确但本章创作方向表达不清时，转 Draft Brief|
|本章目标、故事焦点、POV 决定、读者信息边界或章末方向错误|章节 Draft Brief|Brief 正确但执行越界时，转对应控制单元或执行层|
|前章站位、未完成动作、情绪余温或读者问题丢失|`previous_chapter_handoff` 提取与使用|上一章正式版本选错时，转输入版本引用|
|关键人物为什么说／做或为何不采取直接行动不成立|人物意图审核|卷级基础或单元差异本身缺失时，转上游人物资料或单元上下文包|
|普通场景跳入多人内心|`POV-LIMITED-001`|控制单元未启用是路由问题；已启用仍失败是执行问题|
|全景场景无规模理由、偷读内心或失去人物|`POV-PANORAMIC-001` / `POV-EXTERNAL-EVIDENCE-001`|两单元冲突未裁决时，转模块路由|
|需要某种效果但没有已确认方法|对应模块 Skill 的方法缺口|目标本身不清时先回 Brief 或询问作者|
|方法选择错误或方法本身无效|运行日志记录的方法 ID|选择条件错误改路由；方法证据不足改方法或测试，不改事实资料|
|当前故事不可定位或推进关系不可跟随|故事四项审核结果，再追首个失效上游|可能落到章节结构、POV 单元或 Draft 执行，不直接新增通用规则|
|人物体验弱或缺少继续动力|故事四项审核标记 `RISK`|由作者判断后，再定位结构、方法或执行层|
|事实、结构和控制项都正确，但句面仍僵硬或说明化|Draft 执行层／正文生成 Skill|只有跨实例稳定复现后才形成方法或控制单元候选|
|正文像逐项完成要求的场景说明书|`MOD-SOURCE-DRAFT / generation_context_resolution`|执行核已经去重仍失败时，转章节结构转译或本次执行|
|语言古旧、死板或与人物时代割裂|作品级 `language_profile` 与实际知识选择|配置正确但句面仍失败时，转本次执行；人物不适配时转知识条目元数据|
|网络梗太密、过时、硬贴或所有人物同腔|实际 `knowledge_items` 与人物适配|没有调用知识仍出现时，转模型已有知识的执行选择|
|日志声称已读取或已审核，但实际未发生|统一日志接口或模块路由实现|正式测试中再检查测试 Skill 绑定，不在当前阶段修改|
|只在某个测试输入出现异常|测试用例或测试污染|确认跨实例复现后才上升到模块、方法或控制单元|

## 五、统一定位格式

运行日志中的问题使用下列定位类型：

```text
source_artifact  原始人物、事件、设定或正式正文
input_interface  单元上下文包、总体规划、章节结构、handoff 或 Brief
module           模块入口与能力路由
skill            实际执行 Skill 及其版本
control_unit     最小控制单元 ID
method           实际方法 ID
execution        模型在正确输入和规则下的本次执行
test             测试输入、污染、比较或验证流程
```

定位值必须使用可追溯 ID 或文件路径；Skill、控制单元和方法同时记录版本。不能确定时写 `needs_diagnosis`，不得把同一问题同时归给所有层。

## 六、最小归因流程

```text
描述可观察问题
→ 在运行日志中找到首次偏离的位置
→ 记录一个首要定位点和证据
→ 只修改该责任位置
→ 用相邻章节或保留输入回归验证
```

世界事实、作者审美方向或人物关键选择仍由作者最终决定。一次修改或单章失败不能自动晋升为通用方法或控制单元。

## 七、用户确认

2026-07-22，用户确认：

1. 本地图作为源头 Draft 的固定后台入口；
2. 每个问题先记录一个首要定位点，再按证据转向；
3. 地图只登记位置和责任，不复制各文件内容，也不增加作者日常步骤。

## 八、总纲到单章生产的门禁

```text
逐章讨论并写入单元总纲
→ 完成全部章节间的因果、人物弧线和信息阶梯校验
→ 作者确认单元总纲
→ 按正文生产顺序固化当前章的独立可执行章节结构
→ 生成并确认章节 Draft Brief
→ 进入 Draft 生产
```

“独立可执行章节结构”是对已确认总纲内容的单章固化，不是恢复章节执行包。单元总纲尚在优化时，不因某一章先讨论完成就提前固化；但只有一句章功能提要也不得越过该门禁直接生成正文。
