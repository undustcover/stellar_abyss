# 源头 Draft 模块说明

## 1. 模块目标

本模块是“后台模块化、前台短流程”的第一个样板。它负责让一次生成的章节 Draft 在故事表达、章节承接、可读性和故事性上成立，同时把可独立触发与验证的约束拆成最小控制单元。

本轮只建设正式知识侧的样板结构，不迁移、不安装、不修改 `.codex-tmp/skills/generate-chapter-prose-draft/`。

## 2. 权威需求与边界

- 聚合需求：`../../SKL-GEN-DRAFT-SOURCE-REDIRECT_源头Draft问题重定需求.md`；
- 模块化规范：`../../../00_Skill总控/SKL-ARCH_模块化与最小控制单元规范.md`；
- 当前路线：`../../../../00_总控台/DEV-ROADMAP_当前开发路线.md`。

聚合需求继续留在原位置，避免在样板未验证时制造“已经迁移或正式化”的误解。本模块只建立分类入口和首批候选控制单元。

## 3. 后台结构

|目录|职责|当前状态|
|---|---|---|
|`01_需求/`|经过确认的问题与验收目标|暂引用现有聚合需求，不复制|
|`02_方案/`|待确认的解决路径与取舍|核心方向已确认，见方案索引与阶段交接|
|`03_经验/`|具体创作与测试观察|已建立 `EXP-P0C3-001` 首轮产品失败与能力隔离证据|
|`04_方法论/`|可能跨实例有效的方法|待证据支持后建立|
|`05_控制单元/`|可触发、验证、关闭和归因的候选约束|已建立首批视角单元|
|`06_运行记录_按需/`|高风险、测试、恢复或故障诊断记录|默认不创建|
|`07_测试评估/`|固定输入、验收、失败与回退|P0C3 首稿已评审，能力隔离片段达到当前合格基线|

## 4. 首批控制单元索引

|控制单元|用途|自由度|状态|
|---|---|---|---|
|`POV-LIMITED-001`|普通场景维持单一人物限知视角|`hard`|candidate，未接入运行时|
|`POV-PANORAMIC-001`|规模性场景按条件启用全景客观视角|`hard`|candidate，未接入运行时|
|`POV-EXTERNAL-EVIDENCE-001`|全景客观视角保留外显人物证据|`hard`|candidate，未接入运行时|

控制单元文件位于 `../05_控制单元/`。首批只拆分用户已经确认的视角边界，不把承接、故事表达或可读性条目提前规则化。

## 5. 前台接口（目标形态）

普通创作只暴露短流程：

```text
用户目标 → 按需读取最小资料 → 直接生成 Draft → 检查硬边界 → 用户反馈
```

模块分类、控制单元选择和冲突裁决由未来 Draft Skill 内部完成。当前样板不改变现有测试 Skill 的输入、Prompt、引用或输出。

四类 confirmed 输入在生成前先解析成去重的内部正文执行核，再交给正文生成。执行核不是新增章节产物，不要求作者填写。详细边界见 `../02_方案/PLAN-PROSE-KERNEL-001_正文执行上下文收缩与语言知识接入.md`。

## 6. 本阶段验收

1. 每个控制单元只处理一个问题，能独立启用或关闭；
2. 默认规则与条件特例不会同时无边界生效；
3. 成功证据检查结果，不要求复现固定写法；
4. 失败可以定位到单元，但诊断信息不进入小说正文；
5. 所有文件明确标记“未接入测试 Skill”，避免样板结构被误认为正式运行时。
