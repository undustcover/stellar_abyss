# 人物意图轻量审核

## 定位

人物状态不在章节中重新定义：卷级统一人物基础行为，单元级记录相对差异及重大事件前后的转变。章节在生成 Draft 前只审核本章安排是否符合当前适用状态。

审核把控的是人物方向，不规定 LLM 必须使用的动作、台词、物件、句式或表演路径。只要没有违反已确认方向和事实边界，LLM 应保留现场发现、自由组合和提供作者灵感的空间。

## 上游职责

- 卷级：人物核心诉求、基础行为倾向、关系立场、认知边界和明显禁区；
- 单元级：相对卷级的差异、当前压力、关系偏移，以及重大事件的事件前状态、转变事件和事件后状态；
- 重大事件造成的变化先记录在单元级；只有确认持续影响后续单元或下一卷时，才经用户确认更新长期人物基础。

## 章节审核

每次 Draft 生成前运行一次轻量审核，检查：

1. 本章方向是否符合卷级基础和单元差异；
2. 当前处于重大转变之前、当时还是之后；
3. 人物认知是否支持关键选择；
4. 剧情是否迫使人物违反当前方向；
5. 未采取明显直接行动时，现有资料是否提供成立理由。

审核不重新生成章节人物意图轨道，不替 LLM 选择具体言行。

## 最小运行日志

```yaml
character_intent_audit:
  sources:
    volume_character_basis:
    unit_character_delta:
    chapter_structure:
  applicable_state:
  major_transition: none | before | at | after
  findings: []
  result: PASS | ASK_USER
  question:
```

正常 `PASS` 时保存日志但不展示给用户，直接进入生成。日志用于最终产物出问题后的回看与归因，不增加作者日常审阅负担。

## 必须询问用户

出现以下任一情况时返回 `ASK_USER`，说明已读资料、冲突、不同方向的影响和系统建议，并在作者决定前停止生成：

- 卷级基础与单元差异冲突；
- 本章关键行为违反当前人物方向；
- 人物避开明显直接行动，但资料没有成立理由；
- 多个方向会改变人物、关系、剧情或信息边界；
- 重大事件后的新状态尚未定义；
- 无法判断当前位于转变之前、当时还是之后；
- 当前选择可能形成新的长期人物状态。

若只是具体说法、动作选择、物件使用、节奏或局部表演存在多个合理方向，LLM 自主创作，不询问用户。

## 当前边界

本方案先作为源头 Draft 候选审核设计，不修改、安装或接入现有 `control-character-intent` 与 `generate-chapter-prose-draft` 测试 Skill。真实测试后再决定独立意图 Skill 承担轻量审核还是只保留复杂问题诊断能力。
