# 默认产物收缩与跨窗口接口

## 默认产物决定

- 章节级叙事表达契约不再作为大型默认产物，其必要职责并入简短的章节 Draft Brief；
- 场景演出契约只在方法缺失或方法冲突时按需启用；
- 章节执行包只在复杂调度、恢复、测试或故障复现时按需启用；
- 章节创作总控规划退出默认章节产物，固定职责归入源头 Draft 模块入口，实际调用进入日志。

默认只保留两个章节级中间产物：

1. 生成前的 `章节 Draft Brief`；
2. 生成中和生成后的 `Draft 运行日志`。

## 章节 Draft Brief

Brief 是每个独立章节对话框的唯一前置入口，合并资料引用、前章承接和作者确认的章节创作决策。它不复制大纲、细纲、人物卡、单元资料或方法正文。

最小内容候选：

```yaml
chapter_draft_brief:
  chapter:
  source_refs:
  previous_chapter_handoff:
  creation_decision:
  unresolved_questions: []
```

`creation_decision` 至少覆盖故事焦点、POV、读者信息边界、章末方向、已确认方法和方法缺口。具体模板留待下一阶段设计。

## Draft 运行日志

运行日志用于证明系统读取、审核、Skill与方法选择确实发生，并供失败归因。正常情况下不要求作者逐项审阅；只有关键问题才展示并询问。

日志候选内容包括资料版本、实际 Skill和方法、人物意图审核、故事四项审核以及作者询问与决定。具体模板留待下一阶段设计。

## 跨窗口规则

每章在独立对话框中创作。新对话框依赖经过确认的文件，不依赖上一窗口的聊天记忆。默认不读取上一章完整运行日志；只有失败复现、恢复、漏审检查或作者要求时才读取。

章节执行包保留为按需工具，不因退出默认链路而删除。
