# 故事表达四项审核

## 目标

用少量结果级证据判断故事是否“听得懂、愿意听”，不规定开场字数、固定节拍、反转数量、章末钩子或具体写法。

## 四项结果

|项目|验收问题|性质|
|---|---|---|
|当前故事可定位|读者是否知道当前故事主要跟随谁、处于什么情境、正在处理什么问题|基本边界|
|推进关系可跟随|读者是否能跟上重要变化怎样由行动、信息、决定或后果发展而来|基本边界|
|故事通过人物被经历|读者是在跟随人物经历、误解、选择与承受，还是在听后台设计说明|审美风险|
|存在继续动力|读者是否仍有一个活着的期待、压力、选择、关系变化或问题愿意继续跟随|审美风险|

读者可以暂时不知道真相，但不能不知道眼前正在发生什么。继续动力不等于强制悬念、反转或章末钩子。

## 最小日志

四项只在 Draft 运行日志中记录简短结论和必要证据，不逐场打分，不建立大型验收表：

```yaml
story_audit:
  current_story: PASS
  progression: PASS
  character_experience: PASS
  forward_pull: RISK
  note:
```

前两项明显失败时不应直接交付；后两项由系统标记明显风险，最终审美判断属于作者。

## 方法缺口

四项是目标，不是写作方法。若本章需要达到某项效果，但当前模块 Skill 中没有作者确认的适用方法，必须先与作者讨论方法；已有方法内的具体动作、台词、物件、句式和节奏继续由 LLM 自主创造。
