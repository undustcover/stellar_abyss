# 故事表达四项审核

## 目标

用少量结果级证据判断故事是否“听得懂、愿意听”，不规定开场字数、固定节拍、反转数量、章末钩子或具体写法。

## 四项结果

|项目|验收问题|性质|
|---|---|---|
|当前故事可定位|读者是否知道当前故事主要跟随谁、处于什么情境、正在处理什么问题|基本边界|
|推进关系可跟随|读者是否能跟上重要变化怎样由行动、信息、决定或后果发展而来|基本边界|
|故事通过人物被经历|读者是在跟随人物经历、误解、选择与承受，还是在听后台设计说明|审美风险|
|存在继续动力|读者是否仍有一个活着的期待、压力、选择、关系变化或问题愿意继续跟随|审美风险|

读者可以暂时不知道真相，但不能不知道眼前正在发生什么。继续动力不等于强制悬念、反转或章末钩子。

## 逻辑自洽与小说留白

逻辑审核只保证关键因果、人物选择、章节承接和整体结构没有硬伤或互相矛盾，不追求把每个联系方式、行动步骤、手续过程和信息来源都向读者说明。

审核时区分两层：

- 作者侧因果必须成立：关键事件为什么能够发生，不能依赖明显矛盾或事后强行补设定；
- 读者侧表达允许留白：只要正文中的一句话、一个动作或一个结果已经足以让读者推断，便不继续补全过程。

不能因为后台知道完整因果，就把完整因果全部写进正文。步骤完整不等于故事清楚，解释过量反而会破坏人物经历感并使正文说明化。

## POV 外信息的释放门禁

限知 POV 下，视角人物没有看见、听见或事后接触到的过程，不得直接写入正文。

视角外事件只在同时满足以下条件时释放：

1. 该信息会改变读者对当前关键行动、人物选择或章节主线的理解；
2. 信息能够通过视角人物可接触的台词、动作、物件、结果或外显余波进入；
3. 只释放当前章节所需的最小一点，不顺带解释整个幕后过程。

若一个视角外事实只用于作者保证因果成立，而读者无需立即知道，则保留在大纲或作者侧结构中，不转写为正文说明。

## 最小日志

四项只在 Draft 运行日志中记录简短结论和必要证据，不逐场打分，不建立大型验收表：

```yaml
story_audit:
  current_story: PASS
  progression: PASS
  character_experience: PASS
  forward_pull: RISK
  note:
```

前两项明显失败时不应直接交付；后两项由系统标记明显风险，最终审美判断属于作者。

## 方法缺口

四项是目标，不是写作方法。若本章需要达到某项效果，但当前模块 Skill 中没有作者确认的适用方法，必须先与作者讨论方法；已有方法内的具体动作、台词、物件、句式和节奏继续由 LLM 自主创造。
