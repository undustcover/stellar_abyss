# 单元上下文包与总体规划最小接口

## 本轮目标

只收缩并确认两个单元级稳定产物之间的职责和引用关系。本方案不修改现有模板、生成需求或测试 Skill，也不设计章节 Draft Brief 与运行日志。

## 现有重复

现有单元上下文包模板同时保存单元目标、核心矛盾、信息释放、情绪推进和读者奖励，已经进入创作规划职责。

现有 V2 单元大纲模板又重复保存事实依据、事实锁定、事件摘要、人物前后状态和多类事实约束。P0 样板因此既像事实压缩包，又像单元规划，两个产物无法独立归因。

## 最小职责

### 单元上下文包：本单元必须据以创作的事实与边界

只保留四组内容：

1. `identity_and_sources`：单元 ID、状态、版本、原始来源及其版本；
2. `relevant_facts`：本单元相关的人物进入状态与认知边界、关系事实、事件因果和必要设定规则；
3. `hard_boundaries`：不可改写事实、不可提前暴露的信息和明确禁区；
4. `source_issues`：缺口、冲突和待作者裁决项。

上下文包不保存单元目标、叙事顺序、人物弧线、章节安排、情绪推进、读者奖励、审美方向或写作方法。

### 单元总体规划：作者准备怎样组织这个单元

只保留四组内容：

1. `context_ref`：所依据的 confirmed 单元上下文包 ID 与版本；
2. `unit_direction`：单元目标、叙事起点、预期终点、核心矛盾、单元基底与审美方向；
3. `unit_progression`：必要的阶段推进、人物弧线或重大转变、读者与人物的信息变化；
4. `chapter_functions`：各章只承担的功能、主要 POV、关键信息变化和章末方向。

只有已存在且经作者确认的单元级方法才以 `method_refs` 可选引用；不在总体规划内复制方法正文。规划缺口或冲突统一进入 `unresolved_questions`。

总体规划不复制人物卡、事件摘要、设定规则或事实来源表；需要事实时通过 `context_ref` 回到上下文包。

## 两者之间的唯一接口

```yaml
unit_context:
  id:
  status: confirmed
  version:
  unit:

unit_plan:
  id:
  status: confirmed
  version:
  unit:
  context_ref:
    id:
    version:
  method_refs: []
```

接口规则：

- `unit` 必须一致；总体规划只能引用一个当前 confirmed 上下文包版本；
- `source_issues` 中存在会改变事实或边界的未决冲突时，上下文包不得标记为 `confirmed`；
- 总体规划用实体或事件 ID 指向上下文包中的事实，不复制事实正文；
- 规划需要上下文包未包含的新人物、事件或设定时，先补充并重新确认上下文包，再继续规划；
- 仅调整讲述顺序、章节功能、审美方向或预期终点时，只更新总体规划；
- 事实来源、认知边界或硬禁区变化时，更新上下文包，并复核总体规划是否仍成立；
- 单元内已写章节造成的短期状态不写入任一文件，继续由 `previous_chapter_handoff` 传递。

## 最小生成顺序

```text
作者给出单元范围与目标
→ 从原始资料生成并确认单元上下文包
→ 以该上下文包为事实边界生成并确认单元总体规划
→ 各章共同读取两个 confirmed 产物
```

若总体规划阶段引入范围外内容，只回补受影响的上下文条目，不重做整包。候选资料包仍是临时材料，不进入每章输入。

## 与现有文件的关系

- “单元总体规划”直接承接并收缩现有 `unit_outline`，不新增第三种单元级产物；
- 现有单元上下文包模板中的规划字段应在确认后移出；
- 现有 V2 单元大纲模板中的事实复制和迁移审计字段应在确认后退出默认模板；
- P0 单元大纲示例只作为收缩样板的内容来源，不在本轮改写；
- 章节大纲／细纲继续负责单章结构与场景执行，不由单元总体规划替代。

## 用户确认

2026-07-22，用户确认：

1. “单元总体规划”就是收缩后的现有 `unit_outline`，不新增文件类型；
2. 单元上下文包先于总体规划生成，总体规划只通过 `context_ref` 使用事实；
3. 上述四组内容分别作为两个产物的默认最小正文接口。
