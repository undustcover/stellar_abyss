# 《stellar abyss》网络文化知识库

## 定位

本目录保存《stellar abyss》创作可调用的网络文化知识，包括热门梗、流行表达、网络句式、圈层黑话和人物适配信息。

当前只服务本作品，不作为平台级通用知识库。后续如需平台化，应在保留作品适配层的前提下另行抽象，不直接把本目录改成全平台公共库。

## 权威与可用性

1. 作者补充的条目默认立即可用，不等待正文验证或等级晋升；
2. 模型已有知识可以使用，但不联网检索，也不能声称掌握实时热度；
3. 作者条目与模型判断冲突时，以作者条目为准；
4. “可用”不等于“当前场景必须使用”，每次仍需检查人物、关系、情绪和语境；
5. 正文验证结果只记录使用效果，不撤销作者条目的默认权威；作者明确停用时才转为停用。

## 与其他知识的分工

|位置|职责|
|---|---|
|本目录|保存网络表达材料、含义、适用人物、场景和时效状态|
|`../../03_正文生成/作者经验归纳/关于梗的用法.md`|保存作者对梗如何进入正文的经验|
|`../../03_正文生成/笑话写作方法论库/`|保存如何从场景材料生成幽默的可执行方法|
|源头 Draft Skill|未来负责按章节需要稀疏选择知识与方法，不保存具体热梗|

知识库回答“有什么、谁能用、何时能用”；方法回答“怎样自然使用”；Skill 回答“这次是否调用”。三者不得合并。

## 分类方式

知识条目同时按以下维度分类，不只依赖文件夹位置：

- 表达类型：热门梗、流行句式、缩写黑话、平台表达、网络化比喻、反鸡汤、对话结构；
- 文化圈层：大众、职场、游戏、二次元、直播、电商、情感、家庭、校园及其他；
- 载体：叙述、人物内心、直接对白、群体对白；
- 人物适配：年龄、职业、性格、网络使用习惯和关系位置；
- 使用场景：吐槽、争吵、自嘲、尴尬缓冲、异常合理化、紧张余波等；
- 生命周期：当前可用、热度下降、年代限定、过时、作者停用；
- 验证证据：未在正文验证、单例有效、重复有效、失败记录。

## 运行时边界

后台可以持续增长和细分，但普通 Draft 不全文读取本库。生成时只选择与当前人物和场景直接相关的少量条目；没有合适条目时可以完全不调用。

不设置每章或每场景的使用配额，不把“现代感”误写成强制塞梗。
