## 二、当前总则

### 2.1 系统源驱动原则

正文、大纲、细纲、执行包等生成链路必须由系统资料驱动。

允许作为生成依据的资料：

1. 设定系统；
2. 事件数据库；
3. 叙事系统；
4. confirmed 单元上下文包；
5. 章节大纲、章节细纲、章节执行包；
6. Skill 系统内已批准的规则文件；
7. 用户当轮明确需求，但不得覆盖已确认世界事实。

禁止作为生成依据的资料：

1. 外部成稿；
2. 成品样文；
3. 对比报告；
4. 成品梗；
5. 成品对白；
6. 成品落点；
7. 非系统写作材料中的段落顺序或标志性表达。

如果临时输入中包含非系统写作材料，必须标记为已排除；资料不足时记录缺口，不用外部材料补洞。

### 2.2 世界事实与叙事安排分层

|层级|负责内容|不得越界|
|---|---|---|
|设定系统|人物、组织、能力、地点、规则等世界内事实|不得写成章节服务说明|
|事件数据库|世界中真实发生的事件、因果、状态变化、信息分层|不得写章节安排、读者体验或正文桥段|
|叙事系统|单元、章节、正文等叙事呈现方式|不得改写世界事实|
|Skill 系统|资料调用、生成流程、模板规则、质量检查|不得保存小说世界事实本身|
|总控台|模板、总表、索引、开发进度|不得替代模块正文|

### 2.3 时间轴原则

人物卡、事件卡、设定相关文件描述世界事实时，必须使用世界内时间或明确时期，不用章节、单元、剧情功能来组织人物状态或事件事实。

章节、单元、细纲和正文可以组织叙事呈现，但不能反向改写设定系统和事件数据库中的世界事实。

```text

    模板库 / 开发总表 / 总入口索引

    单元上下文包 / 单元大纲 / 章节大纲 / 正文

    事件卡 / 因果链 / 状态变化 / 信息分层

04_Skill系统
    资料调用 / 大纲创建 / 正文生成 / 规则设计 / 测试 skill 需求
```

当前生成主链路：

```text
设定系统 + 事件数据库 + 叙事系统
        ↓
资料调用 Skill
        ↓
confirmed 单元上下文包
        ↓
单元大纲 / 章节大纲
        ↓
章节细纲 Skill
        ↓
章节执行包 Skill
        ↓
正文生成 Skill（内置分场景导演、去 AI 感、文风一致性、对白幽默）
        ↓
正文草稿 + 生成备注 + 局部替换建议
        ↓
失败场景返修保险（草稿诊断 -> 分场景再导演 -> 正文局部替换）
```

1. 新增模块；
2. 新增关键需求文档；
3. 新增模板；
4. 新增正式样板；
5. 修改总体链路；

7. P0/P1/P2 todolist 状态变化；

更新方式：

1. 先改对应模块文件；
2. 再更新本文件的模块 Index；
3. 再更新 Todolist 状态；
4. 最后追加一条变更记录。

|拆分目标|建议文件|
|---|---|
|详细变更记录|`00_总控台/CHANGELOG.md`|
|详细 todo 看板|`00_总控台/TODO_系统开发待办.md`|

当前阶段先保持单文件，方便快速总览。
