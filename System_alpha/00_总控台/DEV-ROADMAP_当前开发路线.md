# System_alpha 当前开发路线

## 1. 当前阶段

```text
阶段：P2 P0C3 真实章节产品验证
状态：进行中
当前任务源：本文件
```

当前不继续扩展 Draft-to-Public、审美二次改写或新的后处理链路。先确保开发入口、正式/测试边界和一次生成 Draft 的最小输入明确，再进行新一轮章节验证。

## 2. 当前目标

建立一个不会因窗口切换、旧文档或测试分支而漂移的最小开发闭环：

```text
用户当轮目标
+ 必要世界事实
+ 必要叙事结构与视角选择
+ 前章承接状态
+ 最小人物意图边界
+ 简洁创作 Prompt
    ↓
一次生成 Draft
    ↓
用户修改与评价
    ↓
问题归因
    ↓
局部规则 / 通用规则候选 / 上游资料修正
    ↓
回归验证后晋升
```

## 5. P2 任务：真实章节验证与迭代

- [x] 冻结基线章输入，由 Codex 不修改旧 Skill、按新链路人工生成测试 Draft 和紧凑日志；
- [ ] 作者只审阅最终 Draft，以自然语言评价或直接修改；
- [ ] Codex 通过定位地图为每个问题确定一个首要责任位置；
- [ ] 一次只修首要位置并重新生成，直到作者确认可接受 Draft 基线；
- [ ] 基线通过后再修改测试 Skill，用同一冻结输入检查能否复现；
- [ ] 判断每项修改属于本章局部、人物长期、作品级还是系统通用；
- [ ] 只把有证据的抽象方法写入规则候选；
- [ ] 用相邻章节或保留输入回归，防止只适配基线章；
- [ ] 用户确认后晋升有效规则。

当前唯一下一步：在新对话窗口中，以 `测试文本/正文测试/第一单元/P0修订版总结构.md` 和已确认的 `P0前三章大纲_旧案主线对齐讨论稿.md@1.0` 为输入，重新讨论并梳理完整 P0 大纲。旧版具体对白与人物动作示例只作情绪和关系参考，不直接作为成品执行要求。

P2 完成标准：同类错误在后续章节明显减少，同时没有因规则膨胀导致正文更僵硬。
