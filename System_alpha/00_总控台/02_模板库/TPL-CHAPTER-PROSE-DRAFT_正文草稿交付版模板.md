# 正文草稿交付版模板

正文草稿交付版模板用于承载最终小说正文、上游来源索引和生成备注。

它不是正文内容套路，不规定正文句式、场景写法或情节表达方式。正文内容必须由正文草稿生成 Skill 在章节创作总控规划约束下，直接调用章节级叙事表达契约、章节细纲、场景演出契约、章节执行包、人物意图结果和章节运行态生成。

本模板的核心作用是：

1. 保证 `## 正文` 区纯净，只放小说正文。
2. 保证正文可追溯到确认后的上游产物。
3. 用 `## 生成备注` 记录总导演整合情况、风险、缺口和返修建议。

## 文件命名

```text
DRAFT-{单元编号}-{章节编号}_{章节名}_正文草稿.md
```

示例：

```text
DRAFT-P0-C03_这事好像比会所还怪_正文草稿.md
```

## 正文区规则

正式交付实例中，`## 正文` 下只能写小说正文。不得插入：

- 场景编号；
- 场景功能；
- 人物意图；
- 执行包建议；
- 上游产物摘要；
- 分析说明；
- 表格；
- 作者未要求的批注。

## 生成备注规则

`## 生成备注` 是系统交付版的质检区，不是读者正文的一部分。

备注应短、准、可行动。没有风险时可写 `未发现明显风险`，不得为填表而虚构问题。若作者要求作者阅读版或纯正文版，可只保留标题和 `## 正文`，但系统交付版默认保留 `## 生成备注`。

`Public 去 AI 预防映射` 必须记录实际读取的 `方法 ID@版本`；`去 AI 预防映射同步状态` 只能写 `current` 或 `stale`。若为 `stale`，不得继续交付正文，必须先同步或请求作者确认。

## 来源边界规则

正文只能使用系统资料和作者当前明确要求。不得使用外部正文草稿、对比报告、成品梗、成品对白、成品落点、标志性句式或非系统写作材料作为生成依据。

若输入中混有非系统材料，正文生成时必须排除，并在 `非系统资料污染风险` 中说明。
