# 参考文风蒸馏模型：《捞尸人》起点样本 V0.1

## 1. 模型定位

这是参考语料模型，不是当前小说作者文风规则。它回答“这三章的文字通过什么机制成立”，供下一阶段转换成全新实例让作者判断。任何机制在作者校准前均为 `observed`，不得进入 Draft/Public 运行时。

适用域：乡土生活与灵异事件交叠的开篇；多人物家庭/熟人社会；儿童与成人有限视角；日常—惊悚—民俗处置的连续场景。

## 2. 核心机制

### M01 对象的关系承载

先让物件在现场被使用、分配、拒绝、保留或损坏，再让这些变化显出人物的经济处境、偏爱、债与体面。

- 成立条件：对象本来就在场，人物对它有现实需求，处置会影响别人。
- 失败边界：为了表达情绪临时塞入可揉、可捏、可摔的物件；每次情绪都制造物理后果。
- 证据：E01、E04、E10。
- 置信度：高。

### M02 以关系任务生成对白

台词首先完成现场任务：争、哄、试探、催促、遮掩、讲价、求助或立威。人物声音由关系位置、所知信息与想达到的结果共同生成。

- 成立条件：明确说话对象、双方关系、信息差和当前目的。
- 失败边界：只给所有人统一增加方言、粗口、省略句；用口癖代替人物差异。
- 证据：E02、E10。
- 置信度：高。

### M03 被问题激活的说明

允许较长说明，但说明应回答场景刚提出的问题，改变读者对当前行为或风险的理解，并尽快回到正在发生的事。

- 成立条件：没有该信息，当前选择、规矩、危险或笑点无法成立。
- 失败边界：人物一出现就附履历；在危险动作中插入与当前判断无关的百科段。
- 证据：E03、E10。
- 置信度：高。

### M04 目标—操作—反馈动作链

把场景推进写成角色为解决问题而操作具体对象；环境或他人给出反馈，反馈迫使下一步行动。

- 成立条件：角色有可执行目标和现实阻力。
- 失败边界：连续堆表情、手势和身体反应，却不改变信息、空间、物件或关系。
- 证据：E04、E06。
- 置信度：高。

### M05 正常规则只偏移一步

恐怖先依赖读者熟悉的物理或生活规则，再让一个局部事实违反它；随后用可见/可听/可触的反馈逐步证明异常确实存在。

- 成立条件：先建立正常基线；异常可被 POV 按顺序发现。
- 失败边界：直接堆“阴森、诡异、恐怖”；同时出现太多无关异常；复用湿发、滴水、黑影等标志性意象。
- 证据：E05、E06。
- 置信度：高。

### M06 感知修正式揭示

让人物先依据有限感知形成暂时判断，再因距离、角度、动作或新反馈修正判断。信息揭示的顺序就是紧张感的顺序。

- 成立条件：误判符合人物经验，不靠故意降智。
- 失败边界：叙述者明知关键信息却用含混代词拖延；反复“原来”式硬翻转。
- 证据：E06、E05。
- 置信度：高。

### M07 功能性句长与断点

社会因果、人物关系和操作逻辑可用长句讲顺；命令、事实翻转、感知落点和危险确认则缩短或独立成段。短不是目的，改变阅读速度才是。

- 成立条件：断点前后存在认知转向或动作压力变化。
- 失败边界：为“像网文”而逐行断句；每次高潮都使用同一组重复与省略号。
- 证据：E07、E12。
- 置信度：高。

### M08 同一生活逻辑中的幽默与恐怖

幽默来自人物的贫富、体面、身体处境、职业经验和现实欲望；它可以在恐怖前建立活人感，在高压后恢复生活连续性。

- 成立条件：笑点是该人物此刻真的会想到或在意的事。
- 失败边界：独立段子；为反差强行毁掉哀伤；每个高潮后都抖包袱。
- 证据：E08、E02。
- 置信度：中高。

### M09 旧元素换功能的回扣

前置元素先在日常中完成一次自然功能，后续进入新的因果关系并改变意义。回扣依赖功能转化，而不是原词重复。

- 成立条件：前置元素值得人物当时注意；后续变化由事件造成。
- 失败边界：预埋时故意加粗；照搬参考作品的歌舞、水草、供品、吃席链条。
- 证据：E09、E12。
- 置信度：高。

### M10 背景在选择时兑现

人物历史和熟人社会关系只在收费、求助、信任、冒险、站队等当前选择发生时成为有效信息。

- 成立条件：背景能解释“为什么这个人此刻这样做”。
- 失败边界：与选择无关的完整人物小传；用旁白替人物宣布品德。
- 证据：E10、E01、E02。
- 置信度：高。

### M11 叙述距离随任务切换

体验危险时贴近主要感知者；解释乡土规则时适度拉远；补足成人判断时短暂进入其认知。切换服务于理解，不在单一动作瞬间任意跳头。

- 成立条件：每次拉近/拉远都有明确的信息任务。
- 失败边界：把本模型误写成严格单一限知；同一瞬间在多人内心间跳转。
- 证据：E11。
- 置信度：中高。

### M12 在事实改变处停止

段落或章节可停在新事实刚被确认、人物尚未完整解释和消化的位置。下一段再承担反应、追因或后果。

- 成立条件：读者已经能准确说出“刚刚改变了什么”。
- 失败边界：只藏名词、用代词或省略号制造假悬念；所有小段都做钩子。
- 证据：E12、E07。
- 置信度：高。

## 3. 场景模式

|模式|组织顺序|可迁移核心|高风险表面|
|---|---|---|---|
|家庭日常|资源/家务动作 → 关系摩擦 → 外部消息进入|用具体分配显关系，让消息经闲谈进入|方言骂人、多人抢吃食|
|知识/民俗说明|现场问题 → 行家判断 → 必要规则 → 操作验证|规则必须改变当下行动|捞尸黑话、祭祀器物|
|追逐/处置|目标 → 操作 → 反馈 → 改策|动作链和反馈闭环|撑船、撒网、铃铛|
|近身恐怖|正常对象 → 局部偏差 → 感知修正 → 触身后果|异常只偏移一步，按感知递进|湿发、滴水、黑影|
|高压后转阀|危机暂结 → 人物恢复现实欲望|用人物生活逻辑恢复活人感|固定章尾笑话|

## 4. 参考作者标志与禁迁项

以下即使在样本中有效，也不作为深层机制迁移：

- 大量乡土称谓、特定骂法和方言尾音；
- 灵异水域、直立死倒、湿旗袍女人等意象组合；
- 重复“她……”、连续断行、拟声词推进的具体节拍；
- 香烟、吃席、白事班子、供桌等高辨识物件链；
- 以粗粝身体笑话紧邻恐怖的固定配方。

## 5. 留出验证结果

|留出|验证的机制|结果|新增边界|
|---|---|---|---|
|H1 / S04|M02、M03、M10、M11|通过|人物声音差异不能只归因于方言；叙述会直接解释关系史|
|H2 / S09|M01、M03、M04|通过|程序动作可以较长；关键是每步改变资源、隐蔽性或风险|
|H3 / S10|M05、M06、M07、M08、M09、M12|通过|高潮短句与声音重复很显眼，必须禁止表面复刻|

没有发现足以推翻核心机制的留出证据。冲突主要集中在“现场呈现”与“直接解释”并存，已将模型修正为“被现场问题激活的说明”，而不是“少解释”。

## 6. 下一阶段输入

T3 只能从 M01—M12 中选少量机制，使用全新人物、物件、场景和句序生成作者测试卡。首批宜测试：

1. M03：说明是否被现场问题激活；
2. M04：动作是否形成目标—反馈链；
3. M07：句长是否服务认知转向；
4. M02：人物声音是否来自关系任务。

不宜首批测试 M05/M09 的恐怖意象，因为最容易误学表面风格。

## 7. T3 失败回流注记（2026-07-19）

首轮 T3 将 M03 转换为陌生人物自足段落，先产生“完整解释式 AI”，重做时又把 M02/M07 表面化为“短句机锋式 AI”。作者两轮均明确否决。

证据复核后的限制：

- M03 不能等同于角色在现场问题后把原因简洁讲清；参考文本中的说明常由叙述承担，并依赖已建立的连续场景。
- M02 不能等同于每句对白都主动显示人物性格；参考人物声音由多轮关系、资源、称谓与选择累积。
- M07 不能转换成普通交流中的短、断、利落；短句主要用于转向、命令和危险确认。
- “少量文字建立场景和人物”尚未被本模型充分解释，不得以提高单句功能密度代替重新蒸馏。

详见 `AUDIT-T2到T3转换失败复核-V0.1.md`。T3 测试设计已经重建为 `DESIGN-T3-作者实例测试设计-V0.2.md`；该设计优先于“首批宜测试”的旧排序建议。后续不得直接按 M03→M04→M07→M02 顺序机械造例，必须先建立关系与现场基线并通过内部反审计。
