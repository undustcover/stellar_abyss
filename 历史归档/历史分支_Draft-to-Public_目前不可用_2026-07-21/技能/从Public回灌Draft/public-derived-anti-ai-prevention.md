# Public 去 AI 化方法的 Draft 预防映射

## 一、职责

本文件把 Public 阶段已由作者确认的去 AI 化方法转换为首次 Draft 生成时的预防动作。

- 正式方法及边界的单一权威源仍是表达重构方法论库；
- 本文件只保存 Draft 阶段如何预防，不复制 Public 的修复正文、章节方案或成品句；
- `director-prose-quality-rules.md` 保存生成时的通用执行规则；
- `quality-check.md` 保存交付前门禁；
- Public 方法属于事后诊断与重构，Draft 映射属于首次生成预防，两者职责不同但必须同步。

## 三、Draft 预防规则

### 3.1 说明式旁白预防

生成旁白前，先区分：

- 现场可见事实；
- 人物没有做的事；
- 作者希望读者理解的设计意图；
- 希望读者产生的读感。

正文优先写可见事实。人物目的和关系位置能由动作、称呼、站位、物件交接、回答对象或后果承担时，不再补写“没有做什么”“这说明什么”。

禁止默认生成：

```text
动作
→ 未发生动作说明
→ 动机解释
→ 关系或主题结论
```

### 3.2 限知 POV 参与预防

关键场景生成前，建立最小接收链：

```text
现场刺激
→ POV 最先注意到的内容
→ POV 的一个有效参与拍
→ 必要时产生人物专属联想或判断
→ 下一动作或对白返回现场
```

有效参与拍可以是：

- 身体反应；
- 短对白；
- 选择性注意；
- 当下决定；
- 能改变读者接收的即时内心判断。

这不是数量门禁。快速过场不强制插入心理，关键场景也不要求每个动作后都有 POV 反应。

### 3.3 人物专属联想预防

需要比喻、形容、吐槽或内心想法时，按以下来源顺序调用：

1. 人物已确认的职业经验；
2. 人物长期生活压力；
3. 人物欲望、羞耻、恐惧或现实困境；
4. 人物稳定误读方式；
5. 人物常用词汇和关系经验；
6. 当前场景已有物件、动作、程序或关系压力。

联想必须同时通过：

- **换人测试**：原样换给其他主要人物时应明显失真；
- **场景关联检查**：人物经验与当前场景必须共享具体维度；
- **信息边界检查**：不能借内心联想提前揭晓真相；
- **事实检查**：不能为制造专属性而新增未经确认的职业史、生活史或关系史。

## 四、生成前小卡

只在关键场景或高风险片段内部建立，不写入正文：

|字段|填写要求|
|---|---|
|POV 刺激|人物最先注意到什么|
|注意原因|为什么是该人物注意到|
|参与拍|动作、短对白、选择性注意、决定或即时判断|
|人物经验来源|职业、生活压力、欲望、误读方式、常用词汇或关系经验|
|现场连接|与哪个物件、动作、程序或关系压力相连|
|返回现场|联想后由什么动作或对白继续|

若人物经验来源在系统资料中不存在，不得自行补造；改用动作反应或直接写现场。

## 五、交付前门禁

|检查|失败信号|处理|
|---|---|---|
|说明式旁白|一句同时讲动作、未发生动作和意义|删去解释，让现场动作重新承担|
|POV 参与|关键场景中 POV 只有看到／听到|补一个有读者接收作用的参与拍|
|换人测试|联想原样换人仍成立|回人物已确认经验重新生成，或删除联想|
|场景关联|联想虽然属于人物，却与当前现场无关|寻找共享具体维度；找不到则直写事实|
|返回现场|内心段独立表演，场景停住|用动作、对白、物件或后果立即接回|
|过度绑定|连续出现“看见／觉得／想到”或每段强插心理|删除无后果参与，只保留关键拍|
|事实与信息|为专属联想新增经历或越过认知边界|撤销新增，回已确认资料|

## 六、Public → Draft 同步协议

### 6.1 触发条件

出现任一情况即触发同步检查：

1. 正式表达方法新增、拆分、合并、升级、降级或修改边界，并涉及旁白、POV、人物声音、比喻、心理、轻喜或去 AI 化；
2. Public 作者审美门禁在上述领域新增或修改；
3. 作者在 Public 校准中确认的新退化，本可在首次 Draft 生成时预防；
4. 同类问题在两个 Public 片段重复出现；
5. Draft 质量门禁已存在，但 Public 仍重复发现同类问题。

### 6.2 同步步骤

```text
Public 作者确认新判断
→ 更新正式表达方法及其方法版本
→ 判断 draft_prevention: yes / no
→ 若 yes，更新本映射
→ 同步 director-prose-quality-rules.md
→ 同步 quality-check.md
→ 必要时同步 SKILL.md 与 Draft 模板生成备注字段
→ 更新 source_methods、last_synced 和同步日志
→ 在下一次干净 Draft 或专门 Draft 冒烟测试中复验
```

同步只迁移抽象规则，不允许把 Public 成品句、章节桥段、固定比喻或段落顺序喂回 Draft。

### 6.3 版本与失配处理

- 本文件以 `方法 ID@方法版本` 记录来源，不以整个方法库版本判断，避免无关条目变化造成误报；
- 正式来源中标记 `Draft预防映射: yes` 的方法发生变化时，必须提升方法版本；
- Draft 执行前比较正式方法版本与本文件 `source_methods`；
- 版本一致：`current`；
- 正式方法较新：`stale`，不得静默继续生成；
- 发现 `stale` 时，报告 `SYNC-DRAFT-DEAI-001`、变化来源和受影响规则。若当前任务授权系统维护，则先同步再生成；否则停止并请求作者确认是否先同步。

### 6.4 复验要求

- 使用干净系统输入，不读取 Public 成品正文；
- 优先在下一次 Draft 高风险片段检查同类退化是否被首次生成拦截；
- 复验失败时，主归因在 Draft 执行规则、Draft 门禁或映射缺失中三选一；
- 不以“Public 后续还能修”作为 Draft 门禁放行理由。
