# 第一章AI测试稿与作者修正版对比报告

## 总体判断

作者修正版不是在原稿上“增加几个笑话”，而是重建了整章的喜剧机制。

AI测试稿的问题是：按大纲完成了事件，但每个事件之间像“说明书式推进”。笑点多来自旁白比喻和吐槽句，人物只是执行大纲。

作者修正版的问题解决方式是：让每个事件都从生活压力、人物反应、饭桌沉默、话题岔开、对话找补里自然长出来。笑点不是单句抖机灵，而是连续错位。

## 结构差异

|维度|AI测试稿|作者修正版|可反推规则|
|---|---|---|---|
|开场压力|直接给出工作群、甲方、领导三点压力|先把“下班后工作群还亮着”生活化，再用血压仪、首页颜色、需求玄学逐层加压|开场不能只交代处境，要把处境写成连续可感的生活荒诞|
|家庭饭局|较快进入亲戚比较|把亲戚饭局写成“另一种工作群/绩效面谈”，形成职场压力的镜像|场景之间要有主题延续，不是换地点重开一段|
|马木出场|完成消音和造型描写|增加童年反差：过去马木来大家热闹，现在大家改话题|人物出场要带历史落差，哪怕只露一点，不要只有外貌标签|
|职场指导|马木像半懂不懂的社会人，偶尔说中|马木把职场服务讲歪成会所迎宾，直接把后文云水汇误会提前种进语言逻辑|关键段落要承担多重功能：人物喜剧、误会铺垫、后文词汇预热|
|火柴盒|作为物件出现，引发怀疑|火柴盒接在“手牌/服务意识/正规”之后，误会更顺|伏笔物件不要孤立出现，要接在前文语义链上|
|电话结尾|按大纲词汇完整保留|删减和替换部分词汇，让电话更像被偷听到的碎片；最后用“去救人/自投罗网”落钩子|电话碎片要优先自然和误读张力，不必机械保留全部原词|

## 文风差异

### 1. 作者稿更接地气，因为它把抽象吐槽落到具体经验

AI测试稿常见写法：

|类型|问题|
|---|---|
|“成年人在工作群里的尊严，通常只剩下一个收到”|句子顺，但像泛用金句|
|“亲妈救人用排骨”|有梗，但前后铺垫短|
|“像一道没有明说的家规”|解释感偏重|

作者稿处理方式：

|做法|效果|
|---|---|
|工作群亮着 = 半夜有人敲门，后者可能是鬼，前者一定比鬼难缠|生活感更强，起手有读者共鸣|
|公司血压仪不是福利，是预警设备|具体物件带出职场荒诞|
|甲方负责玄，乙方负责学|把“需求玄学”压成一句可传播的笑点|
|亲戚聊天像甲方提需求|把职场和家庭两条压力线接上|

可反推规则：正文skill不能只要求“幽默吐槽”，要要求“吐槽必须绑定具体生活物件、具体职场流程、具体饭桌动作”。

### 2. 作者稿的幽默来自连续递进，不是孤立比喻

AI测试稿经常是一句好笑后就进入下一个功能点。

作者稿会把一个梗连用三到四步：

|连续链条|作用|
|---|---|
|工作群还亮着 -> 血压仪 -> 首页颜色三天 -> 整体感觉差一点 -> 需求玄学 -> 甲方负责玄，乙方负责学|把职场压力铺成一组笑点，而不是一句话带过|
|亲戚比较 -> 妈妈眼神 -> 工资问题 -> 排骨救场 -> 年度绩效面谈 -> 亲戚绩效指标|家庭饭局形成完整小喜剧结构|
|服务意识 -> 二楼贵宾一位 -> 茶、毛巾、手牌 -> 最后一句可以忽略 -> 木哥经验来源复杂|马木吹牛段形成真正的场内包袱|

可反推规则：执行包需要标注每个核心笑点的“铺垫-升级-误导-落点”，不能只写“本场要幽默”。

### 3. 作者稿的对话更自然，因为人物会找补、抢话、岔开

AI测试稿对话偏功能性：

- “你还懂方案？”
- “我不懂方案，我懂人。”
- “那你说怎么办？”
- “然后他就慌了。”

这些句子能推进，但像给设定搭桥。

作者稿对话更生活化：

- 马木喊“二楼贵宾一位！”
- 被饭桌空气打住后，自己补“最后一句可以忽略。”
- 马雯雯接“木哥，你这经验来源挺复杂啊。”
- 马木再用“服务意识”找补。

这里的对话不是传递信息，而是人物当场失误、当场补救、旁人尴尬、马雯雯接刀。

可反推规则：正文skill应要求关键对话至少包含一种自然断裂：抢话、找补、转移、沉默、误解、话说过头后撤回。

## 情节推动差异

### AI稿推动方式

AI稿基本按“事件A发生 -> 主角解读 -> 事件B发生”推进。

例子：

- 马木出场
- 大家安静
- 马木讲职场
- 火柴盒出现
- 电话响

这个顺序没错，但缺少事件之间的内在滑动。

### 作者稿推动方式

作者稿的推进更像“前一个笑点自然滑进下一个误会”：

- 职场被甲方折磨
- 家庭饭局也像甲方
- 马木说“让人折磨了”
- 马木讲客户要服务
- 服务讲歪成会所迎宾
- “手牌”提前出现
- 火柴盒出现后，云水汇误会自然坐实
- 电话里的“几号/里面/加/跑不了/留名”不再突兀

可反推规则：章节执行包需要加入“误会语义链”：本章每个核心伏笔前面要有语言或行为铺垫，不能让伏笔孤零零出现。

## 环境描写差异

作者稿其实不是没有环境描写，而是环境描写大多承担喜剧功能。

|AI测试稿|作者修正版|
|---|---|
|“电视里综艺嘉宾还在笑，笑声忽然显得特别假”|保留，但不滥用|
|“灯光下一段金一段铜”|保留，并强化为“高端路线/批发市场路线”|
|饭桌空气变化|作者稿更多用筷子、鱼、排骨、茶杯承载，不单独写氛围|

可反推规则：正文skill应限制环境描写：只有三类可写，1）推动误会的物件，2）暴露人物反应的动作，3）能参与包袱的饭桌道具。纯气氛描写删掉。

## 关键增强点

### 1. “二楼贵宾一位”是本章喜剧核心新增点

这是作者稿最关键的重构。

它的功能很多：

|功能|说明|
|---|---|
|人物塑造|马木的社会经验来源变得可疑又好笑|
|会所误会铺垫|在火柴盒出现前，读者已经被引到服务/迎宾/手牌语境|
|对话自然化|马木说过头，自己找补，饭桌尴尬|
|后文承接|云水汇出现不再突然|

这说明 skill 需要鼓励“核心笑点支架”，而不是只根据大纲生成情节。

### 2. 结尾从“业余生活挺满”改成“去救人/自投罗网”

AI稿结尾更像完成大纲指定句，笑点较平。

作者稿结尾更强：

- 马木说“去救人”
- 马雯雯不信
- “他到底是去救人，还是去自投罗网？”

这个结尾更好，因为它同时保留三层：

|层级|效果|
|---|---|
|表层误会|马雯雯仍觉得他跟不正经场所有关|
|真实事件|马木确实是去救人|
|后文反杀|“自投罗网”在第11章后会变成冷钩子|

可反推规则：正文skill应允许作者重构式结尾，只要三层功能更强，不必死守大纲原句。

## 对skill优化的直接结论

### 1. 不是新增“幽默skill”优先，而是先新增“作者风格校准skill”

原因：作者稿证明问题不只是“缺笑话”，而是缺作者的喜剧组织方式。

建议新增：

`calibrate-author-prose-style`

它从作者修正版中提取：

- 起手方式
- 吐槽来源
- 对话断裂方式
- 包袱递进方式
- 环境描写可用范围
- AI感禁用句式
- 章节钩子风格

### 2. 正文生成skill必须大改

新增强制要求：

|规则|说明|
|---|---|
|每个核心场景至少有一条生活经验链|例如工作群 -> 甲方 -> 家庭饭局|
|每个核心笑点必须有铺垫、升级、落点|不能只写一句俏皮话|
|关键对话必须包含自然断裂|找补、抢话、沉默、岔开、误解|
|环境描写必须参与动作或笑点|纯氛围描写默认删|
|允许替换大纲原句|只要保留功能并增强三层结构|

### 3. 章节执行包skill需要补“喜剧机制表”

建议加入：

|场景|生活压力来源|误会语义链|笑点铺垫|升级方式|落点|后续反杀材料|
|---|---|---|---|---|---|---|

### 4. 章节细纲skill只需轻改

细纲不应该写太多文风细节，否则会把正文写死。

只加两项即可：

- 本章核心喜剧机制
- 本章误会语义链

## 下一步建议

先不要直接改4个skill。

建议下一步先做两份中间文件：

1. `作者第一章风格规则提取.md`
2. `正文生成skill修改草案.md`

等这两份确认后，再改 `generate-chapter-prose-draft`。执行包和细纲放在第二轮改。
